## 1️⃣｜反转钩：怕被 AI 取代？你可能慌错方向了（借势裁员数据）

**痛点角度**：天天看裁员新闻焦虑，怕码农被 AI 取代，但其实被取代的是"不学 AI 的人"。

**标题**：😶‍🌫️ 怕被 AI 取代而不敢学 AI，这才是最大的坑

**正文（约 500 字）**：

今年群里最常见的一句话是："还学啥码，迟早被 AI 取代。"

我特意去扒了下数据，结果有点反直觉。2026 第一季度科技行业裁了差不多 5.2 万人，里面将近一半确实跟 AI 有关——但同一时间，软件工程师的岗位 listing 反而涨了 30%，是三年来最高。

矛盾吗？一点都不。公司裁的是"只会写 CRUD、AI 一键就能替的活"，招的是"能把 AI 搭起来、部署上线、盯着它别出事"的人。说白了，不是不要工程师了，是不要"只会那一套老活"的工程师了。

真正被取代的，从来不是会用 AI 的人，是死活不肯碰 AI 的人。

那 entry-level 是不是更难了？是的，初级岗这两年缩了快 28%，AI 把很多"junior 顺手能干"的活吃掉了。但这恰恰说明：你得让自己跨过"junior 能被替"的那条线，往"会 build AI 系统"那边站。

具体就是把重心从"刷题进大厂写后端"挪到 AI 应用这条线：prompt、RAG、agentic workflow、调 LLM API、能部署能监控。这几样对有代码底子的人来说，真没那么难。

我自己就是从 SDE 转过来的，转完最大的感受是——焦虑感一下子没了。因为你不再是"等着被替"的那个，你变成了"搭这套东西"的那个。

匠人 AI Engineer Bootcamp 整条线就是冲着"会 build、能落地"设计的，从 Context Engineering 一路到 Multi-Agent，正好帮你从"会被替"挪到"造系统"那一侧。

别再用"反正会被取代"当不学的借口了，这话本身就是个坑。

## 2️⃣｜在职数据分析师，是 2026 最被低估的转 AI Engineer 起点

**痛点角度**：在职 DA / 数据人想跳槽进 AI，又怕自己"不够工程"，其实数据底子是 RAG 的天然优势。

**标题**：📊 做数据的别只盯着 DS 了，转 AI Engineer 你赢在起跑线

**正文（约 500 字）**：

经常有做数据分析的朋友问我：想转 AI，是不是只能往 Data Scientist 那条卷？

我的答案是：别只盯着 DS，AI Engineer 可能更适合你，而且你起跑线比想象中靠前。

为什么？现在 AI Engineer 最核心的活之一是 RAG（检索增强生成）。拆开看就是：把文档切块（chunking）→ 转成 embedding → 存进向量库 → 用户提问时检索最相关的内容拼进 context。

发现没有？这一整套全是"数据"的活。chunking 是数据清洗，embedding 和向量库本质是另一种 index，检索质量要看 recall、precision 这些你早就熟到不能再熟的指标。你有数据 bg，学 RAG 基本是无痛迁移。

你要补的其实就两块：一是工程化——把这套东西封成 API、能部署、能监控；二是 agentic workflow——tool calling、多步任务怎么编排。这两块补上，你就是个完整的 AI Engineer 了。

而且数据人有个隐藏优势：现在大家发现，AI 系统的效果好不好，很多时候不在模型本身，而在 evaluation（评估）。怎么系统地评一个 RAG 准不准、幻觉多不多——这种"用数据说话"的思维，恰恰是纯写代码的人最缺的。RAGAS、DeepEval 这些 eval 工具，你上手会比别人快。

所以真别妄自菲薄。在职转的好处是，你可以先在现有工作里找 AI 相关的小场景练手，攒着攒着简历就有料了。

匠人 AI Engineer Bootcamp 是按"有 Python / 数据基础"门槛设计的，RAG / GraphRAG、向量库、evals 这条线对数据人特别友好。

## 3️⃣｜2026 别再只学 RAG 了，拉开差距的是这两样（借势技能风向）

**痛点角度**：很多人 RAG 学完就停了，但 2026 招聘真正在看的新重点是 MCP 和 evals。

**标题**：🧭 2026 还在只学 RAG？这两样才是今年的分水岭

**正文（约 500 字）**：

去年大家都在卷 RAG，今年我得提醒一句：只会 RAG 已经不够了。2026 真正在拉开差距的，是另外两样——MCP 和 evals。

先说 **MCP（Model Context Protocol）**。简单讲，它是让 AI agent 连外部工具/数据的一套标准接口。以前每接一个工具都得自己写胶水代码，现在有了统一协议。这东西火到什么程度？到 2026 年 2 月，月下载量已经 9700 万，主流 AI 厂商全部支持，基本成了事实标准。现在的 JD 里越来越多直接点名要 MCP 经验。

再说 **evals（评估）**。这是最多教程跳过、但实际最值钱的一块。因为开源模型越来越能打，要不要换模型、prompt 改了到底变好还是变差、RAG 准不准——全靠 eval 说话，不是靠感觉。RAGAS、DeepEval、LangSmith 这些工具会用，你在团队里的话语权完全不一样。一个会做 eval 的人，公司是真的抢。

所以我现在建议的学习重点是这样排的：
1. RAG 打底（还是要会，但别停在这）
2. agentic workflow + tool calling
3. **MCP**——把 agent 接上真实工具
4. **evals**——能量化你搭的东西好不好
5. 工程化部署 + 监控

记住一句话：2026 大家都能 call API、都能搭个 demo，但"能证明自己搭的东西靠谱"的人很少。这就是机会。

匠人 AI Engineer Bootcamp 的大纲今年专门把 Multi-Agent + MCP、AI Evals 这两块拎出来做成完整模块，就是顺着这个风向来的。

## 4️⃣｜简历没项目？用这 3 个开源搭一个能过面的 agent（开源+简历）

**痛点角度**：学完了简历空空，没有能讲的项目；给一条"工程化 agent portfolio"的具体配方。

**标题**：🛠️ 转 AI Engineer 简历没项目？照这个搭一个就够了

**正文（约 500 字）**：

转 AI Engineer 最卡人的从来不是"学不会"，是简历里没有能讲的项目。今天给你一个我觉得性价比最高的 portfolio 配方——搭一个工程化的 agent，用全开源的东西。

**项目长什么样**：做一个多步 agent，接 3 个以上工具（比如代码审查 agent、调研助手、文档处理都行），能处理工具调用失败、能重试、有日志监控。这种"不是 demo、是能上线"的项目，面试官一眼就高看你。

**用哪些开源**：
- **LangGraph**——搭 agent 的状态机，多步流程出问题好定位，生产环境首选
- **MCP**——用它把 agent 接上真实工具，这是 2026 的加分项
- **DeepEval**（GitHub 搜 `confident-ai/deepeval`）——给你的 agent 跑评估，证明它靠谱
- 想找更多现成参考，GitHub 搜 `awesome-ai-agents-2026`，一堆 agent 案例和框架对比

**关键就一句话**：别只做个 demo 跑通就收工。往工程化再走一步——把 Docker、CI/CD、日志监控、简单回滚、重试机制带进去。这样你给面试官的感觉就不是"我会搭原型"，而是"我知道一个东西上线后会遇到什么问题"。这才是 AI Engineer 现在最值钱的地方：不是拼谁最懂模型，是拼谁更像一个能落地的 builder。

我自己复盘，简历上真正帮我拿到面试的，就是这种"看得出工程化思维"的项目，而不是刷了多少课。

## 5️⃣｜agentic AI 岗暴涨 280%，但你得知道去哪找、投什么（找工作+薪资）

**痛点角度**：海投 0 面试，不是岗位少，是没投对方向、关键词没踩准。

**标题**：🔍 海投没回音？2026 真正在招的是 agentic AI 岗

**正文（约 500 字）**：

很多人海投几百份没回音，就觉得"市场完了"。但你看数据会发现，不是没岗，是你没投对地方。

2026 一组很猛的数字：agentic AI 岗位同比涨了 280%，带 "LLM" 或 "RAG" 字样的岗位较 2024 年涨了 340%。钱也不少——agentic AI engineer 均薪约 $190k，做 RAG architecture 中级还能再加 10-15%。岗在那儿，关键是你怎么够得着。

**第一，投对关键词。** 别只搜 "software engineer"，那池子又卷又缩。去搜：AI Engineer、LLM Engineer、Agent Engineer、Applied AI、Forward Deployed Engineer。这些才是现在在放量的池子。

**第二，简历踩中现在的栈。** JD 里高频的：LangChain / LangGraph、CrewAI、Tool Calling、RAG、Vector Database、MLOps、Evaluation。你简历里没有这些词，ATS 直接把你筛了——不是你不行，是没对上关键词。

**第三，先进场再升级。** 别一上来就奔最难那层。市面上大量 AI 岗其实是"软件开发 + AI coding tools + prompt + RAG + agentic workflow"，比转 MLE 要啃的 NLP、深度学习简单太多。先拿到一个能进去的位置，进去了再往上走，比你在门外反复准备有效得多。

说句实在的，窗口期还在，但 entry 也在收紧，越早进场越好。
