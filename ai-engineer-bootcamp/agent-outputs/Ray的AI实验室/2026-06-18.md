## 1️⃣｜不同职业转 AI Engineer：谁能最快上岸

**痛点角度**：SDE / DA / BA / PM 都想转，但起跑线完全不一样，先搞清自己离 AI Engineer 还差几步。

**标题**：🚀 SDE/DA/BA/PM 想转 AI Engineer，谁跑得最快？

**正文（约 500 字）**：

这两年来问我转 AI Engineer 的人，背景五花八门，但说句实话——不是谁都一样快。有人三个月就能投简历，有人得先补半年底子。

我按背景给你拆一下。

**SDE / 软件开发**：底子最好，基本已经赢一半。你会写代码、会调 API、懂工程化，缺的就是 LLM 应用那一层——prompt、RAG、agentic workflow、调 LLM API。这些对比 MLE 要啃的 NLP、深度学习，真的简单太多。我当年从 SDE 转的时候，连 fine-tuning 都没学（内部转组，公司明确不需要），照样转过去了。

**DA / 数据相关**：也很顺。RAG 这块会涉及 database、embedding、向量库，你有数据 bg，学起来基本无痛。补一点工程化部署就能上。

**BA / 产品经理**：工程那层要补得最多，得从写代码、调接口开始。但你别灰心——你懂需求、懂落地、会和人沟通，往 AI 产品 / AI 应用方向走反而吃香。公司里真正能把 AI feature 落地的人，往往就缺这种既懂业务又懂技术的。

说到底，转得快慢不看出身，看你愿不愿意动手做项目。光看视频不动手，哪个背景都白搭。

匠人的 AI Engineer Bootcamp 其实就是按"有 Python/API 基础"这个门槛设计的，12 周把 Context Engineering → RAG → Agent → Fine-Tuning 这条链路打通，正好适合上面这几类人补差距。

## 2️⃣｜AI Engineer 到底要会啥 + 怎么入门

**痛点角度**：网上一搜全是劝你学 NLP/深度学习，被劝退；其实市面上大部分 AI Engineer 岗根本不考这些。

**标题**：📌 2026 转 AI Engineer 到底要会啥？这张清单先收好

**正文（约 500 字）**：

每次有人问我转 AI Engineer 要学啥，自己上网一搜，全是"先学线代、概率、深度学习、调参"，看完直接劝退。

但我想说句大实话：市面上大部分 AI Engineer 岗，根本不考这些。

现在 market 上 AI role 的真实要求，基本就是这一串：**software development ➕ AI coding tools ➕ prompt engineering ➕ RAG ➕ agentic workflow ➕ 调 LLM API**，部分公司有 sensitive data 要本地模型，才会让你学 fine-tuning。

对比转 MLE 要啃的那套 NLP、DL，是不是一下子轻松多了？这也是为什么我一直说：今年转 AI Engineer，真的比你想的容易。

入门顺序我建议这样走，别一上来就奔最难的那层：

1. **LLM 基础**——先搞懂大模型怎么回事、token、embedding 是啥
2. **Prompt + AI coding tools**——把 Cursor、Claude Code 这类工具用熟，这是日常
3. **RAG**——这是现在最主流的方向，会接向量库、做检索
4. **Agent / agentic workflow**——tool calling、多步任务编排
5. **工程化部署**——能把它封成 API、能部署、能监控

先把"能帮你过面试、能帮你进场"的那部分拿下来，进去了再升级。

匠人的 AI Engineer Bootcamp 大纲基本就是按这个顺序排的，Context Engineering → RAG/GraphRAG → Agent SDK → Multi-Agent → Fine-Tuning → AI Evals，一条链路下来，配 7 个能写进简历的项目。

别再被那些"先学三个月数学"的帖子吓住了，方向对了，入门没那么难。

## 3️⃣｜拆一个真实 AI Engineer JD：公司到底在招什么人（借势趋势）

**痛点角度**：JD 看不懂 / 不知道公司到底要什么样的人，结合 agentic AI 岗 +280% 的趋势。

**标题**：🔍 拆一个真实 AI Engineer JD：公司其实在招这种人

**正文（约 500 字）**：

最近 agentic AI 的岗位同比涨了 280%，光美国就有差不多 9 万个在招。但很多人点开 JD 一脸懵——写的东西看着熟，又说不清到底要会啥。今天我把一个典型 AI Engineer 岗位拆给你看。

JD 里高频出现的，基本逃不出这几块：**LLM orchestration、RAG pipeline、tool calling、evaluation、monitoring、guardrail**。

听起来很唬人，但拆开看就是：用户的 query 从哪进来 → 怎么做 retrieval → context 怎么拼 → response 怎么返回，再加上线后怎么盯效果（输出稳不稳、hallucination 多不多、latency 高不高、成本爆没爆）。

这里有个很多人没意识到的真相：**LLM 只是整条链路里的一个环节**，真正复杂、真正值钱的是前后那一堆 workflow。很多时候问题根本不在模型能力，而在 context design 和 workflow 本身——换个 retrieval 策略、调个 tool routing，效果就能差一大截。

所以公司真正缺的，不是"最懂模型"的人，是能把 API 变成一个可用产品的 builder。一句话：today everybody can call an API，but not everybody can build a usable product around it。

而且这岗位现在是真缺人——未来两年预计 130 万 AI 岗缺口，供给连一半都不到，应届起薪也不低。窗口期还在，别犹豫太久。

如果你想系统看懂这条链路每一环怎么搭，匠人 AI Engineer Bootcamp 就是按真实 JD 反推设计的，从 Context Engineering 一直做到 Multi-Agent，跟着搭一遍比你自己啃 JD 快多了。

## 5️⃣｜转 AI Engineer 最大的坎是简历不 match（找工作/改简历，借势市场）

**痛点角度**：技能学完了投简历还是没回音，问题在原经验和 AI role 不 match，结合传统码农岗缩招的市场现实。

**标题**：📄 转 AI Engineer 最大的坎不是学不会，是简历不 match

**正文（约 500 字）**：

见过太多人，RAG、agent 那套都学得差不多了，投简历还是石沉大海，然后开始怀疑自己是不是不行。

我跟你讲，问题大概率不在能力，在你的简历。

道理很简单：你原来是 SDE、DA、BA，那段 work experience 和 AI role 天然就不 100% match。HR 扫一眼对不上关键词，直接划走。这不是你菜，这是简历没改对。

怎么破？两步。

**第一步，用学的 skills 做项目把经验补上去。** 别只做个 demo 跑通就完事，往工程化再走一步——把 Docker、CI/CD、日志监控、简单回滚、权限控制这些带进去。这样面试官的感觉就不是"你会搭原型"，而是"你知道一个东西上线后会遇到什么问题"。这才是 AI Engineer 最值钱的地方：不是拼谁最懂模型，是拼谁更像一个能落地的 builder。项目方向就围绕现在最主流的：LLM app、RAG、agents、MCP。

**第二步，把过去经验重新包装成"AI 相关"。** 同样一段后端经验，换个角度写就能贴上 AI 的标签，再把结果量化出来。

顺便说下市场现实，免得你方向走偏：传统码农岗这两年在缩招，Stanford 的报告里 22-25 岁程序员岗位较 2024 年跌了近 20%；但 AI 岗反而在抢人。所以这时候硬卷传统 SDE 不如把火力集中到 AI 方向。
