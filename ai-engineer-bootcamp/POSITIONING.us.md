## 1. 一句话定位

> AI Engineer Bootcamp 美国班 = 面向在美华人开发者 / CS·Data 留学生 / OPT·STEM OPT 求职者的 **AI 工程项目求职营**——不是教你调 API，而是带你做出能对齐美国 JD、能写进简历、能在面试里讲清楚的 RAG / Agent / Eval / LLMOps 生产级项目。

差异化对打（美国市场真正的对手是 Springboard / Pathrise / Interview Kickstart / Formation，不是国内 bootcamp）：

> 他们做的是**面试刷题 + 投递辅导**（教你「怎么过面」）；我们做的是**真实生产级 AI 项目 + JD 对齐 + 面试可讲**（让你「真的会，所以过面」）。

## 2. 两个核心信任卖点（本次新增 · 重点）

### 2.1 风险反转：试听课后 24 小时内不满意全额退款

- **机制**：报名后先上**试听直播课**，**试听直播结束后 24 小时内**对课程不满意可申请**全额退款**。
- **计时起点**：以**试听直播结束**为准（不是回放、不是报名时间）。
- **流程**：走现有客服 / 报名渠道申请即可，**不另设专门退款入口**（保持简单，不过度工程化）。
- **为什么这条能用（合规）**：退的是**课程内容满意度**，不是就业结果——完全不碰「包就业 / 保 offer」红线。这是合规版的风险反转，替代美国受监管、且与「不承诺结果」冲突的 ISA。
- **文案口径**：「先听一场直播，再决定。试听直播结束 24 小时内不满意，全额退。」
- **唯一要写清的一句**：销售页 / 报名确认页标明「计时起点 = 试听直播结束」，避免「从看回放算」的歧义即可。→ 同步进 [`FUNNEL_PLAN.md`](./FUNNEL_PLAN.md) 引流课→主课转化环节。

### 2.2 讲师背书：澳洲 AI 课师资 + 美国一线大厂导师

美区买家最先问「凭什么信你能帮我在美国找到工作」——师资背书是第一道信任锚。

| 导师 | 背书点 | 角色定位 | 状态 |
|------|--------|----------|------|
| 澳洲 AI 课现有师资 | JR Academy 已跑通的 AI 课程教学团队，有成体系的 AI 工程教学经验 | 主讲 / 课程体系背书 | 沿用现有，可直接用 |
| **Leon 老师** | **Software Engineer · 美国 Google**，长期带 AI 课程 | **直播主讲**，讲美国 JD 真实要求、大厂 LLM 系统设计面试 | ✅ 已确认：title = Software Engineer · **不露脸**（不上头像/出镜）· 直播授课 |
| **Albert 老师** | **Software Engineer · 西雅图 Amazon**，长期带 AI 课程 | **直播主讲**，讲 production AI / 求职 referral / 面试 deep dive | ✅ 已确认：title = Software Engineer · **不露脸** · 直播授课 |

> 🔒 **两位美国导师不露脸**：对外物料（海报 / 销售页 / 讲座）**不用真人头像、不出镜**。可用「Software Engineer · Google / Amazon（在美）」文字背书 + 抽象 avatar / logo 区块替代。直播也以授课为主，不强制露脸。

- **为什么是强卖点**：澳洲师资证明「这套 AI 课教得出来」，美国 Google/Amazon 在职导师证明「懂美国本地 JD 和面试到底要什么」——一个证明教学力，一个证明本地相关性，正好补上美区最缺的本地可信度。
- **文案口径（合规版）**：「导师团队覆盖美国一线大厂在职工程师（Google / Amazon）+ JR Academy AI 课程教学团队，带你对齐真实美国 JD。」
  - ❌ 不要写「Google/Amazon 工程师保你进大厂」「跟着 Google 老师拿 offer」——这是背书，不是结果承诺。
  - ✅ 写他们**带来什么内容**（真实 JD 拆解 / 大厂面试视角 / production 经验），不写他们**保证什么结果**。
- **上线前必做**：拿到 Leon / Albert 本人确认的**对外可用 bio**（可写到什么程度的 title / 公司 / 是否露脸 / 是否实时参与），再上销售页。FTC 对师资背书的真实性要求高。

## 3. 还需要补强的吸引力杠杆（来自前序讨论）

按对转化的影响力排序：

1. **证据层（最缺、最值钱）**：脱敏的往期学员 GitHub repo / demo video / architecture doc 截图——作品本身就是广告。「这是学员第 N 周做出来的东西」比任何文案都强。
2. **AI-native 工作方式做成卖点**：你学完是「用 Claude Code / Cursor / agent 把自己放大 10 倍的工程师」，对齐 2026 美国 JD 越来越看重的「会不会用 AI 干活」。这是和老牌 bootcamp 的代差。
3. **对齐美国招聘节奏的 cohort**：踩 new grad 秋招 / OPT 窗口 / H-1B 抽签节点排课和 mock interview，「为 X 季求职窗口准备」紧迫感真实且合规。
4. **校友 referral 网络**：美国找工一半靠 referral，「进入在美华人 AI 工程师社群 + referral 互助网」是长期价值，利于续费 / 老带新。

## 4. 宣传切入点（钩子优先级）

第一波不从「AI 高薪」入手（俗且风险大）。建议：

1. **「美国 AI Engineer JD 到底要什么」**——10 个真实 JD → 技能词频 → 课程项目对应（最硬，先做）。
2. **「会调 API ≠ AI Engineer」**——讲清四层能力差距：API demo / RAG pipeline / production eval / observability。
3. **「OPT / H-1B 求职不是只投简历」**——合规求职策略，只讲打法不做签证承诺。
4. **「每个项目都能变成一段简历 bullet」**——少讲课时（286 节），多讲作品如何对应 JD 和面试。

每条钩子结尾都可挂上 §2 的两个信任锚：**试听不满意全额退** + **Google/Amazon 在职导师带你对齐 JD**。
