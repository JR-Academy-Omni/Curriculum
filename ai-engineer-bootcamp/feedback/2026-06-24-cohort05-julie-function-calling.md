## 5. 一处有争议的技术点（需复盘）

**学生问题**：模型该调用工具时不调用、自己脑补结果（如问天气不查直接答），System prompt 不能 100% 防住，工程上如何保证**确定性**？

**老师回应**："用什么模型？" → "那么聪明的模型不可能跑不出天气。"

**评估**：该回应**没答到点上**。这是工具调用可控性的工程问题，不是模型智商问题。倪99 的判断成立。更恰当的框架应包含：

- **`tool_choice` 强制调用**（最直接的确定性手段）：Claude 支持 `tool_choice: {type:"tool", name:...}` 或 `"any"`；OpenAI 同理。
- System prompt 只能提高概率，确实无法 100%（学生直觉正确）。
- **意图分类前置**：先判断是否需要工具，再确定性路由调用，而非全交给模型自主决策。
- 对模型输出做**校验 / guardrail**，不合规则重试或回退。
- few-shot 示例、必要时微调。

> 该点同时暴露了 4.3 之外的另一面：对**生产级 Agent 工程**的实战深度仍有欠缺。
