## N1 · 🚨 AI 已经在吃这 5 个岗位（不是预测）

> 钩子：恐惧 / 警示 · 目标：建立"AI 已经在发生"的紧迫感 + 反恐慌收尾

### 标题
🚨 AI 已经在吃这 5 个岗位（不是预测）

### 正文

上个月跟一个做 SEO 5 年的朋友吃饭，他们公司刚裁了一半 SEO 团队。原因不是公司没钱，是 ChatGPT 这一年改变了搜索方式。

不是个例。我整理了 78 个传统岗位的 AI 影响数据，5 个已经在大规模缩水：

1. CRUD Developer · -65%
增删改查后端工程师。Cursor + Claude Code 把这种重复模板代码的活几乎接管了。

2. Manual SEO Specialist · -58%
关键词堆砌 + 外链的老打法已经在死。ChatGPT/Perplexity 改了搜索习惯，GEO（生成式搜索优化）正在替代 SEO。

3. Data Analyst（基础报表）· -52%
做 Excel 报表、画图、跑 SQL 的初级岗。ChatGPT Code Interpreter + Hex AI 一句话就能搞定。

4. Admin Assistant · -45%
排日历、订机票、整理会议纪要。Claude + Zapier 一条工作流接管。

5. Bookkeeper / Paralegal / QA Manual Tester
重复型记账、法务文档初审、手工测试，AI Agent 全在啃。

为什么先吃这一批？观察下来一个共同点：他们都**不是 stakeholder，而且被动接任务**。

不是 stakeholder：决策不在你这。被动接任务：等别人派活。两条都中，AI 接管特别快。

但这里有个反共识的点：消失 ≠ 失业。

那个朋友的 SEO 团队，没被裁的 5 个人在做什么？三个人转 GEO Specialist，两个人转 AI Content Strategist。岗位换了一个名字，原本的行业经验还在用。

## N3 · 💼 AI Engineer 跟普通程序员差什么

> 钩子：澄清 · 目标：给技术背景读者一个具体能力地图

### 标题
💼 AI Engineer 跟普通程序员到底差什么

### 正文

最近不少人来问：我会写代码、做过 SaaS、跑过 Node.js，是不是就能转 AI Engineer？

不是。差着 5 个支柱、77 个具体技能。这是我整理的 AI Engineer 能力地图，2300+ 人测试过：

**支柱 1：Full-stack（23 项 · CORE）**
React / Next.js / Tailwind · Node.js / NestJS · PostgreSQL / MongoDB · Auth / RBAC / Stripe。这部分跟普通全栈差不多，是底座。

**支柱 2：Prompt Engineering（11 项）**
四要素 prompt · CoT 思维链 · Few-shot · System prompt · Prompt 版本化 · Eval-driven 迭代。AI Engineer 独有的"输入指令工程"。

**支柱 3：Cloud Foundation（12 项）**
AWS EC2 / S3 / Lambda · Docker · K8s · CI/CD · Secrets · CloudWatch。AI 应用都跑在云上，绕不开。

**支柱 4：Data Thinking（8 项）**
Vector DB · Embeddings · Chunking strategies · Eval metrics。普通后端基本不碰。

**支柱 5：AI Engineering Core（23 项 · CORE）**
OpenAI / Anthropic SDK · RAG · Agent 框架 · Function calling · Model Eval。这是 AI Engineer 真正吃饭的本事。

—

一句话：AI Engineer = Full-stack + Prompt + Cloud + Data + AI Engineering Core。

普通程序员一般只占 1-2 个支柱，剩下 3-4 个要补。

Shawn Wang（swyx）2023 年写过一篇 "The Rise of the AI Engineer"，他从 AWS / Netlify / Temporal 的 DevRel 转过来。建议去翻原文，挺短的，他写得很直接。

转型路径 4 步：
- 1-2 月：打基础（5 支柱挑薄弱补）
- 2 月：评估与上线（部署第一个 RAG）
- 2 月：Agent 与生产化
- 1-2 月：输出与求职（个人项目 + 作品集）

## N4 · 🏛️ 不会代码也能进 AI（HR / 咨询 / PM 路径）

> 钩子：反共识 · 目标：给非技术背景读者一个具体岗位路径

### 标题
🏛️ 不会代码也能进 AI 的具体路径

### 正文

很多人以为 AI 岗位 = 会写 Python = 程序员。错。

26 个 AI-native 岗位里，治理管理岗占 6 个。其中 AI Adoption Specialist 是文科 / 咨询 / HR / 培训背景特别能打的一个。

**它是干什么的？**

公司花几千万买了 AI 工具，结果员工不会用、不愿用、用错了。AI Adoption Specialist 就是负责把 AI 工具落地到每一个员工、每一个流程里的人。

一半时间做工具培训，一半时间做员工心理疏导。Gartner 预测 2028 年前全球新增 71,000 个这类岗位。

薪资段位：$90K-$190K（澳洲 / 北美公开数据）。

**谁适合？**
- HR / L&D（学习发展）出身
- 咨询 / PM 背景
- 心理学 / 组织行为学专业
- 业务运营老兵

为什么这批人比纯工程师更有优势？AI 工具不缺技术评估能力，缺的是：让一个 50 岁的财务大姐愿意打开 ChatGPT 的人。

软实力反而变成护城河。

**转型路径 4 步：**

- 1-4 周：补 AI 工具栈（ChatGPT / Claude / Cursor / N8N 都用一遍）
- 5-10 周：变革管理方法论（ADKAR / Kotter）
- 11-18 周：自己公司内部做一个试点项目
- 19-22 周：求职打包（把试点 case 写成 portfolio）

大约 5-6 个月可以投简历。

—

🎯 不需要写代码
🎯 不需要数学 / 统计背景
🎯 你"跟人打交道"的能力反而是稀缺资源

不是预测。是 JR Academy 现在每天在跑的系统。

**旧 marketing 团队（6 人）每天在做什么：**
- 2 人选题（刷热点、看竞品）
- 1 人写文案
- 1 人做设计
- 1 人发布 + 排期
- 1 人看数据

4 小时 / 天，4 个平台，1x 产出。

**新 AI-native 团队（1 人）：**
30 分钟 / 天，4 个平台，3x 产出。

差的不是人，是 pipeline。

—

**4 步演进，每一步都比上一步少一倍人力：**

**Step 1 · 手工玩 AI**
你 → ChatGPT → 复制 → 粘贴发布。
每条 2 分钟，1 平台，自动化 0%。
大部分人卡在这。

**Step 2 · 模板化自动化**
Cron 触发 → AI 写文案 → AI 配图 → 推给你审 → Buffer 自动发。
你从写手变成审核员。每条节省 80% 时间。

**Step 3 · 自动选题 + 4 平台分发**
Reddit / X / Google Trends → AI Agent 排 Top3 选题 → 4 个平台同时生成对应版本（X 280 字 / 小红书 400 字 / LinkedIn 长文 / 公众号 1500 字）→ 自动配图 → 自动发布。
你从审核员变成内容架构师。

**Step 4 · 会进化的 pipeline**
4 平台互动数据回流 → AI 学"哪类 hook 在哪个平台转化好" → 下一篇主稿数据驱动。
系统自我进化。你只调权重。
