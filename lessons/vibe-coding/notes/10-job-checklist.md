# 10 · Web 求职 4 步 Checklist

## 10.1 总览 · 求职是个系统工程

- 求职不是"投简历"，是一套 **4 个 Checklist** 的系统工程
- 4 个 checklist 各自独立、缺一不可
- 资源链接：[Web Checklist Wiki](https://jiangren.com.au/wiki/web/12cdd76b576d8054957ad755c0bb6588)

> 讲师提示：先把 4 个 checklist 名字念一遍，让学员心里有结构。

---

## 10.2 Checklist 1 · Resume（占成败 80%）

- 简历占求职成败 **80%**
- **AI Coding 项目要写进简历**：
  - Mini CRM
  - Pizza Maker
  - SigmaQ-style 复刻
- **量化成果**：
  - "用 Cursor 在 4 天内交付 X 功能"
  - "AI 协助下完成 N 个 endpoint + JWT Auth"

> 讲师提示：HR 看简历 30 秒，没数字 = 没记忆点。

---

## 10.3 Checklist 2 · LinkedIn 90% + 求职信 60%

### LinkedIn（90%）

- profile 完整度拉满
- **AI 项目要 pin** 到顶部
- 动态展示**学习路径**（每周更新一条进展）

### 求职信（60%）

- 用 AI 帮你**按岗位定制**
- 但**每封都要人工改**（AI 写完一定要读一遍改语气）
- 模版化求职信 = 直接进垃圾桶

---

## 10.4 Checklist 3 · 面试（占成败 50%）

- 面试准备占求职成败 **50%**
- **必练三项**：
  - **项目拷打** — 能讲清楚每个决策为什么这么做
  - **系统设计** — DB schema / API 设计 / 性能 trade-off
  - **Vibe Coding live demo** — 现场用 Cursor 做一个 feature
- **AI 时代新考点**：让你现场用 Cursor / Claude Code 实现一个 feature
  - 考的不只是代码，是**怎么和 AI 协作**

---

## 10.5 收尾 · 4 个 Checklist 闭环

> 讲师提示：4 个 Checklist 都做齐 = 求职闭环；缺一个就漏漏。完整内容在 jiangren.com.au wiki。

- Resume + LinkedIn + 求职信 + 面试 = 完整漏斗
- 任何一环掉链子 = 整个求职周期拉长 1-3 个月
- 课后行动：把今天 Pizza Maker Lab 的成果**当晚**写进简历 + LinkedIn
