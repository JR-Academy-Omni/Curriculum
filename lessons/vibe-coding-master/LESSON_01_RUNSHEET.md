## 2. 开场 + 课程总览讲稿（念，约 2-3 分钟）

各位晚上好，先说一句最实在的：这门课不是教你怎么让 AI 帮你写两行代码、补个函数。那个用 ChatGPT 一分钟就会了，不值得你花八周。

（真实数据钩子，念）先交个底，凭什么我来讲：AI coding 我自己做了整整一年，去年带了 15 场，这一年大概面对了 300 个同学。我统计过一个挺扎心的数字——这 300 人里，**真正会 AI coding 的不到 10%，能高效用起来的不到 3%**。绝大多数人其实不会 AI coding，他们只是会 **AI chat**——问一句答一句，把 AI 当个聊天机器人。这两件事差着十万八千里：AI chat 是你问它答；**AI coding 是你用结构化的规则、流程、真相去『指挥』它替你把事干成**。

（衡量标尺，念）怎么知道你到底站在哪一档？给你个特别简单的尺子：**一个同时集成 LinkedIn、Google、GitHub 三方 SSO 登录的功能，你能不能一小时内全部做完？** 会 AI coding 的人，一小时搞定；只会 AI chat 的人，光是看文档、配 key、调回调地址，就能卡你一整天。

（把话说重一点，但这是实话，念）你可能觉得「我会 AI chat 也够用了」——不够。当一个会 AI coding 的人，一小时能交付你一周的活，公司留谁、淘汰谁，不用我说。不会 AI coding 的下场，说白了就是：被替代、被挤出去，慢慢告别这个 AI / IT 行业。我不是吓你，这是我这一年带 300 个人，亲眼看到的趋势。

今晚这门课，就是带你从那 90% 跨进前 10%，往 3% 走。而第一步，就是今天要练的 PRD。

（🔴 重点·贯穿全课，对着 S01b 砸实，别一翻而过）还有件事今晚我要你记一辈子：**AI coding 根本不是『写代码』，是『用结构化的语言——规则、流程、真相——去指挥 AI 替你管理任何一件事』。** 写程序只是其中一个域。同一套功夫，还能管你的简历和职业（今晚就做）、做 marketing、管你的知识和记忆、跑公司运营、出设计。真正的 AI coding，大多数都不长得像代码——它长得像一份 PROFILE、一份 rules、一份 PRD、一个 CLAUDE.md。一句话：**大多数人把 AI 当 chat 工具问问题；会 AI coding 的人，把 AI 当一套能替自己管事的操作系统。** 这就是这整门课从头到尾只讲的那一件事——今晚先从『管你自己』开始。

我们要干的是另一件事——带你一块一块，搭出一个属于你自己的「个人 AI 操作系统」。什么意思？就是把你脑子里那些东西——你的经验、你的能力、你做过的项目——先变成机器能读懂的结构化资料，从一份简历开始，慢慢长成一套能替你干活的个人系统。等你把这套东西在自己身上跑通了，换个场景，你就能给一家公司搭它的「企业 AI OS」。这是同一套功夫，只是对象变大了。

而搭每一块的起点——不管是搭你自己的，还是搭公司的——永远是同一件事：先把你想要什么，写清楚成一份 PRD。PRD 你可以理解成「需求说明书」，但今晚我更想让你记住它的另一个身份：它是你和 AI 之间的「唯一真相源」，Source of Truth。AI 不是读不懂你，它是不知道你到底要什么。你说不清，它就猜；它一猜，你就开始返工。所以这门课第一节，我们不碰花活，先练这个底功。

八周大概怎么走，我先给你个预期，今晚只讲到层次，机制后面慢慢拆。第一阶段，立意图、打地基——就是写 PRD、建 SoT 这件事，也是今晚要动手的；第二阶段，造能力组件，给你的系统装上各种本事；第三阶段，选模型、做调优，让它跑得又准又省；第四阶段，组装、上线、交付。

具体这几节：今晚，拿你自己当案例，用「一份简历」入门——你对自己最熟，练手不烧脑；下一节课，换到一个真实项目上去；第三节课，才上 ADLC，那套完整的开发流程框架。先用你自己跑通直觉，再上规矩，顺序不能反。

走出这间教室你能带走三样实的：一份你亲手写的简历 PRD；一版 AI 据此产出的简历草稿，能直接拿去改；最值钱的——「怎么写 PRD、怎么建 SoT」这个能用一辈子的底层功夫。工具会换，模型会迭代，但「把需求说清楚」十年后照样值钱。

好——正式进内容之前，先花几分钟让我认识一下你们，这几分钟不白花。

## 2.5 学生自我介绍（破冰 + 收集 experiences 原料，~6min）

**位置**：开场讲稿之后、SoT 之前。**时长**：小班 ≤8 人 6–7'（每人 ~45 秒，老师每人补 1 句）；人多 >8 人 4–5'（改「打字接龙」打进聊天框，老师挑 3–4 个念，**绝不逐个点名**，否则会崩时间）。

**让学员说 3 句（直接喂给后面简历 PRD）**：
① 我是谁 + 现在做什么
② 我最拿得出手的一件事——做过什么，**最好带个数字**（带过几人 / 多久 / 多少量 / 提升多少）
③ 我想用 AI / 这门课达成什么

当场点扣子：「第②句 = 你简历 PRD 的『输入』；第③句 = 你的『目标』。说完，你 PRD 两个要素已经口头打好草稿了。」

**主持话术**：
- 开场（念）：「正式进内容前，花几分钟认识一下你们——而且不白花，你现在说的这几句，我后面会让你直接拿去用。三句话：你是谁、做过最拿得出手且能带数字的一件事、想用 AI 干嘛。谁先？」
- 控时（有人讲太久）：「打住——这个细节特别好，先记着，等会写简历 PRD 第二段正好用上。下一位。」
- 听到带数字的：「听到没，『带过 6 个人』『响应砍一半』——这种带数字的一句话就是简历最值钱的素材。」
- 听到太虚的：「『负责过很多项目』——等会挑一个，配个数字，它立马站起来。」

**红线**：不承诺包就业；不评判背景好坏（只评素材具不具体）；学员公司名/数字只用于本节现场，不外发、聊天记录课后不留存。

## 3. ② SoT 核心概念 · slide 讲解要点

- **S03 Source of Truth**：一件事只有一个权威出处，所有人和所有 AI 都回到同一处读、同一处改。三维度：规则/规范、数据/事实、决策/历史。点出题眼：**AI coding 的本质 = 不断确认 source of truth 的过程；AI 的能力上限 = 你给它的真相质量。**
- **S04 没 SoT vs 有 SoT**：左边漂移/编数字/重复犯错；右边一次同步/先查库再说/自动规避。
- **S04c + S04c2 互动判断**（建议现场举手）：「PRD 放云端 + 程序员自己又存一份给代码用的，算正确 SoT 吗？」→ 答案「看情况」：不看几份，看**谁说了算 + 改了会不会自动同步**。两份各自能改 = 两个真相 = 经典漂移。
- **S04d SoT 阶梯**：个人 PRD → 团队共享 SoT → 企业级真相源 → 企业 N 份文档聚成 **AI OS**。一句话桥接：「AI OS 不是玄学，就是 SoT 往上长出来的必然。」
- **S04d2 SoT First**：动手前第一件事是确定 SoT，不是先写代码、先配工具。三个触发点：写第一行代码前 / 配第一个 agent 前 / 开新 project 前 → 先问「SoT 在哪」。
- **S04e From SoT**：一份 PRD 长出很多产物（这套 deck 本身就是其一）——你只维护那份真相。

> ⏱ 时间紧可略过 S04b（团队视角）和 S04d3（manager 层思考题），今晚个人视角为主。

## 4. ③ AI OS 北极星 + 4C 地图 · 当作整门课的地图

- **S05**：模型是 CPU，你还缺一个操作系统（文件系统=记忆 / 内核=规则 / 驱动=工具 / 调度=编排）。

## 4.5 ③.5 传统 SDLC vs ADLC（左右对比，简单预告）

> 简单左右对比，**完整 ADLC 放第三节**；今晚只让大家看清「为什么一个人 + AI 现在能干一支团队的活、而 PRD 是支点」。slide 左边放传统 SDLC 角色图，右边放 ADLC。

- **左 · 传统 SDLC（一支团队）**：Stakeholder 提需求 → **BA** 写需求 → UI Designer → **Product Owner** → 前端/后端程序员 + **QA/Tester** → **DevOps/SRE** → Data Engineer → Data Analyst。一个产品要一整排角色，需求在人之间层层传递、层层失真，周期长。
- **右 · ADLC（你 + AI）**：你一个人 + AI OS。你来当那个「看见需求 + 写 PRD」的人（BA/PO 的活），AI 把 PRD 翻成前端/后端/测试/部署。那一整排实现角色，被你 + AI 收进来。
- **点破（关键，呼应今晚主线）**：传统 SDLC 里最上游、最值钱的是 **BA/PO——定义需求的人**；下游的实现角色，AI 正在接管。所以你要站到上游：**会写 PRD = 会当那个定义需求的人**。这就是为什么今晚先练 PRD——它是你在 ADLC 里唯一不可被 AI 替代的支点。

## 7. ⑥ Live Demo：同学简历 → 个人 OS / 第二大脑 → 渲染简历（~12min）

> 找一位同学贡献简历当**真实案例**。把简历当一坨 **raw 数据**，现场结构化成「这个人的迷你第二大脑」，再从它**渲染**出简历。简历不是终点，是第二大脑长出的一个产物（呼应 **S04e** 一份 SoT→多产物 + **S05a** 记忆层=第二大脑）。老师在记事本/编辑器里现场建几个文件即可，零安装。
> （结构照搬讲师真实在用的个人 AI OS：`PROFILE.md` / `experiences.md` / `rules` + raw→结构化数据流。）

**第一步 · 反例：raw 简历直接喂 AI（2'）** —— 把同学简历原文 + 一句 `帮我把这份简历改好` 丢给 AI，出来还是套话/不对焦。点破：「简历现在是**一坨原始数据**，AI 不知道哪条重要、对着哪个岗位、能不能信。没有真相源，它只能瞎改。」

**第二步 · 现场建这个人的迷你第二大脑（5'）** —— 仿个人 AI OS 结构，现场敲 3 个文件（投屏）：
- `PROFILE.md` —— 他是谁、想投什么岗（目标）
- `experiences.md` —— 把简历经历**抽成结构化事实**，每条配真实数字（raw→结构化：原始简历保留，提炼结果单独落一份，这就是他的 SoT）
- `rules.md` —— 红线：不准编造没有的经历/数字、不准夸大（给真相源上锁）
> 边敲边讲：「这三个文件就是这个人『第二大脑』的雏形——所有真实的东西收敛到这，以后谁（包括 AI）要用都回这读。」

**第三步 · 从第二大脑渲染简历（3'）** —— 跟 AI 说：`读这三个文件，按 [目标岗位] 渲染一份简历，只能用 experiences 里的事实，遵守 rules`。出来的简历对焦、有数字、不注水。
**关键现场动作**：再追 `帮我加个 AI 大项目显得厉害点` → AI 拒绝（rules 管着）。「红线是这个第二大脑的法律，它得听这个，不是听我一时的虚荣。」

**第四步 · 点破「一份 SoT 长出多产物」（2'，接 S04e）** —— 「简历只是这个第二大脑长出的**一个**产物。同一套 experiences，我还能渲染 LinkedIn、求职信、个人主页。改一次 experiences，所有产物一起更新——**你维护的永远是那一份真相，不是 N 份到处飘的简历**。这就是第二大脑的威力，也是你这门课要给自己搭的东西。」

> 和 PRD 4 要素的关系：刚才的「目标/输出/红线」就是 PRD；而「输入」从『每次粘一遍经历』升级成『指向 experiences.md 一次、永久复用』——这就是一次性 PRD → 持久第二大脑的跃迁，下节课搬到项目上（CLAUDE.md = 项目的第二大脑）。

## 8. ⑦ 学员动手：写你第二大脑的第一个文件（~14min，浏览器内）

> 框定（念）：「你刚自我介绍说的那件拿得出手的事 + 想投的方向——现在把它写成你**第二大脑的第一个文件**：一份你自己的 `experiences`（= PRD 的输入）+ 一份简历 PRD。今晚你就给自己的个人 OS 起了个头。」

念 Step，学员跟做：

- **Step 1（2'）** 打开 ChatGPT/Claude 网页版，新开对话，先别打字。
- **Step 2（6'）** 照 4 标题写自己的简历 mini-PRD，**重点写「输入」和「红线」**：
  - 目标：投什么岗、给谁看
  - 输入：你真实做过的 2—3 件事，每件**配一个真实数字**（带过几人/做了多久/多少量/提升多少）；没数字就写清事实，别编
  - 输出：几条 bullet、什么语气、突出什么
  - 红线：≥2 条，**必须有一条**「不准编造我没有的经历和数字」
- **Step 3（1'）** 整段 PRD 一次性粘进 AI，发送。
- **Step 4（3'）** 真相校验：逐条问「这是不是我真做过？数字是不是我给的？」发现 AI 自己加的，回 `第 X 条我没做过，请删掉，只用我提供的事实重写`。
- **Step 5（2'）** 截图「PRD + 草稿」，准备被点评。

**完成 Checklist**：4 要素都写了 / 输入≥2 件真实具体事 / ≥1 个真实数字 / 红线含一条禁编造 / 草稿无一条没做过的 / 能指着草稿说「这句因为 PRD 第 2 段写了 ___」。

**挑 2 份点评**：① 红线写得好的 → 验证「SoT 管住了 AI」；② 输入太虚的 → 现场补一个真实数字重发，全班看「真相源越清晰、产出越具体」。**只评 PRD 清不清楚/真不真实，不评简历好坏、不承诺 offer。**

**MCQ 2.** 模糊 prompt vs 4 要素 PRD 的对比说明什么？**C ✓** 产出质量主要取决于你有没有给 AI 一份清晰真相源，而非 AI 强弱（两次同一个 AI）。
