🎙 8 次直播陪跑大纲 · Dispatch AI 项目 Tier 2

Dispatch AI 是一个真实的 AI 调度系统，包含前端（React）、后端（Node.js API）、AI 服务和数据库。你将用完整 DevOps 工程实践把它从本地跑通，一路部署到云端生产环境、接入监控、做好安全加固。

每次直播 2h · 导师实操演示 + 学员跟做 · 课后录像永久回放 · 每次课有实战任务

S1
项目上手 & 本地环境搭建
2h 直播实战：跑通本地
本次内容

认识 Dispatch AI 的整体架构——前端、后端、AI 服务、数据库各模块的职责与关系。导师带大家克隆代码、配置环境变量、用 Docker Compose 在本地一键启动整个系统。

具体任务
  • 阅读项目架构图 + 目录结构
  • 安装 Node.js / Docker / AWS CLI
  • 配置 .env 文件，跑通 docker-compose up
  • 访问前端 + 测试 API 端点通畅
课后产出
✅ 完成标准：本地环境跑通，Dispatch AI 前后端均可访问，API 调用返回正确数据。截图发到群里确认。

为什么先看本地？真实工作中 DevOps 工程师要先理解应用才能设计部署方案。

S2
AWS 手动部署
2h 直播实战：云端跑通
本次内容

在真正写 IaC 之前，先手动在 AWS 上把 Dispatch AI 跑起来。这一步让你深刻理解每个云资源的用途，为后面用 Terraform 自动化打好基础。

具体任务
  • EC2 上部署后端 Node.js 服务
  • S3 + CloudFront 托管前端静态资源
  • RDS 搭建数据库 + 配置 Security Group
  • 配置 VPC 子网隔离 public/private
  • 用 IAM Role 替换 Access Key 直接访问
课后产出
✅ 完成标准：Dispatch AI 在 AWS 上可公网访问，前后端连通，数据库正常读写。记录下所有手动步骤——下次课用 Terraform 复现。
S3
Terraform IaC 基础
2h 直播实战：手工变代码
本次内容

把 S2 手动创建的所有 AWS 资源用 Terraform 重写。理解 IaC 的真正价值：可重复、可版本控制、可审计。导师带你拆解每一个 resource block，讲清楚 plan / apply / destroy 的工作原理。

具体任务
  • 编写 vpc.tf / ec2.tf / rds.tf / s3.tf
  • 配置 S3 Remote Backend + DynamoDB 状态锁
  • 用 variable.tf 和 terraform.tfvars 管理配置
  • 模块化封装 — 把 VPC 提取成可复用 module
课后产出
✅ 完成标准：terraform apply 一条命令可从零重建整个 S2 的云架构。提交 Terraform 代码到个人仓库。
S4
CI/CD 流水线
2h 直播实战：push 自动部署
本次内容

给 Dispatch AI 配上自动化流水线。代码合并到 main 分支后，GitHub Actions 自动：运行测试 → Build Docker 镜像 → Push 到 ECR → 部署到 EC2。这是 DevOps 工程师日常最高频的工作内容。

具体任务
  • 编写 .github/workflows/deploy.yml
  • 配置 GitHub Secrets — AWS 凭证注入
  • Docker Build → ECR Push → EC2 Pull & Restart
  • PR 触发 unit test + lint 检查
  • 部署失败自动回滚策略
课后产出
✅ 完成标准：向仓库 push 一次代码，5 分钟内 AWS 上的服务自动更新。在 GitHub Actions 里截图流水线 pass 状态。
S5
监控基础 · Prometheus + Grafana
2h 直播实战：Dashboard 上线
本次内容

生产环境没有监控等于瞎开车。导师带大家在 Dispatch AI 的服务器上部署 Prometheus，采集系统指标和应用指标，然后在 Grafana 里搭建一个真正有用的 Dashboard，配置告警规则。

具体任务
  • EC2 上安装 Prometheus + Node Exporter
  • 应用端暴露 /metrics 端点（Node.js）
  • Grafana 配置数据源 + 导入 Dashboard
  • 关键指标：CPU / Memory / HTTP 延迟 / 错误率
  • 配置 AlertManager — Slack 告警通知
课后产出
✅ 完成标准：Grafana Dashboard 显示 Dispatch AI 实时指标，人为触发高 CPU 后 Slack 收到告警通知。截图 Dashboard 发到群里。
S6
日志 & ELK Stack
2h 直播实战：日志聚合上线
本次内容

指标告诉你"出问题了"，日志告诉你"为什么出问题"。本次搭建完整的 ELK Stack，把 Dispatch AI 所有服务的日志汇聚到一个地方，配置结构化日志、搜索 + 过滤，设置告警规则。

具体任务
  • 部署 Elasticsearch + Logstash + Kibana
  • Filebeat 采集应用日志 → Logstash → ES
  • Kibana Index Pattern + Discover 搜索
  • 日志 Pipeline — 结构化解析 JSON 日志
  • Watcher 告警 — 5 分钟内 ERROR > 10 条触发
课后产出
✅ 完成标准：Kibana 中可以实时搜索 Dispatch AI 的应用日志，能过滤 ERROR 级别日志并定位到具体请求。
S7
安全 & 代码质量
2h 直播实战：安全门禁上线
本次内容

DevOps 安全（DevSecOps）是澳洲企业越来越看重的能力。本次把安全检查和代码质量扫描集成进 CI/CD 流水线，让每一次代码合并都经过自动化安全审计。

具体任务
  • 搭建 SonarQube 服务器，集成 GitHub Actions
  • 配置 Quality Gate — 低质量代码自动拦截
  • Snyk 集成 — PR 自动扫描依赖漏洞
  • 部署 Vault，迁移 .env 中的 Secrets
  • Pipeline 改用 Vault Dynamic Secrets
课后产出
✅ 完成标准：故意提交一个有漏洞的依赖，CI/CD 自动拦截 + Snyk 报告邮件发出。Vault 管理所有 Secrets，代码库中不再有明文密钥。
S8
高可用 & 答疑 · 项目收尾
2h 直播复盘 + Q&A + 简历指导
本次内容

最后一课，把 Dispatch AI 打磨成生产级高可用系统，同时做完整项目复盘。导师帮你梳理简历如何写这个项目，面试中如何讲清楚你做了什么。

具体任务
  • ALB + Auto Scaling Group — 水平扩展
  • RDS Multi-AZ + Read Replica 高可用
  • Failover 演练 — 手动杀掉一台 EC2，验证自动恢复
  • 负载均衡 + 健康检查配置
  • 项目复盘 + 简历写法 + 面试 Q&A 演练

❓ 常见问题 7 questions
我没有 DevOps 基础能学吗？
可以，但需要有基本的 IT / 编程基础（知道 Linux 命令行、用过 Git）。Tier 1 视频课的 M1 模块会从 DevOps 概念和 Docker 入门讲起，不需要之前有 DevOps 工作经验。

如果你完全没有 IT 背景，建议先完成 JR Academy 的 IT 基础课再来报名。这门课的目标人群是"想往 DevOps 方向转"的 IT 初中级工程师，不是零基础入门课。
视频课和直播课的区别是什么？
视频课（M1-M6）是自学模块，讲理论概念和工具使用，你自己安排时间学，可以随时暂停重看。内容覆盖完整的 DevOps 技术栈知识体系。

直播陪跑（S1-S8）是在视频课基础上，导师带你在真实项目 Dispatch AI 上实操。重点不是再讲概念，而是"怎么把这些工具用在一个真实的项目里"。直播中你跟着导师一起做，遇到问题可以实时问。

如果你的目标是求职，直播陪跑的项目经验远比单看视频有用。
Dispatch AI 是什么项目？
Dispatch AI 是一个真实的 AI 调度系统，包含 React 前端、Node.js 后端 API、AI 服务模块和 PostgreSQL 数据库。这不是专门为教学搭的玩具项目，而是有真实业务逻辑、多服务架构、有一定复杂度的系统。

为什么选这个项目：(1) 多服务架构是真实工作中的常态；(2) AI 应用是澳洲目前最热门的方向；(3) 项目本身足够复杂，DevOps 方案有挑战性，面试时讲起来有深度。

课程结束后，你的 GitHub 仓库里会有完整的 IaC 代码、CI/CD 配置、监控配置，可以直接给面试官看。
8 次直播时间如何安排？
直播时间根据每期 cohort 的学员时区和可用时间确认，通常在工作日晚上或周末。每次 2 小时，8 次课跨越约 8-10 周（每周 1 次）。

如果某次直播没法参加，不用担心：所有直播都有录像，课后 24 小时内上传，可以回看。但建议尽量参加直播，现场可以实时提问，体验远好于看录像。
做完项目能写进简历吗？
完全可以，而且这正是这门课的核心价值之一。完成 8 次直播后，你的 GitHub 上会有：

· Terraform 代码（完整 AWS 基础设施 IaC）
· GitHub Actions / Jenkins CI/CD Pipeline 配置
· Prometheus + Grafana 监控配置
· ELK Stack 日志系统配置
· SonarQube + Snyk 安全流水线
· 高可用架构文档（含架构图）

S8 的最后一次课专门有简历指导环节，导师帮你梳理如何描述这个项目、面试时如何回答"讲一个你做过的 DevOps 项目"这类问题。
如果有一次直播错过了怎么办？
所有直播均有录像，课后 24 小时内上传到学员区，永久可回看。错过一两次直播不影响后续课程——每次课相对独立，看录像 + 完成实战任务即可跟上进度。

如果连续错过 3 次以上，建议联系导师，可以安排转到下一期 cohort（不额外收费）。
