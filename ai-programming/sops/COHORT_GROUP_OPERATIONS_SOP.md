## §0. Meta + 为什么这是 P0

### 0.1 当前现状

`MARKETING_WORKFLOW_GAP_AUDIT.md` 已经把账算清楚了：

- **Phase C 课程运营覆盖率 7%**（最低，远低于 Phase A 71%、Phase B 62%、Phase D 42%、Phase E 39%）
- Phase C 总 16 个 task，0 个 full、3 个 partial、13 个完全裸奔
- 02 期已经开课 14 天（按 2025-11-16 起算到本文档生成日 2026-05-29 → 实际已经结业窗口），**没有任何沉淀文档可参考**
- Beta 一个人扛 24 学员 × 113 lesson × 42-56 天，凭记忆运营 = 必然翻车
- 班级群是 Phase C 的入口，群没人管 = 完课率必跌 < 50% = 作品交付率 < 30% = 03 期销售页无 OFFER WALL = 转化跌 = 04 期更难

### 0.2 02 期没人管 → 03 期 NPS 不及预期的可能性

按照 SaaS 行业基线，**首期 cohort 完课率 60% 是及格线**，没有运营 SOP 兜底的 cohort 历史数据看下来 30-40% 是常态。本课基础班 $860 / 进阶班 $1360 客单价，6-8 周长课程，**学员沉默 3 周不发言 = 已经流失但还没退款**，结业前一周才发现来不及。

**没有 SOP 的具体翻车场景（已经在 02 期发生或正在发生）**：

1. 学员 D+3 卡在 ChatGPT 注册（持卡问题）→ 群里没人回 → 当天放弃 → D+30 退款
2. Lab #6 Cursor 配置失败 → 群里问"我这边白屏"→ 助教没看到 → 第二天 Beta 回了一句"重启"→ 学员没解决继续卡
3. D+21 周末学员发"老师我跟不上"→ Beta 一句"加油"→ 学员一周后退课
4. 结业前 2 周想做 OFFER WALL → 翻群没找到 5 个完整作品 → 03 期销售页没素材

### 0.3 红线（不能干的事）

| 红线 | 来源 | 违反后果 |
|------|------|----------|
| 不发模版群公告（"在这个 AI 时代，让我们一起..."） | `feedback_no_templated_group_announcements.md` | 学员瞬间识别为机器人 → 失去信任 |
| 不承诺金钱 / 副业 / 包就业 | `feedback_no_revenue_promise.md` | 监管 + 投诉 + 退款潮 |
| 不在群里 call out 个人（即使是表扬，未经同意也不点名） | 隐私 | 学员投诉 |
| 不把群当销售渠道（推荐进阶班、推荐推荐礼） | 体验红线 | "我花了钱还要被推销" → NPS 暴跌 |
| 不在晚 10 点后发通知（除非紧急） | 节奏 | 打扰睡眠 → NPS 暴跌 |
| 不让 AI 直接以"老师 / 助教"名义代发（必须人审） | `marketing-agent-orchestra` 5 红线 | 失信 + 监管 |

### 0.4 本文档不覆盖的

- 学员产出收集 → 见 `STUDENT_OUTPUT_COLLECTION_SOP.md`
- 退课退费流程 → 走客服 Amelia / 不在群里讨论

## §1. 班级群人设

### 1.1 谁建群（不要混）

| 角色 | 谁 | 干什么 |
|------|----|----|
| 群创建人 | **Beta（教务）** | 微信号 / 企微建群 · 改群名 · 设管理员 |
| 群主 | **Beta（教务）** | 长期留任主，离职交接 |
| 管理员 | **Beta（主讲）+ 1 名助教** | 协助答疑 / 改群公告 / 踢恶意广告号 |
| 偶尔现身 | **Lightman**（创始人） | 仅 W1 开学 · W3 中点 · 结业典礼 三次出现，平时不打扰学员 |

## §2. 每日运营节奏（5 个时间点 · 节奏图）

> **核心原则**：节奏稳定 > 内容花哨。学员是来上课的不是来看运营秀的。每天 5 个时间点准时出现，比每天 10 个时间点临时改强。
>
> **写法红线**：每一段文案都要避开 `feedback_no_templated_group_announcements.md` 列出的模版词。每天的内容要有具体抓手（具体哪个 lesson / 哪个 Lab / 哪个工具），不是"今日学习愉快"这种无意义播报。

### 2.1 07:00 早安 + 当日学习内容预告

**Owner**: Beta · **执行时间**: 工作日 07:00（周末可放 09:00）· **耗时**: 8 分钟（含找 lesson 链接）

**内容模板（非模版话术，每天换不同抓手）**：

```
☀️ 早 — 今天是 W2 Day 3

【今天的安排】
- 自学：L23 Cursor 高级用法（35min · 录播）
- Lab：Lab #8 让 Cursor 改你昨天那个 todo app（25min）
- 直播：晚 8 点 Beta 带跑 L24 — 如何让 AI 写出能用的代码（90min）

【昨天 Showcase 提醒】
昨天卡在 OpenAI Key 的同学今天可以继续，群文件 [Key 申请失败 3 种情况.pdf] 已上传。
```

**写法红线**：
- ❌ 不要写"在这个 AI 时代我们一起进步"
- ❌ 不要写"准备好开启新的一天了吗"
- ❌ 不要每天用同一个开头（今天 ☀️ / 明天 🌅 / 后天 🌞 这种轮换是 OK 的）
- ✅ 直接告知今日 lesson + Lab + 直播课，给链接，给时间
- ✅ 每周一在早安里多加一条「本周节奏图」链接（见 §4.1）

### 2.2 12:00 午间答疑闸（Beta 集中回）

**Owner**: Beta · **执行时间**: 工作日 12:00-13:00（午饭前后）· **耗时**: 30-40 分钟

**为什么集中回而不是即时回**：
1. Beta 早上自己在直播 / 备课 / 录视频，分心即时回会打断深度工作
2. 学员早上自学时遇到的问题，到中午往往自己已经解决 50%
3. 集中回更高效，且能识别"重复问题"沉淀进 cohort wiki

**操作步骤**：
1. Beta 11:55 打开群，往上翻看上午所有未答问题
2. 把问题分类：L1 工具使用 / L2 课程内容 / L3 行业问题（见 §6 答疑分级）
3. L1 转助教 / L2 自己答 / L3 标记晚上回
4. 答完每个 @ 提问的学员（不是泛回）
5. 重复 ≥ 3 次的问题 → 同时发"📌 这个问题答疑墙已收录"+ 链到 cohort wiki

### 2.3 19:00 晚课直播提醒 / 当日作业截止提醒

**Owner**: Beta · **执行时间**: 直播日 19:00（提前 1h）/ 非直播日 19:00（提作业截止）· **耗时**: 5 分钟

**直播日模板**：

**非直播日（有作业截止）模板**：

### 2.4 21:00 当日 Showcase（昨日作品晒）

**Owner**: Beta（征学员同意后晒）· **执行时间**: 工作日 21:00 · **耗时**: 10 分钟（含截图 / 找学员授权）

**为什么是 21:00 而不是当天**：
1. 学员当天提交的作业要 Beta 看过 + 学员授权 + 整理截图，不可能即时晒
2. 21:00 是学员吃完饭休息时段，群活跃度高，Showcase 能引发讨论
3. 把昨天的作品晒今天 = 给作者一天「沉淀+心理准备」时间

**操作步骤**：
1. Beta 20:30 翻昨天作业 → 选 1-3 个有亮点的
2. 私聊作者获取授权（"今晚 9 点想在群里晒你这个，可以吗？我会写 XX 同学"）
3. 21:00 发：作品截图 + 作者备注（化名 OK）+ 一句具体表扬 + Beta 留言点评
4. 发完 @ 作者请 ta 出来跟大家说两句（可选）

**写法红线**：
- ❌ 不写"这个同学好厉害我们一起为他鼓掌"
- ❌ 不写"看吧 AI Coding 真的能改变人生"（金钱承诺红线衍生）
- ✅ 写"用 Cursor 改了 nut.com.au 商品页的 hero block，重点是把价格和 CTA button 改成了对比测试 A/B 两版"
- ✅ Beta 点评要具体（"hero 这版字号 clamp 用对了 / B 版的 CTA 颜色再降一档对比度会更舒服"）

## §3. 答疑 SLA

### 3.1 SLA 表

| 时段 | 首响 SLA | 给出答案 SLA | 谁负责 |
|------|----------|--------------|--------|
| 工作日 09:00-18:00 | 30 分钟 | 4 小时 | 助教 L1 / Beta L2 |

| 工作日 22:00-09:00 | 不响 | 第二天 09:30 | — |
| 周末 10:00-18:00 | 2 小时 | 当日给答案 | Beta / 助教 |
| 周末 18:00-22:00 | 2 小时 | 第二天 09:30 | Beta |
| 紧急（平台 down / 直播断线） | 立即 | 立即 | Beta 微信私聊（24/7） |

### 3.2 SLA 没达到怎么办

- 单次未达：Beta / 助教在群里跟学员说"抱歉刚才在录课，现在帮你看"（不甩锅，不解释一大堆）
- 一周内 3 次未达：Beta 内部 review 是不是助教不够 / Beta 时间排满
- 持续未达 → 触发 §3.4 升级机制

### 3.3 答不了怎么办

| 情况 | 处理 |
|------|------|
| 助教答不了 L1 工具问题 | 转 Beta · 不要硬答（错答比晚答更伤信任） |
| Beta 答不了 L2 课程内容 | 极少见，但允许说"这个我也要查下今晚回你"· 不允许编 |
| 涉及 L3 行业 / 求职 / AI 工程方向 | Beta 答 + Lightman 必要时补一句（Lightman 不日常在群） |
| 涉及 L4 退课 / 退费 / 不满 | Beta 不答，转给客服 Amelia · 群里只说"已经转给客服 Amelia 私聊你了" |
| AI 行业最新动态 / 工具迭代（如 Claude 4.7 出了） | Beta 当晚 22:00 前在群里发一条「行业小灶」· 不水群 |

### 3.4 重复问题处理（沉淀机制）

**为什么要沉淀**：03 期会出现 02 期一模一样的问题（OpenAI Key 申请失败、Cursor 中文乱码、Lab #6 浏览器白屏）。不沉淀 = Beta 每期都重答一遍。

**操作**：
1. Beta 答完一个问题，标记为「重复 ≥ 3 次」
2. 助教把问答整理成 200 字以内的 wiki 条目
3. 落盘到 `curriculum/ai-programming/cohort-03/qa-wiki/{lesson_code}-{question_slug}.md`
4. 下次同样问题出现 → 群里贴 wiki 链接（不只甩链接，要带一句"这个昨天讲过，[xxx] 同学也卡过，wiki 在这"）
5. 结业前把 wiki 整理一遍 → 喂回 outline.json 的 lesson FAQ 字段 → 04 期开始 lesson 详情页直接显示

## §5. 防潜水机制

> **背景**：cohort 6-8 周长课程，学员 3 周不发言 90% 概率已经心理放弃，5 周不发言 99% 已经流失。Beta 一个人扛 24 学员，必须有机制识别+干预，不能靠记忆。

### 5.1 每周一次「1 句话报到」

**Owner**: Beta · **执行**: 每周日晚 20:00 发起 · **完成率统计**: 周一 09:00 报数

**模板**：
```
🗣️ W2 报到 — 一句话

发在群里：
- 这周学了什么（一个 lesson 名 / 一个 Lab）
- 卡在哪（没卡就说"没卡"，也算）

**红线**：
- ❌ 不催（"你怎么还不来上课"）
- ❌ 不教训（"花了 $860 不学多浪费"）
- ❌ 不暗示退费有损失（合规问题）

3. 等待 7 天 → 还无回应 → Beta 走客服流程默认按 A 处理（保留进度）+ 群里安静移出

## §6. 答疑分级路由（L1-L4 分诊）

> **核心原则**：每条问题先 30 秒分诊，路由到正确的人，不要 Beta 答 L1 工具问题 / 不要助教硬答 L3 行业问题。

### 6.1 L1 工具使用 → 助教

**典型问题**:
- "Cursor 装不上"
- "OpenAI Key 申请失败"
- "Lab #6 浏览器白屏"
- "我电脑是 Windows 怎么装 X"
- "ChatGPT Plus 怎么订阅"

**Owner**: 助教（03 期招的兼职 · 工资可以是按问题数 + 留存率）
**SLA**: 30 分钟首响 / 2 小时给答案
**Escalate**: 助教 30 分钟没解决 → 转 Beta

### 6.2 L2 课程内容 → Beta

**典型问题**:
- "L24 这个 prompt 公式我没搞懂"
- "Lab #9 我做出来跟视频不一样"
- "Cursor 跟 Claude Code 有什么区别"
- "为什么这个代码 AI 写得这么烂"

**Owner**: Beta
**SLA**: 工作日 4 小时 / 周末当日
**Escalate**: Beta 答完一定要让学员回应"懂了/没懂"，没懂就再答

### 6.3 L3 行业 / 求职 / AI 工程方向 → Beta + 必要时 Lightman

**典型问题**:
- "学完这课我能去做 AI Engineer 吗"
- "AI 这个方向 5 年后会不会被 AI 自己取代"
- "我应该学 LangChain 还是 LlamaIndex"
- "Cursor 跟 GitHub Copilot 哪个值得付费"

**Owner**: Beta 主答 + Lightman 必要时一句话点睛
**SLA**: 当晚 22:00 前回 / 第二天给完整答复
**红线**:
- ❌ 不能说"学完保证你能做 AI Engineer"（金钱承诺红线）
- ✅ 可以说"我们已有学员从 X 转到 Y，但每个人情况不同"
- ✅ 涉及简历 / 面试 / 求职咨询 → 推荐升学路径（ai-engineer-bootcamp / `alumni-next-course-recommender`）

### 6.4 L4 退课 / 退费 / 不满 → 客服流程

**典型问题**:
- "我学不动想退费"
- "课程内容跟我以为的不一样"
- "Beta 上周直播没准备好"

**Owner**: 客服 Amelia（不是 Beta / Beta）
**SLA**: Beta 转客服 → 客服 24h 内私聊学员
**红线**:
- ❌ Beta / Beta / 助教不要在群里讨论
- ❌ 不要试图"挽留"学员（"再坚持一下啊")
- ❌ 不要在群里解释"这个学员是因为 X 不满"
- ✅ Beta 收到信号 → "我已经转给客服 Amelia 私聊你了"+ 转 Amelia + 群里不再提

## §7. 学员卡点 1v1 SOP（关怀，不是催）

### 7.1 触发条件（任一即触发）

| 条件 | 数据源 | 触发频率 |
|------|--------|----------|
| 作业未交 2 次 | jr-academy 后端 assignment | 每周一 Beta 查 |
| Lab 失败 5 次 | jr-academy 后端 labAttempt | 每周一 Beta 查（含 Quest 通不过的） |
| 群里 3 周 0 发言 | 群发言记录 | 见 §5.2 |
| 直播课连续缺席 2 次 | jr-academy 后端 attendance | 每周一 Beta 查 |
| 周末 1 句话报到 连续 2 周未参与 | §5.1 报到数据 | 每周一 Beta 查 |

### 7.2 私聊话术原则（4 个不）

1. **不催**（不说"你怎么还不交作业"）
2. **不教训**（不说"花了 $860 不学多浪费"）
3. **不假装是 AI**（开头一定说"hi Beta 这边"）
4. **不给虚泛建议**（不说"加油坚持就是胜利"）

### 7.3 给方案（3 个具体选项）

不要问"你需要什么帮助"——99% 学员答"没事"。直接给 3 个具体选项让 ta 选：

```
hi [化名]，Beta

注意到你 Lab #8 跑了 5 次都没过，想确认下：

3 个选项你看哪个能帮到你：
A. 我把上周晚课录播里 Beta 演示的 Lab #8 段落（17min）单独剪出来发你
B. 安排一次 30 分钟 1v1 答疑（Beta 或 我 都行，挑你方便的时间）
C. 如果最近真的太忙 → 申请暂停 1 周，进度保留，下周继续

或者其他你想到的 → 直接说，我帮你想办法。
```

### 7.4 方案落地（A/B/C 三选一后怎么走）

**A. 录播段落剪辑** → Beta 找助教剪 + 24h 内私发 + 3 天后回访
**B. 1v1 答疑** → Beta / Beta 约时间 → Calendly / 微信视频 → 答完 Beta 留 1 句"这个 Lab 你下次还卡再找我"
**C. 暂停 1 周** → Beta 在 cohort progress 标记 `paused_until=2026-XX-XX` → 下周一查 → 还没回来再问一次

### 7.5 1v1 后回访

- 3 天后 Beta 私聊「上次的方案有效果吗」
- 没效果 → 再换一个方案
- 有效果 → 鼓励一句（不夸张）+ 邀请回群发言

### 7.6 1v1 数据落盘

每次 1v1 落盘到 `curriculum/ai-programming/cohort-03/1v1-log/{学员化名}-{日期}.md`，包含：
- 触发条件
- 私聊话术
- 学员回应
- 给的方案 + 落地情况
- 3 天后回访结果

**为什么落盘**：03 期沉淀的 1v1 数据 = 04 期识别"哪类学员 W2 必卡 Lab #6" = 直接喂回 `outline.json` 改进 Lab 设计。

## §8. NPS 调研节奏

> **核心原则**：不能等结业才问。结业问 = 问出来也没法改 = 数据只能给 04 期用。
>
> **下游消费**：NPS 数据 → `/marketing-data-collector` 自动汇总 → `FUNNEL_PLAN.md` 真实转化率回写 → PERSONAS refresh

### 8.1 周中 NPS（轻量 1 题）— 每周三 20:00

**Owner**: Beta · **频率**: 每周三晚 20:00 群内发起 · **完成时长**: 学员 20 秒

**问题（1 题）**:
```
本周到现在为止，你给这门课打几分（0-10）？

回复数字即可（10 = 非常推荐 / 0 = 完全不推荐）
不需要解释。我会私聊问 0-6 分的同学发生了什么。
```

**为什么是周三而不是周末**：
- 周末学员在补作业 / 休息，参与度低
- 周三 = 本周已经过半 + 还有时间补救，数据最有用
- 0-6 分 → §8.4 同步触发 1v1

### 8.2 周末 NPS（5 题）— 每周日 20:00

**Owner**: Beta · **频率**: 每周日晚 20:00 · **完成时长**: 学员 3-5 分钟

**问题（5 题，腾讯问卷 / Google Form 都行）**:
1. 本周 NPS 分数（0-10）
2. 本周最有收获的一个 lesson 是哪个？
3. 本周最卡的一个点是什么？（lesson 名 / Lab / 工具配置 / 其他）
4. 这周的直播课 Beta 的节奏对你来说是（太快 / 刚好 / 太慢）
5. 还有什么想跟我们说的？（选填）

**回收 SLA**: 周一中午前回收 → Beta 周一下午整理 → 周一晚发到 admin Cohort Ops tab（未来建）

### 8.3 课程末完整 NPS（10 题 + 续费意向）— 结业前 D-3

**Owner**: Beta + Lightman · **执行**: 结业典礼前 3 天

**问题（10 题）**:
1. 整体 NPS（0-10）
2. 完课率自评（你跟完了多少 lesson）
3. 最有价值的 3 个 lesson / Lab（开放）
4. 最没价值的 1 个 lesson / Lab（开放，做 outline 改进用）
5. Beta 的教学节奏（太快 / 刚好 / 太慢）
6. Beta 的群运营（满意 / 一般 / 不满意 + 一句话）
7. 想加 / 想砍的内容
8. 你愿意把这门课推荐给身边人吗（是 / 否 + 理由）
9. **续报进阶班的意向**（一定会 / 可能会 / 不会 + 理由 — 这是 FUNNEL 的转化率回写源头）
10. 愿不愿意做 testimonial 视频（5min · 我们会付 AUD $50 / 不愿意也完全 OK）

**红线**:
- ❌ 不能问"你赚到钱了吗 / 接到单了吗"（承诺红线 + 没意义）
- ❌ 不能问"愿不愿意 1v1 给你推销进阶班"
- ✅ 可以问推荐意向 / 续报意向（标准 NPS 题，不算推销）

### 8.4 NPS 数据下游（不能让数据死在表格里）

```

  ↓

  ↓
