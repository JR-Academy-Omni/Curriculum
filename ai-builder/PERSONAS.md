# Vibe Coding 大师课（ai-builder）· 课程人群画像 PERSONAS

> 课程真相源见 `public/outline.json`（官方名「Vibe Coding 提升班」，对外宣传名「Vibe Coding 大师课」）。
> 本文件是这门课的**目标人群真相源**，营销文案 / 渠道 / 漏斗都以此为准。

## 课程定位（先对齐，别跑偏）

- 这**不是**求职 bootcamp，也**不是**零基础转码课。
- 核心是**已经会写代码的人，怎么用 AI 把产品从 PRD 做到上线**——能力 / 效率升级。
- 钩子是「**产出能上线的全栈产品 + 全流程 AI 开发工作流（Skills/MCP/Agent + 四工具）**」，**不是**「保 offer / 保涨薪」。
- 8 周 42 节课，毕业产出一个可上线的 AI 全栈项目 + Demo。

## 市场范围（首轮）

- **澳洲华人开发者 + 留学生为主，全球华人开发者为辅**（线上交付可扩散）。
- 和 Alex 的 AI Engineer 求职课（北美 + 简历项目）**区分开**，避免抢同一池子。

## 人群画像

### 1. 在职华人开发者，想跟上 AI Coding 浪潮 ⭐首轮核心
- **背景**：2–6 年 backend / full-stack / DevOps 经验，有 Python / JS 基础
- **状态**：会用 ChatGPT / Copilot 写片段，但停在"问一句答一句"，没系统的 AI 开发工作流
- **痛点**：怕被"会指挥 AI 的人"卷掉；想系统学，但市面课多是 prompt 速成、不够工程化
- **主钩子**：从"让 AI 写片段"到"用 AI 把产品从 PRD 做到上线"；Skills + MCP + Agent + 四工具实操
- **渠道**：LinkedIn、公众号、微信技术群、澳洲本地华人 IT 群、X / 掘金 / V2EX

### 2. 技术创业者 / 想做独立产品的人 ⭐首轮核心
- **背景**：有技术底子的 indie hacker、小团队 CTO、想做 side project / SaaS 的工程师
- **状态**：有想法，但从 0 到能收款总卡在"做不完 / 太慢"
- **痛点**：一个人精力有限，想用 AI 提速、快速验证 MVP
- **主钩子**：一个人用 AI 把想法跑成能上线的全栈产品（42 节系统课 + 可 demo 毕业项目）
- **渠道**：小红书、公众号、独立开发 / 创业社群、X、即刻

### 3. 留学生 / 应届 CS，想用"能上线的项目"做简历差异化 ·副线
- **背景**：CS / SE / DS 高年级或应届，有课程项目但偏作业感
- **状态**：会写代码，但简历缺一个"能打开链接给面试官看"的真实产品
- **痛点**：项目太像 homework，投开发岗没亮点；担心 AI 工具只会皮毛
- **主钩子**：毕业带走一个可上线、能 demo 的 AI 全栈项目，进简历作品集
- **渠道**：小红书、留学生群、Discord、一亩三分地

### 4. 训练营毕业 / 转码全栈，想升级到"AI 驱动开发" ·副线
- **背景**：上过全栈训练营或自学，有 Web / Python 基础，做过 CRUD 项目
- **状态**：会搭传统全栈，但不会把 AI / Agent 嵌进开发流程
- **痛点**：怕停在"只会基础 CRUD"，跟不上 JD 里的 AI 要求
- **主钩子**：有基础即可，把 AI 工具流 + Agent 嵌进真实项目
- **渠道**：小红书、转码群、Bootcamp 校友群、公众号

## 首轮主攻

优先打 **第 1、2 类**：
- **在职开发者**：客单价承受力强，吃"production 全流程 / 工程化 / 工具对比深度"，转化逻辑清晰。
- **技术创业者 / 独立开发**：对"一个人用 AI 做出能上线产品"最敏感，付费意愿高、爱晒作品传播性强。

第 3、4 类作内容副线（蹭"作品集 / 升级"角度引流），首轮不单独做一套 campaign。

## 红线

- 不写"保 offer / 保涨薪 / 保证赚钱"；卖点用"能搭出能上线的产品 / 全流程工作流"这类能力陈述。
- 课程内容数据（42 节 / 四工具 / MCP·Agent / 可上线项目）以 `public/outline.json` 为准，不臆造。
