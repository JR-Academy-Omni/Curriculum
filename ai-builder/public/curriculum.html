L01

什么是 Vibe Coding？未来的开发范式

LIVE 1h
Vibe Coding 不是传统的逐行写代码，而是用自然语言描述你要什么，AI 帮你实现。你负责架构决策和质量把关，AI 负责写代码和调 bug。这节课讲 Vibe Coding 为什么是未来的开发范式、和传统开发的本质区别、以及 AI Coding Agent 正在怎样改变软件工程的工作方式。

L20

Claude Prompt Engineering 与 Context Engineering（上下文工程）实践

LIVE 1h 30min
Prompt Engineering 管的是单条指令的质量，Context Engineering 管的是 AI 能看到的所有信息。同一条 Prompt 在不同 Context 下输出完全不同。这节课讲 Context Engineering 的核心：CLAUDE.md 文件写什么（项目规范和约束）、.cursorrules 怎么配（编码风格和模式）、什么信息放上下文什么不放、怎么设计项目结构让 AI 更好理解你的代码。

Q02

Quest 实战：给你的真实项目写一套 .cursorrules + CLAUDE.md

QUEST 40min

🎯 这节课解决什么问题

L20-L24 讲了 Context Engineering 是 AI 协作的核心、.cursorrules 怎么写、CLAUDE.md 干什么用。但「读完文档」≠「能给自己项目写出来」。这节 Quest 由小花老师带你拿你自己的真实项目（不管是工作 repo 还是 side project），写出一套实际有效的 .cursorrules 和 CLAUDE.md 配置——告诉 AI 你的代码风格、技术栈、禁止做的事。做完后 Cursor 和 Claude Code 在你这个项目里就能按你的规矩干活。

📖 你将学到

  • 识别项目里值得告诉 AI 的关键约定 — 不是写「请写好代码」，是写「禁止用 any、必须 strict TypeScript」
  • 把 .cursorrules 和 CLAUDE.md 分工写 — 一个给 Cursor 看，一个给 Claude Code 看
  • 验证 AI 真的按规矩干活 — 故意让 AI 写一段，看它有没有遵守

💡 适合谁

已经在用 Cursor 但每次都觉得 AI 不懂你项目的工程师。这一节让你的 AI 真正变成「了解你项目」的协作者。

✅ 学完能做什么

你的所有后续 AI 协作都基于这套配置——AI 不会再写 any 类型、不会再用错的库、不会再忽略你的命名规范。这是 Vibe Coding 提升 10x 效率的关键。
