L67

自学：最新 AI Coding 模型选型参考（Claude / Codex / GPT-5.5）

INFO 2h
截至 2026-06-28，AI Coding 的主线已经不是“多模型猎奇对比”，而是根据任务风险和交付方式选择工具：Claude Fable 5 / Opus 4.8 适合复杂推理、长上下文和高自治 agentic coding；Codex 适合本地仓库里的小步修改、运行命令、调试和 PR 级交付；GPT-5.5 适合复杂 reasoning + coding 的 API 工作流，GPT-5.4 mini/nano 更适合低延迟、低成本批量任务。Gemini 只保留为可选参考，不作为本课主线。
L68

多模型 Prompt Engineering 模板参考

INFO 2h
不同模型的 Prompt 写法不一样。Claude 喜欢 XML 标签结构，Gemini 喜欢简洁直接，Codex 需要更多上下文。这节课给你每个模型的 Prompt Engineering 最佳实践模板：System Prompt 怎么写、Few-shot 示例怎么给、长上下文怎么组织。同一个需求的三种 Prompt 写法对比。

LAB 30min

L77

Codex CLI 与高效调试技巧全解析

LIVE 1h 30min
Codex CLI 是 OpenAI 的命令行 AI 编程工具——在终端里一条命令让 AI 写代码、改代码、跑测试。和 Claude Code 类似但风格不同：Codex 更轻量、启动更快、适合小任务。这节课讲 Codex CLI 的高效使用：常用命令、配置文件优化、和 Claude Code 配合使用的工作流。

Codex 准确率、Prompt 精调与 AI Coding 优化闭环

LIVE 2h 30min
这节直播把 Codex 准确率提升、Prompt 精调模板和 AI Coding 优化闭环合并讲透。你会用 Step-by-step、Few-shot、CoT、Schema 约束、Skills 加载和 Self-Consistency 做对比实验，并把结果落到质量、速度、成本三个指标上：少返工、少等待、少无效调用。
