L44

用 AI 搭建 MCP Server（Model Context Protocol）

INFO 30min
MCP 是 Model Context Protocol——让 AI 调用外部工具和服务的标准协议。Claude 要查数据库、读文件系统、调第三方 API，都通过 MCP。这节课讲 MCP 的 Client-Server 架构：Client（AI 工具端）发起请求 → Server（你写的服务端）执行操作 → 返回结果。Tool 和 Resource 的区别、Transport 层怎么工作。

Claude Skills 与 ToolUse 工具调用机制详解

LIVE 2h

Q03

Quest 实战：从零搭建你自己的 MCP Server

🎯 这节课解决什么问题

L44-L52 讲了 MCP 是什么、怎么调外部工具、Anthropic 的官方协议长什么样。但所有人都看完文档还不敢自己写一个 MCP Server——怕协议复杂、怕调不通、怕不知道从哪开始。这节 Quest 由小花老师带你用 TypeScript 从零搭一个最小可用的 MCP Server（一个能查天气或者搜本地文件的工具），然后接到 Claude Code 里实际调用一次。做完你就真正理解 MCP 不是黑盒。

  • 把 Server 接到 Claude Code 实际跑 — 在 Claude 对话里看到你的 tool 被调用

💡 适合谁

L54

什么是 Agentic Workflow

INFO 30min
Agentic Workflow 就是让 AI 自主规划和执行多步任务，而不是你一步步指挥。传统方式：你写 Prompt → AI 回答 → 你再写 Prompt → AI 再回答。Agentic 方式：你给一个目标 → AI 自己规划步骤 → 调工具执行 → 检查结果 → 决定下一步 → 直到完成。这节课讲 ReAct 模式、Tool Use 循环、以及什么任务适合 Agentic 什么不适合。

Q04

Quest 实战：搭建一个能跑的 Claude Code Sub-agents 工作流

QUEST 45min

🎯 这节课解决什么问题

L54-L64 讲了 Agentic Workflow、Multi-agent、Sub-agent 这些概念。但「概念懂了」≠「能搭一个真在干活的 Agent 团队」。这节 Quest 由小花老师带你用 Claude Code 的 Sub-agents 机制，搭一个 3 个 agent 协作的真实工作流——比如：Researcher 负责查资料、Writer 负责起草、Reviewer 负责质检。做完你就有一个能反复调用、专门干这件事的 Agent 团队。

📖 你将学到

  • Claude Code Sub-agents 怎么定义 — .claude/agents/*.md 文件结构、frontmatter、prompt
  • 三个 Agent 怎么分工 — 各自的 description、tools、prompt 不一样
  • 主对话怎么调用 Sub-agent — Task tool 触发 + 看输出

💡 适合谁

已经做完 Q03 会写 MCP 的工程师，想再进一步搭 Multi-agent 系统。这一节让你的 AI 协作从「单聊」升级到「团队」。
