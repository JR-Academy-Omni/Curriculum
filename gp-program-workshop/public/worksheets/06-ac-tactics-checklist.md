# Assessment Centre 战术清单

> GP Program Workshop L06 配套资源 · AC 群面 / Case Study / Role Play / 1v1 各环节战术 · AC 当天 24h 准备清单。

## 一、AC 是什么 · 各家组合

**Assessment Centre** = GP 招聘最后一关，通常 **3-5 小时**，混合多个环节考察你。各家组合不同：

| 行业 | 典型组合 |
|------|---------|
| Big 4 Audit | 群面（讨论）+ Case Study + 1v1 Partner 面 |
| Banks (Macquarie / NAB / CBA) | 群面 + Role Play (Client Simulation) + 1v1 |
| Tier-1 Consulting (Bain / BCG / McK) | Case Interview 多轮 + 1v1 + 群面 |
| Gov Grad (NSW / VIC / Federal) | 群面 + Briefing 写作 + Panel Interview |
| Tech (Atlassian / Canva / REA) | 技术面 + 系统设计 / 行为面 + 1v1 |
| Engineering (Aurecon / WSP) | Technical Case + 群面 + 1v1 |

## 二、群面（Group Discussion / Group Exercise）

### 形式
- 6-8 人一组 · 30-45 分钟讨论一个商业问题
- 评估官在房间四周观察、不发言
- 题目可能：开放讨论 / case 分析 / 投资决策 / 危机处理

### 3 个角色（隐形）

| 角色 | 行为 | 招聘官评什么 |
|------|------|-------------|
| **Leader** | 推进议程 / 拉回偏题 / 总结观点 | 但**不抢话** — 评的是"能不能让团队产出结果" |
| **Timekeeper** | 提醒"还剩 15 分钟" / "我们花了 8 分钟在这点" | 评结构感 |
| **Note-taker** | 白板 / 纸上记关键 / 最后帮总结 | 评 listening + synthesis |

**你不必固定一个角色** — 题目开始时谁先抢谁就 Leader，你可以做 Timekeeper / Note-taker 同样能得高分。

### ✅ 高分行为

- **先问问题 confirm understanding** — 不要题目读完就直接说"我觉得..."
- **结构化推进** — "Let's structure this into 3 parts: A, B, C. Should we start with A?"
- **拉回偏题** — "That's interesting — let me note it and bring us back to..."
- **建立在别人观点上** — "Building on what [name] said about X, I'd add that..."
- **明确点名** — "[name], what do you think?" — 拉沉默的人入场
- **承认错误** — "Actually, I'd revise my earlier point — [name] is right that..."

### ❌ 低分行为（小心）

- 抢话 / 不让别人说
- 完全不说话（被评 "did not contribute"）
- 翻白眼 / 不耐烦的肢体语言
- 在最后 5 分钟才发言（认为别人说完自己 wrap-up 就行 — 错）
- 答非所问 / 偏离题目
- 听不懂英文 accent 不问 confirm（要主动 confirm 别装懂）

### 30 秒开场（每次群面通用）

```
"Before we dive in, can I suggest we take 60 seconds individually to read
the brief and jot down our initial thoughts? Then let's go round-table
30 seconds each to surface ideas before we structure."
```
— 这一句直接拉你到 Leader 位置 + 显示结构思维。

## 三、Case Study

### 形式
- 30-60 分钟独立读案例（10-15 页材料）
- 给商业问题：要不要进入 X 市场 / 公司利润下降原因 / 收购方案对比
- 输出：5-10 分钟向 panel presentation + Q&A

### 4 个结构化框架（按问题选）

#### 1. Profit Tree
```
Profit = Revenue - Cost
        ↓                ↓
    Volume × Price    Fixed + Variable
```
**适用**：利润下降 / 利润提升类问题

#### 2. MECE（Mutually Exclusive, Collectively Exhaustive）
拆分问题成不重叠、不遗漏的几个部分。例如客户分群：B2B / B2C / Internal · 地区：AU / NZ / SE Asia。
**适用**：任何分析类问题，作为底层原则

#### 3. 4P / 7P Marketing
Product / Price / Place / Promotion (+ People / Process / Physical evidence)
**适用**：市场进入 / 产品定位类

#### 4. Porter 5 Forces
- 行业内竞争 · 潜在进入者 · 替代品 · 供方议价权 · 买方议价权
**适用**：行业分析 / 竞争策略

### 输出 Presentation 5 段结构

```
1. Executive Summary (30 秒)
   "My recommendation is X. Here are 3 reasons why."

2. Approach (1 分钟)
   "I structured this into [framework]. I focused on [N] key dimensions."

3. Analysis (3-5 分钟)
   - Dimension 1: data → finding
   - Dimension 2: data → finding
   - Dimension 3: data → finding

4. Risks / Trade-offs (1 分钟)
   "The main risk to this recommendation is X, mitigated by Y."

5. Next Steps (30 秒)
   "If I had another week, I'd validate [data gap] and analyse [follow-up question]."
```

### 数字估算技巧

Case Study 经常要求估算（Estimate market size / Customer base）。**说出思路比说出正确答案更重要**：
- 从 known 开始（澳洲人口 25M）
- 一步步乘除（25M × 70% adults × 30% segment × $X spend = $Y）
- 标 assumption（"I'm assuming 30% based on..."）

## 四、Role Play / Client Simulation

### 形式
- 你扮演 Consultant / Banker / Account Manager
- 评估官扮演 Client / Stakeholder
- 15-20 分钟一对一对话

### 4 步走

```
Step 1: 听需求 (5 min)
  - 问开放问题："What outcome would success look like for you?"
  - Repeat back："So if I'm understanding correctly, you need..."

Step 2: 澄清细节 (3-5 min)
  - 问具体数据："What's the current X?"
  - 问时间线："When do you need this delivered?"
  - 问约束："What's the budget / non-negotiable constraint?"

Step 3: 给 1 个清晰建议 (5 min)
  - "Based on what you've shared, I'd recommend [option]. Here's why..."
  - 给 2-3 个支持理由
  - 主动 acknowledge trade-offs："The downside of this is X, but..."

Step 4: 确认下一步 (2 min)
  - "Does this resonate? What questions do you have?"
  - "If we proceed, I'd suggest [concrete next step]."
```

### ✅ 高分点

- **不要假装专家** — Client 比你懂得多，你的价值是结构 + 框架
- **数字 / 案例支持** — 引用一个相关案例 ("In a similar situation with ASX 200 retail client...")
- **acknowledge emotion** — 如果 Client 表达 frustration，先 acknowledge："That sounds frustrating — let me see how we can help."
- **say no when needed** — 如果 Client 要求 unreasonable，礼貌拒绝："I want to make sure we deliver real value here — that timeline would compromise the analysis. Could we discuss alternatives?"

## 五、1v1 Behavioural / Partner 面

- 这是你 STAR 故事的主场（worksheet 03 Story Bank）
- 通常 30-60 分钟 · 5-8 个 Behavioural + Why this firm + Why you
- AC 最后阶段 — 你已经累了 · 但 Panel 在评"压力下你的状态"

**AC 1v1 vs 普通面试 4 个差别**：
1. 面试官可能就是看你群面的人 — 不要前后矛盾
2. 问题更深入 follow-up（"Tell me more about Result"）
3. 经常问 "Why us vs [Competitor]" — 你必须做过具体调研
4. 最后通常会问 "Do you have any questions for us?" — 准备 3 个深度问题

## 六、AC 当天 24 小时检查清单

### T-24h（前一晚）

- [ ] 衣着确定（澳洲 GP 默认 business formal · Tech 公司 smart casual）
- [ ] 鞋子擦干净 / 头发整理
- [ ] 公司前一天 fix 通勤路线（地铁 / 车）— 早到 30 分钟为底线
- [ ] 5 个深度反问准备好（"Do you have questions for us?" 必问）
- [ ] 故事 5-7 个再过一遍（worksheet 03）
- [ ] 笔 + 笔记本 + 一份打印简历（5 份）+ 一份 photo ID
- [ ] 公司最近 1 个月新闻 / 财报 / LinkedIn post 扫一遍
- [ ] 8 小时睡眠 · 不熬夜补简历 · 不咖啡过量

### T-1h（出门前）

- [ ] 早饭吃了
- [ ] 充电宝 / 手机充满
- [ ] 公司地址 + 联系人 + 紧急电话记在手机外（万一手机没电）

### T-30 min（到 office 楼下）

- [ ] 提前 30 分钟到 — 楼下咖啡店 / 公园定 15 分钟
- [ ] 提前 15 分钟进 building / 报到
- [ ] 见到接待 / 其他候选人主动 self-introduce + 微笑

### T 内（AC 过程中）

- [ ] **早到的 30 分钟 + 午餐时间也在评估** — 不要在 break 抱怨 / 不要刷手机不理人 / 主动和其他候选人聊天
- [ ] 群面：先听后说 · 拉沉默的人 · 主动 timekeeping
- [ ] Case：写思路给 panel 看 · 主动 acknowledge gaps
- [ ] Role Play：repeat back 需求 · 给 1 个清晰建议
- [ ] 1v1：眼神 + 微笑 · 故事按 STAR · 不要"我感觉..."（说事实）
- [ ] 结尾：5 个反问 · 主动 thank panel members 个人

### T+24h（AC 后）

- [ ] 24 小时内给 panel 发 thank-you email（如果有名字）
- [ ] 你自己 Notion 复盘：哪个环节做得好 / 哪个环节卡住 / 下次怎么改
- [ ] 等结果（通常 2-4 周）— 这期间继续投其他 GP，不要停

## 七、AC 5 个反问清单

最后 panel 问 "Do you have any questions for us?" 时，准备 3-5 个：

1. **"Looking back at this year's grad cohort, what's the one thing you think differentiated the ones who really thrived in the first 6 months?"**
   — 拿成功标准

2. **"How does feedback typically work in this team — formal review cycle vs day-to-day?"**
   — 看公司文化

3. **"Given the recent [industry trend / company news], how do you see the [team/sector] evolving over the next 2-3 years?"**
   — 显示你做过功课

4. **"What's a challenge the team is working on right now that you wish you had more help with?"**
   — 试探团队真实需求 · 让 panel 觉得你能解决问题

5. **"What's been your favorite project to work on in your time here?"**
   — 让 panel 谈自己 · 建立 rapport
