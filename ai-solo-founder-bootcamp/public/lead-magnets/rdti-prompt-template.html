背景

什么是 RDTI · 为什么 OPC 必看

R&D Tax Incentive (RDTI) 是澳洲税务局（ATO）+ AusIndustry 联合管理的研发税务激励计划。年营业额 $20M 以下的小公司：符合条件的 R&D 支出可拿 43.5% 现金退税（refundable tax offset，亏损公司也能拿到现金）；年营业额 $20M+ 的拿 38.5%（non-refundable）。

为什么 OPC 学员特别该看：① AI / 软件研发本身可被认定为 R&D activity；② 退税额度通常 $30k-$200k 一笔，远超 OPC 创业营的学费 ROI；③ AusIndustry 在 2024 年起特别欢迎 AI / Generative AI 类研发申请。

ATO 审计严格的 3 个点：① 必须有明确的『技术不确定性』(technical uncertainty) — 申请人不知道结果，需要实验；② 必须有系统化实验设计（hypothesis → method → result → analysis）；③ 必须有R&D 活动 vs 普通商业活动的清晰区分（写代码不等于 R&D，但解决新算法 / 新模型微调可能是）。

SECTIONS

ATO RDTI 申请书 6 大 section（必填）

RDTI 申请通过 R&D Tax Incentive Customer Portal 提交，每年 4 月 30 日是上一财年的截止日（错过即失效）。完整申请要覆盖以下 6 部分：

SECTION 1

R&D Entity Details

实体类型（Pty Ltd 必备 — sole trader 不能申请）、ABN、ACN、年营业额、行业代码。

SECTION 2

Core R&D Activities

核心 R&D 活动 — 含 Hypothesis / 实验方法 / 预期结果 / 知识鸿沟。每个 activity 单独描述。

SECTION 3

Supporting Activities

支持性活动 — 服务于 core 但不构成实验的部分（数据准备、文档、UI 等）。区分 vs core 是 ATO 重点审计项。

SECTION 4

Project Description

整体项目背景 + 商业目标 + 与 R&D 关系。500-2000 字，要展示 R&D 与商业化路径。

SECTION 5

Expenditure

R&D 支出明细：员工工资 / 合同工 / 软件订阅 / 云服务 / 测试设备。每一笔要有发票 + 时间日志支撑。

SECTION 6

Records & Compliance

R&D 记录证据：实验日志 / 测试结果 / 代码 commit history / 会议纪要。ATO 可能审计要求出示。

Supporting vs Core 区分清晰 5. 输出长度：Section 2 (Core Activities) 800-1500 字 / Section 3 (Supporting) 300-500 字 / Section 4 (Project Description) 600-1000 字 / 其他按需 6. 用 plain English 写（避免行业黑话），ATO 评审是普通会计 / 公务员可能不懂深度技术 7. 末尾列出 5 个 ATO 可能的审计追问，并给出回答方向 输出格式：Markdown，6 section 用 ## 二级标题分隔。 开始起草。

使用 checklist

  • 所有 [尖括号] 替换为你的真实信息（不留空）
  • Core Activity 必须有真实 hypothesis + 真实结果（编造的会被 ATO 看穿）
  • 分段 prompt — 先 Section 2，再 Section 3，再 Section 4
  • AI 输出后人工 review — 特别是 Knowledge Gap 和 Technical Uncertainty 的语言精度
  • 提交前必须找有牌照的 Grant Consultant / 税务师 review（不要省这步）
  • 保存所有 R&D 记录（Git commits / 实验日志 / 发票）至少 5 年（ATO 可追溯审计）
常见拒理由（避免）：① Hypothesis 写成『我们想做 X』而非『我们假设 Y 方法可能解决 Z 问题』；② Core 跟 Supporting 混淆 — 写代码本身不算 Core，解决某个新技术问题才算；③ Expenditure 没发票支撑；④ 没有系统化实验记录；⑤ 把普通商业开发包装成 R&D（ATO 审计组对软件类审计经验丰富，看得出）。
INPUTS

跑 prompt 前要准备的真实数据

在喂给 AI 之前，先收齐这些信息（用 30-60 分钟）：

公司信息
ABN + ACN（实体注册号）+ ANZSIC 行业代码（查 abr.business.gov.au）+ 财年 + 营业额估算
R&D 时间线
列出本财年内（7/1 - 6/30）你做的 1-3 个核心 R&D activity，每个 activity 起止时间
技术不确定性证据
每个 activity 开始时『不知道什么 / 已知方法不够用』 — 找到当时的对比文献 / Stack Overflow 问题 / 内部讨论记录
实验记录
每个 activity 的实验设计 + 数据 + 最终结果（即使失败也要写 — ATO 接受『实验失败』的 R&D）
Expenditure 明细
从 Xero / MYOB / 银行流水拉本财年 R&D 相关支出（工资 / API / 云服务 / 软件 / 外包）
证据文件链接
Git repo URL / Notion 实验日志链接 / Slack 决策记录 / 关键发票扫描件
