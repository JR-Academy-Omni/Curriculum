🎯 第一节课定调：OPC 不是创业，是另一种自由

OPC 跟传统创业最大的区别是：拒绝融资、永远不雇人、用 AI 替代员工、目标 $1k-$10k MRR 的自由而不是 $1B 估值的雄心。课上拆 5 个澳洲华人真实案例（含程序员、服务业、小老板各类型），嘉宾来 1-2 个本地 indie hacker / OPC founder 讲产品 / 业务、用户 / 客户、定价、营销、踩过的坑。课后发"OPC 适配评估表"，低分学员 W2 末有最后一次退款机会。

课上 4 个 step：① OPC vs 传统创业 vs VC startup 三条路对比 ② 嘉宾分享 1（墨尔本 indie $3-10k MRR）③ 嘉宾分享 2（澳洲华人服务业 AI 化 case / 小公司 AI 接管业务 case）④ OPC 适配 5 维度自评。

🎯 idea 选完后必须做的事：跟 5 个真实潜在用户聊一次

不是问"你愿意付费吗？"——这是 indie hacker 最容易踩的坑（用户嘴上说愿意，掏钱时全跑）。正确问法是 Mom Test 那一套："你过去一个月遇到这个问题几次？""上次遇到时怎么解决？""为这个问题花过多少钱？"用 AI 模拟用户面试一遍后，去真实用户那聊 5 个。

L05a
LIVE · 90min
[Live] 定价模型选型：一次性 / 订阅 / Freemium / B2B / 服务+软件混合

🎯 建产品前必须先定的一件事：你的 pricing model

同一个 idea，选不同定价模型 → 产品结构、目标用户、获客渠道、技术架构全不同。不能等到 W3 接 Stripe 时才想——那时候产品已经按错的模型建好了。这节课在 user interview 之后、build 之前，把 pricing 一次定下来。

5 大定价模型 + 适用场景：

  • ① 一次性买断 (One-time) — 工具类 / 模板类 / 信息产品（Gumroad / Setapp 一次买）。决策门槛低、复购低。例：设计模板包、Notion 模板、AI prompt 包、ebook。
  • ② 订阅 (Subscription) — SaaS 标配。月付 $9-99 / 年付 8 折。MRR 可预测但 churn 是命门。例：Notion / Linear / Cursor / Lovable。
  • ③ Freemium — 免费拉用户 + 付费档转化（典型转化 1-5%）。用户基数大但服务器成本高。例：Notion / Figma / Loom。
  • ④ B2B / Enterprise — Per-seat / per-team / contract。客单价 $500-50k+/年。销售周期 1-6 月。例：Slack / Linear team / Vanta。
  • ⑤ 服务+软件混合 (Productized Service) — 软件做骨架 + 人工服务做交付。AI 简历 + 人工 review / AI 营销 + 人工策略。高毛利 + 高客单价 + 难替代，OPC 一个人最适合的模型之一，律师 / 会计 / consultant 通常走这条。

3 维决策框架：① 客户类型（C 端 vs B 端）；② 使用频次（每天 vs 一次性）；③ 创始人时间投入度（纯 SaaS vs 服务+软件）。课上每个学员用 3 维表评估自己 idea，得出推荐 pricing model + 价格 anchor。

价格 anchor 设置：选定模型后，定价不是拍脑袋——竞品调研 (5+ 家) + 价值锚 (替代方案多少钱) + 早鸟折扣 + Annual vs Monthly 折扣。课上发"定价计算器 + 竞品对照表 + 5 行业 anchor 数据"。

跟 W3 L13 联动：定价模型决定 Stripe / Lemonsqueezy 的接入方式（一次性 = Checkout / 订阅 = Subscriptions / B2B = Custom invoicing / 混合 = 多产品组合）。

L11a
LIVE · 120min
[Live] Source of Truth + AI 一稿出全套（Website / Pitch / Brochure / 一页纸 / 公众号）

🎯 W2 末核心动作：一份 source of truth → AI 一次喂出全套对外 asset

OPC 学员最大的时间黑洞是反复写产品文案——网站、pitch deck、brochure、公众号、小红书、Email pitch、媒体稿，每一份都从头写。这节课教你做对的方式：先写 1 份 markdown source of truth（产品定位 / USP / pricing / target audience / FAQ / 5 个 use case / 对比同类产品），再用 AI 一次性喂出所有对外 asset。

实操产出（同一份 source 跑 6 个工具）：① Lovable / v0 → landing page；② Gamma / Beautiful.ai / Pitch.com → pitch deck（投 VC + 找客户两版）；③ Canva AI / Adobe Express → A4 brochure + 一页纸 PDF；④ Notion AI → 客户 onboarding 文档；⑤ 公众号长文 + 小红书爆款笔记标题；⑥ X / LinkedIn 个人简介 + bio。

核心约束：source 改一个字，所有 asset 重新生成。绝不在某一份 asset 里手改文案——改了就回去改 source。这是 AI 时代 OPC 的唯一可持续做法。

L11b
LIVE · 120min
[Live] 选你的 AI 秘书 — 5 方案深度对比 + 选型决策

🎯 Agent OS 第一节：从 5 个方案里选定接下来 4 节课要装的那个

OPC 学员配置 AI 秘书有 5 个主流方案，定位重叠（都是个人 AI assistant），各有偏向。这节课不教装机——先选好走哪条路，后面 4 节课都按你选的方案配。

5 方案介绍：

  • ① Claude Cowork（推荐 default）—Anthropic 2026 GA 桌面 App，Claude Desktop 第三个 tab。给目标 → 它在桌面跑：开 Excel / 读本地文件 / 调 Connectors / 按 /schedule 自动跑 / 手机 Dispatch 远程派活。不用终端、不用 npm、全是说人话 — 1 小时上手。强项：GUI + 非技术友好；弱项：数据走 Anthropic 云。参考 JR 自家 claude-cowork-workshop。
  • ② OpenClaw—NousResearch 开源 CLI agent（前身 Moltbot / ClawdBot），类 Claude Code 的 CLI 形态，MEMORY.md + USER.md 本地 markdown。强项：本地优先 + 开源可改 + 支持非 Claude 的 LLM；弱项：要会一点终端。JR 4 周 OpenClaw Bootcamp 专门教这个。
  • ③ Hermes Agent—NousResearch 另一款开源跨平台 agent。~/.hermes/ FTS5 memory + Slack/Telegram/Discord/WhatsApp/Signal/CLI gateway。强项：手机 IM 直接对话 + 跨设备同步；弱项：编码任务弱。
  • ④ Lindy.ai—第三方 SaaS workflow agent，UI 漂亮、集成 100+ 工具。$50-300/月 + 数据上他们云。
  • ⑤ 自建—Crew AI / LangGraph + 自己 memory store。要写 200-500 行 Python，OPC 阶段不推。

决策树（直接选）：

  • 选 ① Cowork：1 小时上手 + GUI + 已订 Claude Pro + 桌面重度（PPT/Excel/浏览器）+ 不在意数据上云 + 想手机派活
  • 选 ② OpenClaw：律师/会计/医疗/合规岗（数据不能上云）+ 想用非 Claude 的 LLM + 已会终端
  • 选 ③ Hermes：团队已用 WhatsApp/飞书/Slack + 跨设备 IM 派任务
  • 选 ④ Lindy：完全零代码 + 不想 self-host + $50-300/月预算 OK
  • 选 ⑤ 自建：写代码的洁癖型（OPC 阶段不推荐）

🎯 推荐：大多数 OPC 学员选 ① Claude Cowork。隐私敏感行业（律师/会计/医疗）走 ② OpenClaw。其实大多数实战场景是 Cowork + OpenClaw 配合用 — Cowork 干桌面 GUI 活、OpenClaw 干合规敏感 + IM 协作活。L11f 进阶课讲什么时候补第二个。

Lab 分 4 站点（自建路径学员自学），按 L11b 选的方案进对应站点，TA 1:8 陪跑。已上过 Claude Cowork Workshop 或 OpenClaw Bootcamp 的学员可以兼任 peer mentor。

L11d
LIVE · 120min
[Live] 接通公司核心数据：Gmail / Calendar / Drive / Notion / Slack 全部喂进 agent

🎯 让你的 agent 真正"知道你的公司在发生什么"

L11b 装好 OpenClaw、L11c 接好 Hermes 记忆层。这节课把公司核心数据源全部接进来 — agent 才有上下文回答 OPC 的真实问题（不接数据 = agent 只能聊天，没法干活）。

5 个数据源 + 接入方式（120min 全演示）：

  1. Gmail / Outlook 邮件（30min） — OpenClaw skill：OAuth → 索引最近 90 天邮件 → Hermes FTS5 入库。本地索引优先，邮件正文不上云
  2. Google Calendar / Outlook Calendar（15min） — Calendar API 读未来 30 天事件 + 历史 1 年画像
  3. Google Drive / OneDrive 文档（25min） — 选择性接：Working / Customer / 财务文件夹，敏感文件夹排除
  4. Notion / Confluence（20min） — Notion API integration token 全 workspace 索引
  5. Slack / Discord / Telegram 群消息（20min） — Hermes gateway 直接接，channel 白名单管理
  6. 整合验证（10min）：现场问"客户 X 最近一次邮件提到什么 + standup 决定了什么"，agent 跨数据源回答

隐私 4 档：① 本地 only（Gmail / WhatsApp 必须）② 自建云（Supabase pgvector）③ zero-retention 端点（Anthropic / OpenAI）④ 完全云端（仅公开内容）。课上发数据源 × 隐私级别决策表。

L11e
LIVE · 120min
[Live] 7 个秘书任务现场 ship — 让你有"企业秘书 / 大脑"的真实感觉

🎯 W2 Agent OS 总收官 — 让 agent 真正接管 OPC 的日常工作

前面 L11b/c/d 把 OpenClaw + Hermes + 数据源都装好了。这节课现场把 agent 配成"真的秘书" —— 不是聊天玩具，是每天 8 点 push 总结、自动跟进客户、出月报的执行机器。

7 个秘书任务（现场至少跑通 4 个，剩下作为 W2 末 deliverable）：

  1. ① 收件箱总结 — 每天 8:00 push 到 Telegram，重要/待回/噪音三档 + 重要邮件回复 draft（OpenClaw skill + cron + Hermes Telegram gateway）
  2. ② 会议笔记自动整理 — Otter.ai / Granola 录音 → Hermes 入库 → AI 抽决策清单 + action items + owner
  3. ③ 日程协调 + 客户预约 — Cal.com / Calendly 接 OpenClaw skill，agent 看日历给客户回时段
  4. ④ 客户跟进队列 — 每周一 7:00 push：最近 7 天没回的客户 + 上次对话上下文 + AI 跟进文案 draft
  5. ⑤ 合同审查 — 联动 W3 L16 AI 律师角色 — agent 看到合同 PDF 自动 review 风险点 + 红线清单
  6. ⑥ 财务月报 — 每月 1 日 8:00 push — Stripe API + Xero API → MRR / churn / GST 准备金 / 现金流月报
  7. ⑦ 周报生成 — 每周日 18:00 — agent 看本周 git commit + 邮件 + 日历 + Hermes memory，自动出周报

课上 demo 顺序：先 ① 收件箱总结（最容易出效果），再 ⑥ 财务月报（OPC 学员最爽）。验收：W2 末每个学员至少跑通 4 个秘书任务（一周的 push 输出截图）。

L11f
SELF · 60min
[Self] 进阶 — 切换 / 双 agent 配合 / 自建 / 团队 Brain

🎯 L11b 选了主线 agent 跑通后 — 下一步怎么演化

L11b-e 让你装好了一个主线 AI 秘书。这节自学课讲 4 种进阶场景：

① 切换主线 agent（用一段时间想换） — 用了 Cowork 半年觉得隐私顶不住 → 切 OpenClaw；用了 OpenClaw 觉得 CLI 累 → 切 Cowork。每对组合的迁移路径：导出 system prompt + memory + skill 列表 → 在新 agent 重新喂入。Cowork ↔ OpenClaw 的 USER.md / system prompt 格式 80% 兼容。

② 双 agent 配合（80% 学员最终走这条） — 实战中大多数 OPC 学员最后是 Cowork + OpenClaw 配合用：Cowork 干桌面 GUI 活（PPT / Excel / deck / Connectors），OpenClaw 干合规敏感 + IM 协作 + daemon 任务。两套都装、各管一边，memory 不互通但任务清单分明。课上发『双 agent 分工 SOP』。

📐 Agent Schedule 跨平台对照（30min 概念 + 配置）

  • Claude Cowork /schedule：内置 5 种 cadence（hourly / daily / weekly / monthly / custom），UI 配置最简单。限制：必须 Mac awake + Claude Desktop App 开着
  • Claude Code cron skill：自定义 skill + 系统 cron / launchd，要写 wrapper script，灵活但要会 shell
  • Codex CLI 定时任务：通过系统 cron 或 OpenAI Scheduled Tasks，跟 Claude Code 思路类似
  • OpenClaw cron：自由配 cron 表达式，可后台 daemon 常驻不依赖 Mac awake
  • n8n / Zapier scheduled trigger：可视化工作流编排，免装机直接 SaaS 跑
  • 系统 cron / launchd / GitHub Actions cron：最底层兜底，跑任意脚本（包括调任意 agent CLI）

选型决策：① Cowork 学员用 /schedule 直接配；② OpenClaw 学员用 cron + daemon；③ 想完全脱离本机依赖 → n8n self-host on VPS / GitHub Actions cron + 调 Claude API 直跑。

LIVE · 120min
[Live] AI 生成法律文件 + AI 律师审核（Privacy / TOS / Disclaimer / Refund / Cookie · 澳洲版）

🎯 OPC 不雇律师怎么搞定法律文件 — AI 双层流程

第一笔付费收完之前必须挂的 5 份法律文件：Privacy Policy（澳洲 APP 隐私准则 + GDPR 双合规）、Terms of Service、Refund Policy、Cookie Policy、Disclaimer。雇澳洲律师起草这套 = $3-8k；用 AI 生成 + AI 律师审核 = $0 + 2 小时。

第一层：AI 生成（30min）：Termly / iubenda 模板 + Claude / GPT 个性化填充（产品类型、收款方式、数据处理位置、用户地区）。课上发"澳洲科技产品 5 文件 prompt 包"，学员现场跑出 5 份文件初稿。

第二层：AI 律师审核（45min）：把生成的文件喂回 Claude / GPT，用专门的"澳洲科技律师"角色 prompt review — 检查 ACL（澳洲消费者法）条款、ATO（澳洲税务局）表述、GDPR 跨境数据传输条款、consumer protection 漏洞、SaaS 退款政策合规性。AI 律师会列出"高风险 / 中风险 / 低风险"三档清单，学员逐条修。

什么时候必须找真人律师（30min）：① 投资协议 / SAFE notes / 股权分配；② 首位员工劳动合同（澳洲 Fair Work 极复杂）；③ 重大客户合同（B2B 单笔 $20k+）；④ 跨境出海（美国 Delaware C-corp / 中国数据出境）；⑤ 被人 sue 或威胁 sue。课上发"澳洲华人科技律师推荐名单"（3 位 fixed-fee 律师，跟 Bootcamp 学员合作）。

毕业 deliverable：5 份文件挂在你的产品 footer，AI 律师审核报告作为附件。
