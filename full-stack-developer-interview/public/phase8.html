项目深挖表达框架

面试官听项目时在想什么？

  1. 技术判断力 — 你能不能做合理的技术选型？
  2. 问题解决能力 — 遇到困难你怎么处理？
  3. 影响力 — 你在团队中推动了什么改变？
  4. 自我认知 — 你知道自己做得好/不好的地方吗？

项目表达模板 (3-5 分钟)

1. 一句话概括 (30s)
   "我在 XX 公司负责了一个 XX 系统，核心是解决 XX 问题"

2. 技术架构 (60s)
   前端什么框架 + 后端什么 + 数据库 + 部署

3. 你的关键决策 (60s)
   "我选择了 A 而不是 B，因为..."
   "权衡是..."

4. 最大的技术挑战 (60s)
   "遇到的问题是..."
   "我的解决方案是..."
   "具体做了 1) ... 2) ... 3) ..."

5. 量化结果 (30s)
   "性能提升 XX% / 用户增长 XX / 错误率降低 XX"

权衡取舍的高级表达

面试官最看重的不是你选了什么，而是你为什么选、放弃了什么、代价是什么。

初级回答: "我们用了 React"
高级回答: "我们对比了 React 和 Vue。选 React 因为：
  1) 团队 4 个人都有 React 经验，上手成本低
  2) 项目需要 SSR，Next.js 比 Nuxt 生态更成熟
  3) 大量现成的组件库可以加速开发
  代价是 React 学习曲线比 Vue 陡，但长期维护性更好"

Agile/Scrum 面试速通

本节是 Agile/Scrum 面试题的快速回顾。详细内容请看 Phase 0 的 V04-V09 录播课。

Agile 4 个价值观（30 秒说清楚）

  1. 个体和互动 > 流程和工具 — 有问题先找人聊，不是先开 ticket
  2. 工作的软件 > 详尽的文档 — MVP 先上线，文档够用就行
  3. 客户合作 > 合同谈判 — 持续对齐需求
  4. 响应变化 > 遵循计划 — Sprint 中方向错了及时调整

Scrum 流程速查

事件时长做什么
Sprint Planning2-4hPO 讲优先级，团队选 story，估点
Daily Standup15min昨天做了啥、今天做啥、有没有 blocker
Sprint Review1-2hDemo 给 stakeholder，收集反馈
Retrospective1h什么做得好、什么要改进

Top 5 面试题 + 标准回答

Q1: Sprint 中需求变了怎么办？

和 PO 讨论优先级变化。如果新需求更紧急，可以协商替换掉低优先级 story。但不能无限加需求（scope creep）。

Q2: 你怎么估时间？

用 Story Points 估复杂度而不是时间。Planning Poker 同时出牌避免锚定效应。用过去 3-5 个 Sprint 的 Velocity 预测交付能力。

Q3: 和 PM/设计师意见不同怎么办？

用数据和用户反馈说话，不靠技术权威。提出替代方案，让数据决定。

Q4: Scrum 和 Kanban 有什么区别？

Scrum 有固定 Sprint（2周），有角色（PO/SM），Sprint 内不加需求。Kanban 持续流动无固定周期，用 WIP Limit 控制在制品。产品开发用 Scrum，运维/支持用 Kanban。

Q5: 什么是 Definition of Done？

团队统一的完成标准：代码合并 + 测试通过 + Code Review + 部署到 staging + PO 验收。
