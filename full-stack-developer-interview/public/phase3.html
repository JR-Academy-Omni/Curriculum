HTML/CSS 面试高频题

HTML 面试题

1. 语义化标签有哪些？为什么要用？

语义化让 HTML 结构有意义，利于 SEO、无障碍和维护。

标签语义替代的 div 用法
<header>页头/区块头<div class="header">
<nav>导航区域<div class="nav">
<main>页面主要内容<div class="main">
<article>独立的内容块<div class="post">
<section>内容分组<div class="section">
<aside>侧边栏/附属内容<div class="sidebar">
<footer>页脚<div class="footer">

2. DOCTYPE 的作用

<!DOCTYPE html> 告诉浏览器用标准模式（Standards Mode）渲染页面，而不是怪异模式（Quirks Mode）。怪异模式下盒模型计算方式不同，CSS 表现不一致。

3. meta viewport 的作用

<meta name="viewport" content="width=device-width, initial-scale=1.0">

让移动端浏览器按设备宽度渲染页面，而不是按桌面宽度缩放。没有这个标签，你的响应式 CSS 在手机上不生效。

CSS 面试题

1. 盒模型

每个元素 = content + padding + border + margin

  • box-sizing: content-box（默认）— width/height 只包含 content
  • box-sizing: border-box（推荐）— width/height 包含 content + padding + border

面试追问：为什么推荐 border-box？—— 设置 width: 200px 就是 200px 的可视宽度，不用额外计算 padding 和 border。

2. BFC (Block Formatting Context)

BFC 是一个独立的渲染区域，内部元素的布局不影响外部。触发 BFC 的方式：

  • overflow: hidden/auto
  • display: flex/grid/inline-block
  • position: absolute/fixed
  • float: left/right

BFC 的用途：清除浮动、防止 margin 塌陷、自适应多栏布局。

3. Flexbox vs Grid

一维布局用 Flexbox，二维布局用 Grid。

/* Flexbox — 一行/一列排列 */
.container { display: flex; justify-content: center; align-items: center; }

/* Grid — 行列二维网格 */
.container { display: grid; grid-template-columns: repeat(3, 1fr); gap: 16px; }

4. 居中的 5 种写法

/* 1. Flexbox（最推荐）*/
display: flex; justify-content: center; align-items: center;

/* 2. Grid */
display: grid; place-items: center;

/* 3. Position + Transform */
position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%);

/* 4. Margin auto（块级水平居中）*/
margin: 0 auto;

/* 5. Line-height（单行文本垂直居中）*/
line-height: 等于容器高度;

5. 响应式设计

Mobile-first：先写移动端样式，用 min-width 媒体查询扩展到大屏。

/* 移动端基础样式 */
.container { padding: 16px; }

/* 平板 */
@media (min-width: 768px) { .container { padding: 24px; } }

/* 桌面 */
@media (min-width: 1024px) { .container { max-width: 1200px; margin: 0 auto; } }

JavaScript 核心机制 — 面试最高频考点

1. var / let / const

varletconst
作用域函数作用域块级作用域块级作用域
提升有（值为 undefined）有（TDZ 不可访问）有（TDZ 不可访问）
重复声明可以不可以不可以
重新赋值可以可以不可以（引用不变）

经典面试题：for 循环中的闭包

// var — 输出 5 个 5
for (var i = 0; i < 5; i++) {
  setTimeout(() => console.log(i), 0);
}

// let — 输出 0,1,2,3,4
for (let i = 0; i < 5; i++) {
  setTimeout(() => console.log(i), 0);
}

原因：var 是函数作用域，循环结束后 i=5。let 是块级作用域，每次迭代创建新的绑定。

2. 闭包

函数能记住并访问定义时的词法作用域。实际用途：

// 防抖（面试高频手写题）
function debounce(fn, delay) {
  let timer = null;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 模块模式
const counter = (() => {
  let count = 0;
  return {
    increment: () => ++count,
    getCount: () => count
  };
})();

3. 原型链

每个对象都有 __proto__ 指向其构造函数的 prototype。属性查找沿着原型链向上，直到 null。

function Person(name) { this.name = name; }
Person.prototype.greet = function() { return 'Hi, ' + this.name; };

const p = new Person('Stan');
p.greet(); // 'Hi, Stan' — 在 prototype 上找到
p.toString(); // 在 Object.prototype 上找到

4. Event Loop

console.log('1');                         // 同步
setTimeout(() => console.log('2'), 0);    // 宏任务
Promise.resolve().then(() => console.log('3')); // 微任务
queueMicrotask(() => console.log('4'));   // 微任务
console.log('5');                         // 同步
// 输出: 1 → 5 → 3 → 4 → 2

5. this 绑定

四条规则（优先级从高到低）：

  1. new 绑定 — new Foo() 中 this 指向新对象
  2. 显式绑定 — call/apply/bind 指定 this
  3. 隐式绑定 — obj.method() 中 this 指向 obj
  4. 默认绑定 — 独立调用，this 指向 window（严格模式 undefined）

箭头函数没有自己的 this，继承外层的 this。

6. Promise / async-await

// Promise 链
fetch('/api/user')
  .then(res => res.json())
  .then(user => fetch('/api/orders/' + user.id))
  .then(res => res.json())
  .catch(err => console.error(err));

// async/await（更清晰）
async function getUserOrders() {
  try {
    const res = await fetch('/api/user');
    const user = await res.json();
    const ordersRes = await fetch('/api/orders/' + user.id);
    return await ordersRes.json();
  } catch (err) {
    console.error(err);
  }
}

React 高频面试题

1. Virtual DOM

Virtual DOM 是真实 DOM 的 JS 对象表示。React 用 Diffing 算法比较新旧虚拟 DOM 树，找出最小变更集，批量更新真实 DOM。

为什么不直接操作 DOM？—— DOM 操作触发 reflow/repaint 很昂贵。Virtual DOM 在内存中计算差异，一次性 commit 到真实 DOM。

追问：React Fiber？—— 把渲染拆成可中断的小单元，支持优先级调度，解决同步渲染阻塞主线程的问题。

2. Hooks 核心

useState

const [count, setCount] = useState(0);
// 函数式更新（依赖前一个状态时必须用）
setCount(prev => prev + 1);

useEffect

// 挂载时执行一次
useEffect(() => { fetchData(); }, []);

// 依赖变化时执行 + 清理
useEffect(() => {
  const sub = subscribe(userId);
  return () => sub.unsubscribe(); // cleanup
}, [userId]);

追问：useEffect 里能直接 async 吗？—— 不能，因为 useEffect 回调要返回 cleanup function 或 undefined，async 返回 Promise。解决：内部定义 async 函数再调用。

useCallback / useMemo

Hook缓存什么用途
useMemo计算结果昂贵计算（排序/过滤大数组）
useCallback函数引用传给 React.memo 子组件的回调

3. 状态管理选型

方案适用场景复杂度
useState + Props简单组件状态低
useContext跨层级共享（主题、语言）低
useReducer复杂本地状态逻辑中
Redux Toolkit大型应用全局状态高
Zustand中型应用，轻量替代 Redux中

4. Re-render 触发条件

  • setState 被调用（即使新值相同，class component 会 re-render，function component 用 Object.is 比较后跳过）
  • 父组件 re-render（除非子组件用 React.memo 包裹）
  • Context 值变化（所有消费者 re-render）

5. Custom Hook

function useDebounce(value, delay) {
  const [debounced, setDebounced] = useState(value);
  useEffect(() => {
    const timer = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(timer);
  }, [value, delay]);
  return debounced;
}

2. 泛型

// API 响应类型安全
async function fetchData<T>(url: string): Promise<T> {
  const res = await fetch(url);
  return res.json() as T;
}
const user = await fetchData<User>('/api/users/1');

// 泛型约束
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

3. Utility Types

Partial<User>     // 所有字段变可选（PATCH 请求）
Required<User>    // 所有字段变必选
Pick<User, 'id' | 'name'>  // 只取部分字段
Omit<User, 'password'>     // 排除字段
Record<string, number>     // 键值对
Readonly<User>    // 只读

4. 类型守卫

// typeof
function process(value: string | number) {
  if (typeof value === 'string') {
    return value.toUpperCase(); // TS 知道这里是 string
  }
  return value.toFixed(2); // TS 知道这里是 number
}

// 自定义类型守卫
function isUser(obj: any): obj is User {
  return 'id' in obj && 'name' in obj;
}
