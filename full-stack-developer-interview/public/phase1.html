全栈面试到底考什么？

澳洲 IT 公司的全栈面试通常分 4-5 轮：

  • Phone Screen (15-30min) — HR 聊背景、签证、薪资预期
  • Coding Test / Take-home (1-3h) — 写代码解决实际问题，关键是代码质量 + 边界处理 + 测试
  • Technical Interview (45-60min) — 前端/后端高频题，回答结构：结论 → 原理 → 示例 → 权衡
  • System Design (45-60min) — 需求澄清 → 高层架构 → 数据模型 → 扩展性
  • Behavioral / Culture Fit (30-45min) — STAR 格式讲真实经历

求职匠是什么？

JR Academy 开发的 Chrome Extension：

  • JD 一键分析 — 打开 Seek、LinkedIn 职位页，AI 自动分析匹配度
  • 技能差距识别 — 对比简历和 JD，列出缺少的技能
  • 面试准备建议 — 根据 JD 生成可能的面试问题
  • 申请追踪 — 保存分析结果，跟踪申请状态

简历优化

5 个常见致命错误

  1. 没有量化成果 — 用数字说话
  2. 技术栈堆砌 — 只列能讲清楚的
  3. 项目描述流水账 — 用成果驱动
  4. 格式不规范 — PDF，一页以内
  5. 没针对 JD 定制

Behavioral 面试 STAR 框架

  • S — 2-3 句话交代背景
  • T — 你的具体职责
  • A — 具体做了什么（重点，占 50%）
  • R — 量化结果
