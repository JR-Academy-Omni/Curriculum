Prompt Engineering 核心技巧

Zero-shot

直接给指令，不给示例。适合简单任务。

Few-shot

给 2-3 个输入输出示例。适合格式要求严格的任务。

将以下文本分类为 positive/negative:
"这个产品太好了" → positive
"服务很差" → negative
"价格合理质量也不错" →

Chain-of-Thought

要求模型一步一步推理。适合数学和逻辑题。

Q: 一个班有 30 个学生，60% 是女生，其中 1/3 戴眼镜。戴眼镜的女生有几个？
Let's think step by step:
1. 女生数量: 30 × 60% = 18
2. 戴眼镜的女生: 18 × 1/3 = 6
A: 6 个

Prompt Chaining

复杂任务拆成多步，前一步输出作为后一步输入。

AI Agent + RAG 实战

RAG 架构

用户提问 → Embedding 向量化 → 向量数据库检索 Top-K
         → 拼接 Prompt (System + 检索结果 + 用户问题)
         → LLM 生成回答 → 返回用户

关键设计决策

环节选项推荐
Chunk 大小200/500/1000 token500-1000，overlap 10-20%
Embedding 模型OpenAI ada/small/largetext-embedding-3-small（性价比）
向量数据库Pinecone/Weaviate/pgvectorPinecone（托管）或 pgvector（已有 PG 时）
Reranking向量相似度 / Reranker 模型Top 20 → Rerank → Top 5

RAG 常见问题

  • 检索不准 → 优化 Chunk 策略、添加 metadata 过滤
  • 答案有幻觉 → 要求模型引用来源、设置温度为 0
  • 延迟高 → 缓存高频问题、预计算 Embedding

Streaming 实现

// 后端: Server-Sent Events
app.post('/api/chat', async (req, res) => {
  res.setHeader('Content-Type', 'text/event-stream');
  const stream = await openai.chat.completions.create({
    model: 'gpt-4o', messages: [...], stream: true
  });
  for await (const chunk of stream) {
    const content = chunk.choices[0]?.delta?.content || '';
    res.write('data: ' + JSON.stringify({ content }) + '\n\n');
  }
  res.end();
});

// 前端: fetch + ReadableStream
const response = await fetch('/api/chat', { method: 'POST', body });
const reader = response.body.getReader();
const decoder = new TextDecoder();
while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  const text = decoder.decode(value);
  // 追加到 UI
}

AI 项目面试表达模板

  1. 问题 (30s) — 用户需要 X，手工做效率低
  2. 方案选型 (30s) — 对比了 OpenAI/Claude/本地模型，选了 X 因为成本/质量/延迟/隐私
  3. 实现 (60s) — 前端→后端→LLM→结果处理的完整架构
  4. 效果 (30s) — 节省 X 小时/准确率 X%/用户满意度 X%
  5. 踩坑 (30s) — 幻觉处理/Token 超限/成本控制

面试追问应对

  • 「为什么选这个模型？」— 讲清楚 成本/质量/延迟/隐私 四个维度的权衡
  • 「成本多少？」— 估算 token 用量 × 价格，说出月度成本
  • 「幻觉怎么处理？」— RAG 检索增强 + 事实验证 + 温度设为 0
  • 「Prompt 注入怎么防？」— Input 验证 + System Prompt 防护 + Output 过滤
