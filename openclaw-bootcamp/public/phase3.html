L21

W3.1 自动化思维 — 把任务变成可重复跑的工作流

30 min 老师主讲
🔄 模式切换

W1-W2 你跟 Agent 是『一对一 chat 模式』 — 你说一句它做一件。这节起切到『工作流模式』 — 你定义一次（什么时候 / 做什么 / 发哪里），它自己跑。

📅 Cron 5 个常用场景

每天 8 点 / 每周一 9 点 / 每月 1 号 / 每小时 / 每隔 15 分钟。律师/地产/会计/内容各看 3 个本行业例子。

⚠️ 触发器 vs 定时

有些任务不是定时跑，是『新邮件 / 新客户进 CRM / 日历有新会议』触发。哪些用 cron 哪些用触发器，决策法老师讲。

L22

W3.2 Quest: 配第一个定时任务 — 早上 8 点自动整理收件箱

45 min 🌸 Quest
🌸 5 步 Quest

所有学员第一个定时任务都是『早上 8 点整理过夜收件箱』 — 几乎所有职业都需要、效果立刻看见。AI 小花带你写 cron 表达式 → 写 YAML 工作流 → 测试 → 启动 → 看效果。

🛟 守护规则

① YAML 缩进必须空格不能 tab
② cron 5 字段必须解释每个含义不能直接抄
③ 必须明确『电脑关机 cron 不跑』
④ 起草回复必须放草稿箱

📂 产出物

~/openclaw/configs/workflows/morning-inbox.yaml + 第二天早上 8 点真的跑 + Slack/WhatsApp 收到总结。

L23

W3.3 多步任务编排 — 一个工作流跑 5 个步骤

30 min 老师主讲
📦 真实工作流 5-10 步

L22 是单步（拉邮件 → 起草），但真实工作流多是 5-10 步。比如律师『新案件 intake』：客户邮件进 → 提取案件信息 → 写 Notion 案件库 → 生成首次咨询邀请发客户 → 提醒律师准备相关法规材料。

🔧 编排技术点

步骤间数据传递 / 条件分支（if 客户付了押金 then 走 A 否则走 B）/ 并发执行 / 依赖工作流（A 跑完才跑 B）

🐛 怎么调试

5 步工作流哪步挂了？怎么定位？老师演示日志查看 + 单步重跑 + 跳步执行。

L24

W3.4 Live Demo — 错误处理与监控

30 min Live Demo
🎬 老师故意把 Demo 跑挂

真实情况工作流会挂 — Gmail rate limit / Notion 写入冲突 / 邮件格式跟预期不一样 / LLM 抽风。老师故意触发 4 种典型挂法。

🛠 4 个工具

① 退避重试机制怎么配
② Slack/邮件错误通知
③ 错误日志在哪看（~/openclaw/data/logs/）
④ 『跑挂了不要重试』场景（重复发邮件给客户的事故）怎么避免

⚠️ 一句话总结

『Agent 自己跑』前提是挂了你能知道 + 修得快，否则跑越久越危险。

L25

W3.5 Workshop — 配你自己的 3 个工作流

75 min Tutor 1:8 陪练
📋 每人配 3 个

不是 1 个，不是 5 个。3 个本行业工作流，每个配完立刻跑测试。

🛟 Tutor 看着

YAML 格式 / cron 表达式 / 数据传递 / 错误处理 — Tutor 1:8 一对一帮你过。

🎯 走出去 3 个跑着

下课时这 3 个工作流不是『写好了等以后跑』，是已经在你电脑上启动 + 配好了定时。

⚖️ 律师专场推荐 3 个
  • 新案件 intake 工作流（客户邮件 → 案件库）
  • 合同审核 prep（合同上传 → 关键条款提取）
  • 月度 client 状态报告（每月 1 号自动生成）
🏠 地产专场推荐 3 个
  • 客户跟进每日提醒（8 点拉 CRM 该跟进的客户）
  • 房源新上自动通知客户（按客户偏好匹配）
  • 周报给 vendor（每周五自动生成本周房源数据）
💰 会计专场推荐 3 个
  • 每日 Xero 异常检查（早上 8 点扫 + Slack 通知）
  • 月底 BAS 提醒（每月 25 号给所有客户提醒）
  • 客户对账单自动生成（月初拉数据 + 起草邮件）
📱 内容专场推荐 3 个
  • 每天热点 → 选题建议（早上 7 点扫 5 个热点源）
  • 发文后 24h 互动数据汇总（自动拉评论 + 数据）
  • 周末选题包准备（每周日生成下周选题清单）
