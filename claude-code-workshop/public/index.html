学完能做到

5 件具体的事 · 不是 5 句口号

只承诺过程、作品、技能 —— 走出会场，你的 repo 比来之前多了真实的东西。

  1. 在动手写代码前，先写出一份「AI 能听懂」的需求清单 + 架构设计 —— 不是「帮我写一个登录系统」那种话
  2. 会用 Claude Code 几个关键命令：/init 让它读 repo、/plan 出方案、/resume 续上次的活、/clear 清空记忆、/compact 压缩长对话
  3. 同时开 3 个 AI 助手分工干活：一个查 repo 里现有的相似代码、一个写实现、一个做 review。半小时拿到完整 PR
  4. 给自己 repo 写一份「项目规则书」（CLAUDE.md）+ 一个团队自定义命令（比如 /code-review） + 一个提交前自动检查 hook。AI 自动遵守，新人不用再口头培训
  5. 跑通一次完整开发流程：从「我想做 X」到「PR 已提交」，AI 全程参与（写需求 → 写代码 → review → 提 PR）
最常被问的问题

你说我能自学吗？

来到这页的人 90% 在心里问这一句。我直接回答。

✅ 当然能。Anthropic docs 是免费的，YouTube 上有视频，ChatGPT 也能问。

但你要自己趟这些坑：

  • 哪几个命令真的常用、哪些其实没人用，看 docs 看不出来
  • /plan 出来的方案不对 —— 是 prompt 问题还是 model 问题？怎么调？
  • hooks 怎么配才不会卡 git commit 卡到崩溃？
  • 同时跑 3 个 AI 助手怎么避免它们互相覆盖文件？
  • CLAUDE.md 写多少条规则刚好？少了 AI 不听，多了它记不住
  • 哪些 MCP server 真的值得装？官方推的清单里 80% 用不上

自学这些坑大概要 30 小时（Reddit r/ClaudeAI 上学习者的常见反馈）。
