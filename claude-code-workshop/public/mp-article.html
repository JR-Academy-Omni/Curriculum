用 ChatGPT 写代码累？是时候认识 Claude Code 了

上周我帮一个朋友 review 他 GitHub 上的作品集 repo。三个 Next.js 项目，每个 README 都是同一段——「This is a Next.js project bootstrapped with create-next-app」。我问他为什么没改，他说他每次都让 ChatGPT 帮写，回来的版本永远是「项目介绍 + 技术栈 + 启动方式」三段模板。看一眼就知道是 AI 模板生成的。

这不是 AI 不够聪明，是他用 AI 的方式没跟上 2025 年的工具。市面上 AI 编程工具其实走过了三代：

  • 第一代 ChatGPT / 豆包 / 文心一言：网页聊天框。你贴一段代码问，它回一段建议。它根本不知道你整个项目长什么样。
  • 第二代 Cursor / GitHub Copilot：编辑器插件。你打字时它帮你补完这一行 / 这个方法。看的是光标附近几十行代码。
  • 第三代 Claude Code（这场大师课 讲的工具）：Anthropic（Claude 那家公司）自己出的命令行 AI 工具。装好之后你在 terminal 里跟它说话 —— 它能读你整个 repo，记住所有文件、命名规范、技术栈。你说「给登录模块加 Google OAuth」，它自己看哪些文件要改、自己改、跑测试、最后提一个 PR 给你看。你是它的产品经理，不是它的打字员。

差距不在 AI 智商，差距在你怎么让 AI 干活。这场 3 小时大师课 就是教你切到第三代工具的工作方式。

同样写一个登录接口 · 两种工具差在哪

具体一点。我观察过两类工程师同样要写「用户登录接口」时的全过程。

第一类用 Cursor：光标停在 auth.ts，问「帮我写一个用户登录」。AI 写了，跑一下报错（数据库连接没配），回去问「为什么报错」，改一下，再跑（密码加密用错库了），再问，再改。一上午折腾下来产出一个能跑但没人能维护的接口 —— 他一周后自己看都不知道当时为啥这么写。

第二类用 Claude Code：先在 spec/login.md 写一份「需求清单」 —— 要支持哪几种登录方式、密码怎么存、错误怎么提示、要不要限流。再写一份「架构说明」 —— 这个模块在 repo 里放哪、依赖哪些已有的组件、对外暴露什么接口。然后让 Claude Code 自己读这两份文件 + 整个 repo，同时开 3 个 AI 助手：一个查 repo 里已有的相似登录代码、一个写实现、一个准备 review checklist。半小时后他拿到的不是一段代码，是一个完整的 PR：含文档、实现、测试、和给 reviewer 的说明。

差别不在 AI 聪明不聪明，差别在 「你有没有把开发流程教给工具」。这场大师课 教的就是后者。

3 小时五段式 · 在你自己 repo 上跑一遍

线上速通版 180 分钟，匠人课堂 Zoom 实操。每段都有跟着敲的 demo，不是看视频。

  • Part 1（50min）开发前准备 + 关键命令 —— 动 Claude Code 之前你应该准备的事：怎么写一份「让 AI 听懂」的需求清单（不是「帮我写一个登录系统」这种话），怎么写一份「让 AI 不乱写」的架构设计。然后是 5 个核心命令：/init 让它读 repo、/plan 出方案、/resume 续上次的活、/clear 清记忆、/compact 压缩长对话。
  • Part 2（40min）同时跑多个 AI 助手干活 —— Claude Code 支持你一次开 3 个甚至更多 AI 助手并行。一个查 repo 里相似代码 / 一个写实现 / 一个做 review。这一段教你怎么分工、怎么避免它们互相覆盖。比串行干活快 3 倍。
  • Part 3（35min）让 AI 自动遵守你的规则 —— 给自己 repo 写一份「项目规则书」（CLAUDE.md）：命名规范、技术栈红线、不许动的文件、PR 必须做的事。AI 从此自动遵守，新人入职不用再口头培训。再配一个提交前自动检查，强制跑 lint / 测试 / 你的自定义检查。
  • Part 4（35min）完整开发流程跑一遍 —— 从「我想做 X」到「PR 已提交」的完整闭环：写需求 → AI 读 repo + 出方案 → 同时跑 3 个 AI 助手分工 → AI 自己 review → 提 PR。跟着老师在 demo repo 提一个真实 PR 出来。
  • Q&A 10min + 拿走 6 件模板 —— 加群继续答疑。

前 50 分钟是基础段，零基础学员（会 git clone / branch / commit + 命令行就行）保底跟上。后续 2 小时是高阶段，在职工程师拿到能直接用的技巧。中间用同一个 demo repo 串起来，跟不上的随时 git checkout part-3-start 跳过。

走出 3 小时 · 6 件能 commit 到自己 repo 的模板

不是「提升认知」这种虚的承诺。每一件你都能 clone 回自己 repo 直接用：

  1. 需求清单模板 —— 「让 AI 听懂」的写法。比「帮我写一个登录系统」这种话清晰 10 倍，AI 拿到能直接拆成具体步骤动手干。
  2. 架构设计模板 —— 把模块、接口、数据流先想清楚再让 AI 动手，AI 不会乱写一气。
  3. Claude Code 关键命令速查卡 —— 单页纸 5 个核心命令 + 什么时候用，打印贴墙。
  4. 项目规则书 CLAUDE.md —— 给自己 repo 写一份能直接 commit 的规则文件，AI 自动按你的命名习惯 / 技术栈红线 / 不许动的文件来工作。
  5. 3 件 AI 助手定制工具 —— ① 分身助手配置 ② 团队自定义命令（比如 /code-review） ③ 提交前自动检查。
  6. 完整开发流程 prompt 串 —— 从「我想做 X」到「PR 已提交」的对话模板，跟着填空就能跑。

加上持续答疑社群 —— 会后碰到坑能继续问，不是听完就散。

谁来听 · 谁不要来

这场是给两类人写的：

  • ① 在职 IT 工程师 —— 每天用 Cursor / Copilot 但还停在「让 AI 写下一行」的阶段，想知道怎么让 AI 帮你写完整个 feature。
  • ② 在校 / 转码学习者 —— 写过 demo 但项目永远停在「能跑」阶段，想让 GitHub 作品集像「真工程项目」而不是「教程项目」。

不适合从没写过代码的纯小白。前 30min 拉齐起点段也要求会基础 git（clone / branch / commit）+ 命令行。如果这两个你都没接触过，建议先看 1 小时入门视频再来。

三个最常被问的问题

Q：完全没用过 Claude Code 能跟上吗？

A：可以。前 30 分钟会拉齐起点，带你从零跑通第一个任务。但要求会基础 git + 命令行（如果你这两个都没接触过，建议先看 1 小时入门视频再来）。

Q：可以用 Cursor / Copilot 代替来跟课吗？

A：不可以。Claude Code 能做的事（读整个 repo / 同时开多个 AI 助手 / 自动跑测试 / 给 repo 写规则 AI 自动遵守）和 Cursor / Copilot 是两种工具，替换不了。

Q：线上 A$99 和线下 A$299 差别只是价格吗？

A：不是。线下 5 小时实操 vs 线上 3 小时 / 线下 ~15 人小班 + 助教 1v1 救场 / 线下含午餐茶歇 + 现场互评。线上更适合「想自己快速上手」，线下更适合「想被陪着把工具真正落到自己 repo 里」。
