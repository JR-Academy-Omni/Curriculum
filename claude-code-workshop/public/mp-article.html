把 AI 从「补全工具」升级到「任务编排」

上周我帮一个朋友 review 他的作品集 repo。三个 Next.js 项目，每个 README 都是同一段话——「This is a Next.js project bootstrapped with create-next-app」。问他为什么没改，他说每次都想 AI 帮他写一个像样的，但 Cursor 给的版本永远是「项目介绍+技术栈+如何启动」三段模板。

这不是 AI 不够聪明，是用 AI 的方式还停留在 2023 年——把它当聊天框，要什么补什么。Cursor / Copilot / Windsurf 这一代工具的天花板就在「行级补全」，再往上走就是另一种范式：把 AI 当成多线程协作的工程师来编排，不是当聊天对象来「问」。

同样在用 AI 编程，差距在哪

我观察过两类人写代码的过程。

第一类：打开 Cursor，光标停在文件里，问"帮我写一个用户登录的接口"。AI 写了，跑一下，报错，再问"为什么报错"，改一下，再跑。一上午改三个 bug，最后产出一个能跑但没人能维护的接口——连他自己一周后都看不懂为什么这么写。

第二类：打开 Claude Code，先写一份 PRD（产品需求文档）和 design.md（架构说明），把要做什么、模块怎么拆、接口怎么设计先想清楚。然后 /plan 让 Claude 读懂整个 repo，再发 3 个 subagent 并行——一个查现有相似代码、一个出实现方案、一个准备 review checklist。半小时后拿到的不是一段代码，是一个完整的 PR：含设计文档、实现、测试、和给 reviewer 的说明。

两种产出的差别，不是「AI 写得好不好」，是有没有把开发流程沉到工具里。这场 workshop 讲的就是后者。

不是教按钮，是教工作流

市面上 90% 的 AI 编程课在讲「怎么写 prompt 让 AI 给你写代码」。本场 workshop 不讲这个。讲的是把 AI 当成多线程工程师怎么编排——这是工具范式的差异，不是技巧差异。

3 小时跟着敲，覆盖 Agent 开发全生命周期。顺序刻意——前 50 分钟讲的不是 Claude Code 的命令，而是动 Claude Code 之前你应该准备的事：怎么写一份 AI 能直接读懂的 PRD，怎么写一份让 AI 不乱写的 design.md。这两件事写好了，后面的 /init /plan /resume 才有意义。

很多人卡在「AI 写出来不是我想要的」这一步，根本原因不是 AI 不行，是你给的输入是「写一个用户管理系统」这种话——AI 也只能给你一个泛泛的系统。换成有结构的 PRD + design.md，AI 拿到的信息量增加 10 倍，产出的代码自然不一样。

3 小时五段式 · 走通一遍

线上速通版 180 分钟，匠人课堂 Zoom 实操，每段都有跟着敲的 demo。

  • Part 1（50min）开发前准备 + 常用命令——写 PRD 模板 + design.md 模板 + 常用命令（/init /plan /resume /clear /compact）+ context window 工作原理。在自己 repo 写一份 mini-PRD，用 /plan 让 Claude 读懂全局。
  • Part 2（40min）Agents 管理——Fleet View / Agents View 演示。主 agent 和 subagent 怎么分工，一次发 3 个 subagent 并行跑 research / implementation / review。对比串行和并行的速度差异。
  • Part 3（35min）最佳实践——CLAUDE.md（项目宪法，让 AI 遵守你的命名约定、技术栈红线）、Skills（把团队 SOP 沉成可复用命令）、Slash Commands、Hooks（强制规则，比如 pre-commit 自动跑检查）、推荐 MCP server（context7 实时查文档、playwright 让 Claude 自己测页面）。
  • Part 4（35min）ADLC 实战——Agent Development Lifecycle 完整跑通：Plan（PRD / design.md）→ Code（subagent 并行）→ Review（自动跑检查）→ Ship（提 PR）。跟着老师在 demo repo 提一个真实 PR。
  • Q&A（10min）+ 拿走 cheat sheet——拿 PRD 模板 + design.md 模板 + 命令 cheat sheet + ADLC prompts + 加群。

前 50 分钟是基础段，零基础学员（会基础 git + 命令行就行）保底跟上；后续两小时是高阶段，在职工程师拿到能直接用的技巧。中间用同一个 demo repo 串起来，跟不上的随时 git checkout part-3-start 跳过。

走出 3 小时 · 6 件资产带回家

不是「提升认知」这种虚的承诺。每一件你都能 clone 回自己 repo 直接用：

  1. PRD 模板——AI 能直接读懂、拆 task、生成代码的写法。含结构 + 正反示例。
  2. design.md 模板——架构 / 模块拆分 / 接口约定，让 AI 不乱写。一页纸覆盖一个中型 feature。
  3. 单页命令 cheat sheet——/init /plan /resume /clear /compact + context window 原理图，打印贴墙。
  4. 自己写的 CLAUDE.md——给自己 repo 写一份能直接 commit 的项目宪法。
  5. Subagent + Slash + Hook 三件套——自定义 subagent 配置 + 一个 /<命令> + 一个 pre-commit hook。
  6. ADLC 工作流模板——Plan → Code → Review → Ship 完整的 prompt 串 + 教学示范。

加上持续答疑社群——会后碰到坑能继续问，不是听完就散。

谁来听 · 谁不要来

这场是给两类人写的：① IT 在职工程师——每天用 Cursor / Copilot 但还停在「高级补全」阶段，想把 AI 升级成「任务编排」；② 在校 / 转码学习者——写过 demo 但项目永远停在「能跑」阶段，想让作品集像「工程项目」而非「教程项目」。

不适合从没写过代码的纯小白。前 30min 拉齐起点段也要求会基础 git（clone / branch / commit）+ 命令行。如果这两个你都没接触过，建议先看 1 小时入门视频再来。
