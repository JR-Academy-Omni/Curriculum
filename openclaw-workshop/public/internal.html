🛟 Tutor 陪跑 Playbook（Tutor 必读）
装机环节（L02）最常见 5 个坑
  1. Windows 学员没装 WSL2 — 最常见，课前预习没做完。现场重装需要 15+ 分钟，建议让这类学员现在开始装，同时让 AI 小花 Quest 跳过 L02 前半段，他追上为止。
  2. Node.js 版本 < 22 — 报错是 ERR_MODULE_NOT_FOUND，看不出是版本问题。直接跑 nvm install 22 && nvm use 22。没装 nvm 的先 curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash。
  3. PATH 没刷新 — 装完 openclaw --version 提示 command not found。让学员重开终端或 source ~/.zshrc。
  4. API Key 填错 — 最常见是复制时带了换行 / 空格。让学员重新复制 + 确认没换行。
  5. Rate limit — 如果多个学员同时用 JR 共享 Key 会触发。改用个人 Key 或轮流测试。

🚨 什么时候必须从 AI 小花那里接手

  • 学员同一步卡了超过 5 分钟没进展
  • 学员在 Windows 原生 PowerShell 装（不是 WSL2）
  • 学员尝试用 Bun（WhatsApp/Telegram 会出 bug）
  • 学员情绪挫败 / 焦虑 — 这不是 AI 擅长的，人类共情接管

📣 每个 Tutor 课前必做

  • ☐ 课前 1 天在自己电脑上装一遍 OpenClaw（Mac + WSL2 各走一遍）
  • ☐ 提前准备好 nvm / pnpm 安装命令可以复制
  • ☐ 准备 2-3 个 JR 共享测试 Key（OpenAI + Claude + Gemini 各一）
  • ☐ 熟悉 Quest 6 步的每一步（见下面 L02 Quest 配置区）
🌸 L02 Quest 完整配置（给 AI 小花）
Quest 6 步 stepSkeleton
  1. 确认操作系统和终端 — verify uname -a 输出。Win 必须在 WSL2 bash 里。
  2. 检查 Node.js ≥ 22 — verify node --version。< 22 引导 nvm 升级。
  3. 全局安装 OpenClaw — pnpm add -g openclaw@latest（禁用 Bun）。
  4. 验证安装 — verify openclaw --version。
  5. 配置 LLM Provider — self-confirm （学员确认 key 填好）。
  6. 发第一条指令验收 — verify AI 自然语言回复。

完整 context / systemStylePrompt / customGuardrails 在 public/outline.json → L02 → quest 字段里。
