## 课程红线

写新 prompt 时必避：

- ❌ **承诺加薪 / 升职 / 包就业** —— 写"业绩翻倍"、"管理岗内推"是红线，用"输出方案"、"Demo Day"、"毕业作品"
- ❌ **承诺具体 ROI 数字 / 回本时间**（"3 个月回本 100 万"）—— 课程教 ROI 度量框架，不替企业承诺数字
- ❌ **真实公司 logo / 真实产品 UI**（OpenAI / Anthropic / ChatGPT / Microsoft / Slack / Teams / Notion 等不能直接画 logo；文字 chip 形式可以引用）
- ❌ **"取代 CEO / 替代决策"** —— Manager 受众不接受这种夸张话术
- ❌ **混淆受众** —— 不要把"教你写代码"卖给 manager，本课明确不写代码；不要把"省 10 小时"卖给 director，那是 ai-essentials 个人入门话术
- ❌ **空洞模版** —— 不要"在数字化转型大潮中..."、"全面构建 AI 思维..."；用具体模块名 / 具体框架名（ADKAR / AI Champion / Maturity Model / 2x2 Matrix）
- ❌ **真人脸正面**（任何 manager / office worker 真人）—— 一律 back-view / 三分之一侧背 / overhead 桌面构图
- ❌ **真实学员姓名 / FAQ 案例真实公司名**（FAQ 里的咨询公司 / 大厂案例不能直接放海报）
- ✅ **能用的真实承诺**："8 周输出方案" / "22 直播 + 39 Lab" / "Demo Day 毕业作品" / "ADKAR + Champion + Governance" / "OpenClaw 本地部署"

## taxonomy 扩展声明

**本 bootcamp 没有新增 taxonomy 字段。** 全部沿用 ai-essentials 已加进的：

- role: `pm-manager`（部门经理 / Director / Change Lead / IT Leader）
- role: `office-worker`（office worker 升级路径 → AI Champion）
- painPoint: `ai-fomo`（这里特指**组织级 FOMO**：公司在用 AI 但没人管 AI）
- painPoint: `ai-replace`（这里特指**会带团队用 AI 的 manager 替代不会的**）
- painPoint: `time-poverty`（这里特指**会议级时间黑洞**：无限讨论 AI 没产出）

> 同 role / 同 painPoint 在不同 bootcamp 的叙事完全不一样。ai-adoption 永远从"组织 / 团队 / 落地"切入，不从"个人 / 提效 / 求职"切入。
