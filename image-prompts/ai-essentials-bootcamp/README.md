## 与 ai-engineer 视角差异（同 role / 同 pain，叙事完全不同）

| Combo | ai-engineer 角度 | ai-essentials 角度 |
|-------|----------------|------------------|
| pm-manager × ai-replace/fomo | "57% 是 context 设计问题" + Context 金字塔（深度技术）| "团队都在用 AI，PM 还在写没人看的 PRD" + 4 行业 playbook（应用层）|

**绝对不在 ai-essentials 用以下 ai-engineer 锚点**：
- ❌ Context Engineering / RAG / Agent SDK / Eval（受众听不懂）
- ❌ 92% JD coverage / $150K 薪资数据（他们已就业，不为 JD 学）
- ❌ 7 production projects（他们不要 ship 项目）
- ❌ Production / Eval / 三 SDK 终端分屏（视觉脱节）

**ai-essentials 专属锚点**：
- ✅ 4 元素 prompt 公式卡片（角色 / 任务 / 格式 / 约束）
- ✅ 时钟 30 分钟红色楔形（time-poverty 视觉）
- ✅ 4 周进度环（绿/青/靛/橙 圆圈编号 1-4）
- ✅ 4 行业 playbook 2x2 网格（市场/财务/HR/销售）
- ✅ Office desk overhead（笔记本 + Excel 截图 + 便利贴）

## 用过的真实卖点（来自课程研究）

每张 prompt 至少用一个真实数据：

- **4 周 / 33 节课 / 17 个互动 Lab / ~10 小时总投入**
- **4 元素 Prompt 公式** Role / Task / Format / Constraint（贯穿全课）
- **3 大工作场景** 邮件 / 报告 / Excel（Phase 2 整周）
- **4 大行业 playbook** Marketing / Finance / HR / Sales（L31）
- **Phase 0 从零开始** L01: 怎么打开浏览器（不会用电脑也能学）
- **省 10 小时/月** 课程明文承诺，每天 30 分钟救出来

## 课程红线

写新 prompt 时必避：

- ❌ **承诺加薪 / 升职** —— 这是入门课，不承诺职业结果
- ❌ **"100% 学会"** —— 课程有 70% 测试解锁机制，不保证人人通过
- ❌ **"取代行业专家"** —— 受众是普通职场人，不卖"超越 PhD"叙事
- ❌ **真实公司 logo / 真实软件截图** —— 法律红线
- ✅ "省 10 小时/月" / "4 周入门" / "不写代码不调 API" —— 课程文档明文卖点
