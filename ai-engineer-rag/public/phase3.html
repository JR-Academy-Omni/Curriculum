Part 3 学完目标

Prompt Engineering 不是"会写中文就能写"。真实的工程化 Prompt 有明确的技巧决策树——什么时候用 Chain-of-Thought，什么时候用 Few-shot，什么时候让模型 Self-Consistency 采样多次投票。

Part 3 的高潮是 L27 Project 1：发布你的第一个 GPT。不写代码、不用 API，在 OpenAI GPT Builder 里配置 Instructions + Knowledge + Actions，发布到 GPT Store 并拿到真实用户反馈。

学完 Part 3 你应该能：

  • 熟练使用 CoT / Self-Consistency / ReAct / Tree-of-Thought 等进阶技巧
  • 在 Playground 里调 temperature / top_p / max_tokens 做快速迭代
  • 判断业务场景该用 Prompt / RAG / Fine-tune 中的哪一个（决策树）
  • ✅ 拿出一个真实发布、有用户反馈的 GPT 作品
🎯 Project 1 — 发布你的第一个 GPT

为什么从 GPT Store 开始？因为它是零代码 Prompt Engineering 的最佳试验场。你要面对的问题和企业 AI 产品经理一样：定义目标用户、写 Instructions、上传 Knowledge、调工具、发布、收集反馈。

🧩 4 个 Step
  1. 选定主题（从真实工作场景出发）
  2. 配置 Instructions（四要素框架）
  3. 上传 Knowledge（3-5 个相关文档）
  4. 发布 + 邀请 3 个真实用户试用
📦 交付物
  • ✅ GPT Store 分享链接
  • ✅ 3 份用户真实反馈（截图）
  • ✅ 一份迭代总结（改了哪些 Instructions）
