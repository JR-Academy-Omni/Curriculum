学习目标

  • 理解 Cursor、Claude Code、GitHub Copilot 等 AI 编程工具如何改变开发者日常工作
  • 掌握 Vibe Coding 概念，知道它对初级和高级开发者分别意味着什么
  • 判断哪些 IT 岗位在 AI 时代会增长、哪些会缩减
  • 建立每个 IT 从业者都需要的 AI 基础技能清单

核心概念

AI 编程工具三巨头：Cursor / Claude Code / GitHub Copilot

2025-2026 年，AI 编程工具已经从"尝鲜"变成了"标配"。简单说：

GitHub Copilot — 最早普及的 AI 编程助手。它直接嵌入 VS Code，你写一行注释，它自动补全代码。适合已经知道要写什么、只是懒得敲的情况。价格 $10/月（个人版）。

Cursor — 基于 VS Code 的独立 IDE，内置 AI 对话和代码编辑能力。你可以选中一段代码说"把这个改成 TypeScript"，它直接帮你改。它的 Composer 功能允许你用自然语言描述需求，AI 会跨文件修改代码。很多开发者已经从 VS Code 完全切换到 Cursor。

Claude Code — Anthropic 出的 CLI 工具，直接在终端里运行。你在项目目录里输入 claude，然后用自然语言描述你要做的事，它会读你的代码、理解项目结构、然后修改文件。和 Cursor 的区别是：Claude Code 更擅长理解大型项目的上下文（支持 200K token 上下文窗口），适合复杂的重构和跨模块修改。

三者的核心区别：

工具使用方式擅长场景月费
CopilotVS Code 插件，自动补全写新代码、补全函数$10
Cursor独立 IDE，对话 + 编辑修改现有代码、跨文件重构$20
Claude CodeCLI 终端工具大型项目理解、复杂重构按 API 用量

什么是 Vibe Coding

Vibe Coding 这个词是 Andrej Karpathy（前 Tesla AI 总监、OpenAI 联合创始人）在 2025 年初提出的。原话大意是："我不再真正写代码了，我只是描述我想要什么，AI 写代码，我看一眼觉得 vibe 对了就接受。"

Vibe Coding 的本质：你从"写代码的人"变成"指导 AI 写代码的人"。

这对不同级别的开发者意味着完全不同的事：

  • 初级开发者 — 能力被放大。以前需要 3 年经验才能独立完成的项目，现在用 AI 辅助可能 6 个月经验就能做。但风险是：如果你不理解 AI 生成的代码，出了 bug 你修不了。
  • 高级开发者 — 效率飞升。你知道要什么、知道怎么审查代码质量，AI 帮你省掉大量重复劳动。你从"写代码"变成"架构设计 + 代码审查 + AI 指导"。
  • 完全不会编程的人 — 可以做出原型了。用 Cursor 的 Composer 或 Claude Code，纯自然语言就能生成一个能跑的 Web 应用。但离生产环境部署还有很大差距。

哪些岗位在增长

基于 2025-2026 年的招聘数据和行业趋势：

明确增长的岗位：

  • AI/ML Engineer — 训练、部署、优化模型。需求量从 2024 到 2025 翻了近一倍。
  • Prompt Engineer / AI Application Developer — 把 AI 能力集成到产品里。虽然"Prompt Engineer"这个 title 在变少，但实际工作内容（构建 AI 工作流、RAG 系统、Agent）需求在爆发。
  • Platform Engineer / DevOps — AI 应用需要更复杂的基础设施（GPU 集群、模型部署、监控）。
  • Security Engineer — AI 引入了新的攻击面（prompt injection、数据泄露），安全需求增加。
  • Data Engineer — AI 需要高质量数据。数据管道、数据清洗、特征工程的需求持续增长。

明确缩减的岗位：

  • 纯手动测试（Manual QA） — AI 能写测试用例、能做回归测试。手动测试岗位在快速减少。
  • 初级前端切图 — AI 能直接从设计稿生成代码。纯 HTML/CSS 的工作价值在降低。
  • 基础运维 — 云平台 + AI 自动化运维，传统 SA（System Admin）的日常工作被大量替代。
  • 简单 CRUD 后端开发 — AI 能快速生成标准的增删改查代码，这类工作的"人工价值"在下降。

关键结论：不是岗位消失，是岗位的"门槛技能"在变。 以前你会写 React 就能找到工作，现在你需要会写 React + 会用 AI 高效写 React + 理解 AI 工具的局限性。

每个 IT 人都需要的 AI 技能

不管你是前端、后端、测试、运维还是项目经理，以下技能在 2026 年是必需的：

  1. Prompt Engineering 基础 — 知道怎么给 AI 写清晰的指令，理解 system prompt、few-shot、chain-of-thought 这些概念。
  2. AI 编程工具使用 — 至少熟练使用 Copilot 或 Cursor 其中之一。能用 AI 加速日常编码。
  3. AI 输出审查能力 — AI 会"一本正经地胡说八道"（hallucination）。你需要能判断 AI 生成的代码/方案是否正确。
  4. AI 工作流设计 — 知道什么任务适合用 AI、什么不适合。能设计"人 + AI"的协作流程。
  5. 基本的 LLM 理解 — 不需要会训练模型，但要理解 token、上下文窗口、temperature 这些基本概念，这样才能有效使用 AI 工具。

操作步骤

用 AI 工具加速你的日常工作（3 个实操场景）

场景 1：用 Cursor 快速理解一个陌生代码库

你刚入职，面对一个 50 万行代码的项目，不知道从哪里开始。

打开 Cursor，按 Cmd+L 打开对话框，输入：

Prompt: "这个项目的架构是什么？主要的入口文件在哪里？用户登录的流程经过哪些文件？请用简单的中文解释。"

Cursor 会扫描你的项目结构，给出架构概览。然后你可以追问：

Prompt: "用户从点击登录按钮到看到 dashboard，代码执行路径是什么？列出经过的文件和函数名。"

场景 2：用 Claude Code 做跨文件重构

你需要把项目里所有用到 moment.js 的地方改成 dayjs。

在项目根目录运行 claude，输入：

Prompt: "帮我把这个项目里所有 import moment from 'moment' 改成 import dayjs from 'dayjs'，同时把所有 moment() 调用改成 dayjs() 调用。注意 moment.format() 和 dayjs.format() 的 API 差异，特别是 moment().startOf('week') 在 dayjs 需要额外插件。列出所有需要修改的文件，然后逐个修改。"

场景 3：用 Copilot 加速单元测试编写

在 VS Code 里打开你的测试文件，写一行注释：

在代码中输入注释:
// Test: should return 401 when token is expired

Copilot 会自动生成完整的测试用例。你需要审查：mock 数据是否合理、断言是否完整、edge case 是否覆盖。

评估你当前岗位的 AI 影响

用这个框架评估：

Prompt（给 ChatGPT 或 Claude）:

"我目前的岗位是 [你的岗位，如 Frontend Developer]，主要工作内容包括：
1. [日常任务1，如：根据 Figma 设计稿写 React 组件]
2. [日常任务2，如：修复线上 bug]
3. [日常任务3，如：参加需求评审会]
4. [日常任务4，如：写 API 集成代码]

请分析每项任务被 AI 替代的可能性（高/中/低），以及我应该如何调整技能方向来保持竞争力。给出具体的学习建议。"

动手练习

  1. 工具体验（30 分钟） — 如果你还没用过 Cursor，去 cursor.sh 下载安装。打开一个你自己的项目，用 Cmd+L 对话功能问它 3 个关于你代码的问题。感受一下 AI 理解代码上下文的能力。
  2. 岗位影响分析（20 分钟） — 用上面的 Prompt 模板，分析你当前（或目标）岗位受 AI 影响的程度。把结果保存下来，作为你学习规划的参考。
  3. AI 技能自评（10 分钟） — 对照"每个 IT 人都需要的 AI 技能"清单，给自己打分（1-5 分）。找出最薄弱的 1-2 项，制定下周的学习计划。

常见问题

Q: AI 会不会导致程序员大面积失业？

短期内不会。AI 目前擅长的是"按指令生成代码"，但不擅长理解业务需求、做架构决策、处理复杂的跨团队协作。实际发生的是：用 AI 的程序员替代不用 AI 的程序员。一个会用 AI 工具的开发者，产出可能是不用 AI 的 3-5 倍。公司会倾向于雇佣更少但更高效的开发者。

Q: 我是转行新手，应该先学编程还是先学 AI 工具？

两个同时学。但基础编程知识是必须的 — 你至少要能看懂 AI 生成的代码、能判断它是否正确。建议路径：先学 3 个月基础（HTML/CSS/JS 或 Python），同时就开始用 Copilot 辅助编码。不要等"学完了再用 AI"，那样你会浪费很多时间。

Q: Cursor 和 Claude Code 我该学哪个？

如果你是做 Web 开发的，先学 Cursor，因为它有图形界面、上手更快。如果你习惯终端操作、或者经常需要处理大型项目的重构，Claude Code 更强。两个都免费试用，花半天时间各试一下，选适合你的。

小结

AI 编程工具（Cursor、Claude Code、Copilot）已经从可选变成必需。Vibe Coding 不是让你不学编程，而是改变了编程的方式 — 你需要更强的架构思维和 AI 输出审查能力。IT 行业没有在"消失岗位"，而是在重新定义每个岗位需要的技能组合。现在开始学习和使用 AI 工具，是保持竞争力的最低要求。
