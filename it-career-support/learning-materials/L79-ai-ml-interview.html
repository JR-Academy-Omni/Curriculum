核心概念

AI/ML 面试的三大支柱

不管是 Google Brain 还是澳洲本地 AI 创业公司，AI/ML 面试考的核心能力可以归为三类：

  1. ML Fundamentals（机器学习基础） — 50% 的面试都会考
  2. Deep Learning（深度学习） — Senior 级别必考，Junior 可能只问概念
  3. ML Engineering（工程实践） — 越来越重要，特别是 MLE 岗位

支柱一：ML Fundamentals

Supervised vs Unsupervised Learning

面试不是考你背定义，而是考你能不能判断什么场景用哪个。

  • Supervised：有标签数据。你知道正确答案是什么，让模型去拟合。典型：邮件分类（spam/not spam），房价预测。
  • Unsupervised：没有标签。让模型自己找 pattern。典型：客户分群（K-means clustering），异常检测。
  • 面试追问点："如果你有少量标签数据怎么办？" — 答 semi-supervised learning 或 self-supervised learning。

Overfitting vs Underfitting

这个问题 90% 的 ML 面试会考，但大多数人只能背出定义。面试官想听的是你怎么诊断和解决。

  • 诊断方法：看 training loss vs validation loss 的曲线。training loss 低但 validation loss 高 = overfitting；两个都高 = underfitting。
  • Overfitting 解决方案（按优先级）：
    1. 加数据（最有效但最贵）
    2. Data augmentation（图片翻转旋转、文本同义词替换）
    3. Regularization（L1/L2、Dropout）
    4. 减小模型复杂度（减少层数/参数）
    5. Early stopping
  • Underfitting 解决方案：增加模型复杂度、增加训练时间、减少 regularization。

Cross-Validation

面试常问："为什么不直接用 train/test split？" 答案：单次 split 的结果取决于你怎么分数据，可能碰巧分到一个"简单"的 test set。K-fold cross-validation 把数据分成 K 份，每份轮流做 test set，最后取平均。这样的评估更稳定。

实际面试中，面试官可能追问："K 一般取多少？" — 通常 5 或 10。数据量很小时可以用 Leave-One-Out（K = N）。

Bias-Variance Trade-off

不要背公式，用一个例子说清楚：

  • High bias（高偏差）：模型太简单，假设太强。比如用线性模型拟合明显非线性的数据。
  • High variance（高方差）：模型太复杂，对训练数据的噪声过于敏感。换一批训练数据，预测结果变化很大。
  • Trade-off：增加模型复杂度 → bias 降低但 variance 升高。关键是找到 sweet spot。

支柱二：Deep Learning

Transformer 和 Attention 机制

2024-2026 年 AI 面试绕不开 Transformer。你不需要能手写 multi-head attention，但要能解释清楚核心思想：

  • 为什么需要 Attention：传统 RNN/LSTM 处理长序列时会丢失早期信息（vanishing gradient）。Attention 让模型直接"看到"序列中任意位置的信息。
  • Self-Attention 怎么工作：每个 token 生成 Q（Query）、K（Key）、V（Value）三个向量。Q 和所有 K 做点积 → softmax → 加权求和 V。结果是每个 token 都包含了对其他所有 token 的"注意力"信息。
  • Multi-Head：做多组独立的 attention，让模型从不同角度理解关系（比如一组关注语法结构，一组关注语义）。
  • 面试常见追问："Transformer 的时间复杂度是什么？" — O(n²d)，n 是序列长度，d 是维度。这也是为什么 context window 有限制。

CNN vs RNN vs Transformer 的使用场景

架构最适合2025 年还用吗？
CNN图像处理、固定大小输入是，Computer Vision 仍然是主力（ResNet, EfficientNet）
RNN/LSTM序列数据、时间序列大部分被 Transformer 替代，少数场景（实时流数据）还在用
TransformerNLP、多模态、几乎所有需要理解上下文的任务绝对主力，GPT/BERT/LLaMA 都是 Transformer

Fine-tuning vs RAG

这是 2025 年 AI Engineer 面试必问题：

  • Fine-tuning：修改模型参数，让模型学会新知识。适合需要改变模型行为的场景（比如让模型学会特定的写作风格）。成本高，需要 GPU + 标注数据。
  • RAG（Retrieval-Augmented Generation）：不改模型参数，而是把相关文档检索出来塞进 prompt。适合知识库问答、企业文档搜索。成本低，更新快（改文档就行）。
  • 面试答题框架："如果知识更新频繁且格式固定，选 RAG；如果需要模型真正理解某个领域的推理模式，选 Fine-tuning；大多数场景先试 RAG，效果不够再考虑 Fine-tuning。"

支柱三：ML Engineering

Model Deployment

面试官不只关心你能不能训练模型，更关心你能不能把模型部署到生产环境。常见问题："How would you deploy a model to serve 10,000 requests per second?"

答题框架：

  1. Model Serving：把模型包装成 API（FastAPI/Flask + Docker），或用 TorchServe/TF Serving
  2. Scaling：水平扩展（多个 pod），负载均衡，auto-scaling based on GPU utilization
  3. Latency 优化：模型量化（FP32 → FP16/INT8）、模型剪枝、batch inference、缓存热门查询
  4. Monitoring：data drift detection（输入数据分布变了吗？）、model performance decay（准确率在下降吗？）

MLOps

MLOps = ML + DevOps。面试中提到以下关键词能展示你的工程成熟度：

  • Feature Store：统一管理特征，避免 training 和 serving 用不同的特征计算逻辑（training-serving skew）
  • Model Registry：管理模型版本，A/B test 不同版本。比如 MLflow Model Registry
  • Pipeline Orchestration：Airflow/Kubeflow，自动化 data → training → evaluation → deployment
  • Experiment Tracking：MLflow/W&B，记录每次实验的超参数、metrics、artifacts

RAG Architecture（2025 热门）

很多公司正在构建 RAG 系统，面试高频考点：

  1. Indexing：文档分块（chunking strategy）→ embedding → 存入向量数据库（Pinecone/Weaviate/pgvector）
  2. Retrieval：用户 query → embedding → 向量相似度搜索 → top-K 文档
  3. Generation：retrieved documents + user query → LLM → 回答
  4. 常见优化：hybrid search（向量 + keyword）、reranking（Cohere Rerank）、chunk overlap、metadata filtering

10 个高频面试问题及回答框架

  1. "Explain the difference between L1 and L2 regularization."
    L1（Lasso）→ 权重稀疏（部分变为 0）→ 自动特征选择。L2（Ridge）→ 权重变小但不为 0 → 防止任何单个特征权重过大。用哪个？特征很多且怀疑有不相关特征 → L1；所有特征都可能相关 → L2。
  2. "How do you handle imbalanced datasets?"
    SMOTE（oversampling 少数类）、undersampling 多数类、class weight adjustment、focal loss。面试追问："哪个最好？" — 没有最好的，先试 class weight（最简单），效果不够再用 SMOTE。
  3. "What's the vanishing gradient problem?"
    深层网络反向传播时，梯度经过多层连乘越来越小，底层参数几乎不更新。解法：ReLU 替代 sigmoid、Batch Normalization、Residual Connection（ResNet 的核心思想）。
  4. "Explain batch normalization."
    每一层的输入做标准化（mean=0, std=1），然后用可学习参数 γ 和 β 做 scale 和 shift。效果：训练更稳定、可以用更大 learning rate、有轻微 regularization 效果。
  5. "What metrics would you use for a classification problem?"
    Accuracy（数据平衡时）、Precision/Recall/F1（数据不平衡时）、AUC-ROC（需要评估不同 threshold 下的表现）。关键追问："什么时候 precision 比 recall 重要？" — 误报成本高时（比如 spam 检测，宁漏不错）。
  6. "How does a random forest work?"
    多棵决策树投票。每棵树用 bootstrap sample（有放回采样）训练，每次分裂只看随机子集的特征。这叫 bagging + feature randomness。效果：降低 variance，不容易 overfitting。
  7. "Explain the attention mechanism in transformers."
    （见上面 Deep Learning 部分的详细解释）
  8. "When would you use RAG vs fine-tuning?"
    （见上面 Fine-tuning vs RAG 部分）
  9. "How do you detect and handle data drift?"
    统计检验（KS test, PSI）监测输入数据分布变化。处理：轻微 drift → 重新训练；严重 drift → 检查数据源是否出了问题。生产中用 Evidently AI 或 Whylabs 做自动化监控。
  10. "Design a recommendation system."
    分两阶段：Candidate Generation（粗筛，协同过滤或 embedding-based retrieval 快速缩小范围到几百个）→ Ranking（精排，用 deep model 对候选项打分排序）。工程角度：冷启动问题用 content-based 兜底，热门物品用 popularity bias correction。

常见问题

Q：我是前端/后端开发者，想转 AI 方向，面试准备需要多久？

如果你有编程基础但 ML 知识接近零，系统准备需要 3-6 个月。路线：先花 1 个月学 ML 基础（Andrew Ng 的 Machine Learning Specialization），再花 1 个月学 Deep Learning（特别是 Transformer），最后 1-2 个月学 LLM 应用开发（RAG、prompt engineering）。如果你目标是 AI Engineer 而不是 ML Researcher，工程能力的权重比数学推导高。

Q：面试时被问到不会的 ML 概念怎么办？

说实话："I haven't worked with X directly, but based on my understanding of Y (相关概念), I think X works like this..." 然后给出你的推理。面试官考察的不只是你知不知道答案，还有你的推理能力。瞎编一个答案比承认不知道更扣分。

Q：澳洲本地 AI 岗位多吗？面试难度和 FAANG 比怎么样？

澳洲 AI 岗位集中在几个领域：银行（CBA, ANZ 的 fraud detection、risk modeling）、医疗（medical imaging）、mining tech、和 AI 创业公司。面试难度比 FAANG 低 1-2 个 level — 通常不会让你手推反向传播公式，但 ML 基础概念和工程实践是必考的。Canva、Atlassian 的 ML 岗位难度接近 FAANG。

小结

AI/ML 面试覆盖面广，但核心就是三件事：ML 基础概念能不能讲清楚（不是背定义，是能举例子说 trade-off）、Deep Learning 特别是 Transformer 理不理解、模型能不能部署到生产环境。10 个高频问题先过一遍，发现哪个答不上来就集中补。2025 年的 AI 面试越来越偏工程（RAG、MLOps），纯理论的考察在减少。
