学习目标

  • 理解 AI 对 IT 岗位的真实影响 — 哪些技能在贬值，哪些在增值
  • 掌握 2026 年每个 IT 从业者必须具备的 3 项 AI 技能
  • 学会构建一个"AI 作品集"来展示你的 AI 能力
  • 找到适合自己当前阶段的 AI 学习路径

核心概念

AI 不会取代你，但会用 AI 的人会

这句话已经被说烂了，但大部分人理解错了。重点不是"AI 会不会取代程序员"，而是：同样的岗位，会用 AI 的人产出是不会用的人的 3-5 倍。公司不会因为有了 AI 就少招人，但会优先招能用 AI 放大产出的人。

具体来说：

  • 2024 年：会用 Copilot 是加分项
  • 2025 年：会用 AI 写代码是基本要求
  • 2026 年：不会用 AI 的人简历直接被筛掉

不是危言耸听。看一下现在 Seek 和 LinkedIn 上的 JD，越来越多出现这样的要求：

  • "Experience with AI-assisted development tools (Copilot, Cursor)"
  • "Ability to leverage LLMs for productivity"
  • "Familiarity with prompt engineering"

2026 年必须掌握的 3 项 AI 技能

技能 1：Prompt Engineering

Prompt Engineering 不是"跟 ChatGPT 聊天"。它是一种系统化地从 AI 获取高质量输出的能力。

核心技术：

  • Zero-shot vs Few-shot — 给不给例子，输出质量差别巨大
  • Chain of Thought (CoT) — 让 AI 分步思考而不是直接给答案
  • Role + Context + Task + Format — 四要素框架，写好 prompt 的公式
  • Retrieval Augmented Generation (RAG) — 让 AI 基于你提供的文档回答，而不是凭训练数据编

举个实际例子。你要让 AI 帮你写一个 API endpoint：

差的 prompt：

帮我写一个用户注册 API

好的 prompt：

你是一个 NestJS 后端工程师。项目使用 NestJS + TypeScript + MongoDB (Mongoose)。

请帮我写一个用户注册的 POST /auth/register endpoint，要求：
- DTO 校验：email (必填, 格式校验), password (必填, 最少 8 位), name (必填)
- 密码用 bcrypt hash，salt rounds = 10
- 如果 email 已存在返回 409
- 成功返回 201 + user object（不包含 password）
- 包含完整的 controller, service, dto 文件

代码风格：使用 class-validator 做 DTO 校验，service 里做业务逻辑，controller 只做路由。

第二个 prompt 给的输出直接能用，第一个给的可能是 Express 代码、可能缺校验、可能返回密码。

技能 2：AI 辅助编程（Cursor / Copilot）

AI 编程工具不是玩具，是生产力工具。2026 年的主流工具：

工具 适用场景 月费 特点
GitHub Copilot 日常编码补全 $10-19/月 最成熟，VS Code 深度集成
Cursor 整个项目级别的 AI 编程 $20/月 可以读整个 codebase，重构、新增 feature 特别强
Claude Code 终端里的 AI 编程 agent 按 token 计费 可以自主执行命令、读写文件、跑测试，自动化程度最高

实际工作中的用法：

  • 写新功能 — 在 Cursor 里描述需求，AI 生成代码框架，你 review 和调整
  • 写测试 — 给 AI 你的 service 代码，让它生成 unit test，覆盖率一下从 0 到 80%
  • Debug — 把报错信息 + 相关代码丢给 AI，它能定位大部分问题
  • 重构 — "把这个 500 行的 service 拆成 3 个小 service" — Cursor 可以一次性搞定
  • 学习新技术 — 边用 AI 写代码边学，比看文档快 5 倍

技能 3：AI 辅助决策

这个技能不只是给技术人员的。AI 辅助决策 = 用 AI 处理信息、分析数据、辅助判断。

实际应用场景：

  • 技术选型 — 把几个方案的 pros/cons 喂给 AI，让它做结构化对比
  • Incident 分析 — 把 error log 扔给 AI，让它分析 root cause
  • Architecture Review — 描述你的系统架构，让 AI 找出潜在问题
  • 估时 — 描述一个 feature，让 AI 帮你拆 task 和估工时

Prompt 示例 — 技术选型辅助：

我们团队在选消息队列。候选方案：RabbitMQ、Kafka、AWS SQS。

使用场景：
- 每天约 50 万条消息
- 需要保证消息不丢失
- 团队 5 个人，DevOps 能力一般
- 已经在用 AWS，不想引入太多新基础设施

请对比三个方案的：运维复杂度、成本、吞吐量、消息可靠性、团队学习曲线。给出你的推荐。

如何构建 AI 作品集

光说"我会用 AI"没用，要有证据。以下是你可以在简历和 LinkedIn 上展示的东西：

作品类型 展示什么 怎么做
AI 辅助项目 你用 AI 工具完成的完整项目 在 GitHub README 里写清用了什么 AI 工具、怎么用的
Prompt 模板库 你积累的高质量 prompt 集 GitHub repo 或 Notion page，按场景分类
效率对比 AI 前后的效率差异 写一篇博客，展示用 AI 把某个任务从 4 小时缩短到 30 分钟
AI 集成经验 在真实项目中集成 AI 功能 比如给一个 app 加了 AI chat、自动摘要、智能搜索

LinkedIn 上怎么体现：

  • Skills 里加上：Prompt Engineering, AI-Assisted Development, LLM Integration
  • Experience 里加具体描述："Leveraged Cursor AI to increase development velocity by 40% on [project]"
  • 发 1-2 篇 post 分享你用 AI 的经验（不用长，3-5 段就行）

学习路径建议

根据你当前的阶段选择起点：

阶段 行动 平台资源
完全没用过 AI 先花 2 小时用 ChatGPT 完成一个日常任务（写邮件、整理笔记、代码调试） Prompt 实验室（hello-ai 模块）
用过 ChatGPT 但不系统 学习 Prompt Engineering 四要素框架，练习 10 个不同场景 Learn: Prompt 大师
会用 ChatGPT 但没用过编程工具 装 Cursor，用一个下午把一个 side project 的功能用 Cursor 完成 Learn: Vibe Coding Hub, Wiki: cursor
已经在用 Copilot/Cursor 开始学 Claude Code（终端 agent），尝试用 AI 做更复杂的任务（重构、架构设计） Wiki: claude-code-guide

操作步骤

Step 1：评估你的 AI 技能现状

用这个自评表打分（1-5）：

  • 我能写出结构化的 prompt 获取高质量输出：___/5
  • 我每天使用 AI 编程工具（Copilot/Cursor）：___/5
  • 我能用 AI 辅助做技术决策和分析：___/5
  • 我在简历/LinkedIn 上有 AI 相关经验展示：___/5
  • 我能向非技术人员解释 AI 的能力和局限：___/5

总分低于 15 分 → 需要立刻开始系统学习。

Step 2：选一个工具深入使用

不要同时学所有工具。先选一个：

  • 非技术背景 → 先把 ChatGPT/Claude 用到极致
  • 开发者 → 直接上 Cursor，日常开发全部切过去
  • 想走 AI 方向 → 学 Claude Code + LangChain，做一个 AI 项目

Step 3：建立你的第一个 AI 作品

Prompt — 快速启动 AI Side Project：

我是一个 [你的技术栈] 开发者，想做一个 side project 展示我的 AI 能力。要求：
- 用到 LLM API（OpenAI 或 Claude）
- 有实际用途（不是 toy project）
- 一个周末能完成 MVP
- 能部署到 Vercel/Netlify 让人访问

请给我 5 个项目创意，每个包含：项目名、一句话描述、技术栈、核心功能、周末能完成的 MVP scope。

动手练习

练习 1：Prompt Engineering 实战

去 JR 平台的 Prompt 实验室，完成至少 3 个练习场景。记录你的 prompt 和 AI 的输出，思考如何改进。

练习 2：用 AI 完成一个真实工作任务

选一个你这周要做的工作任务（写代码、写文档、分析数据），全程用 AI 辅助完成。记录：

  • 没有 AI 你预估要多久？
  • 用 AI 实际花了多久？
  • AI 在哪些环节最有帮助？
  • AI 在哪些环节不靠谱，需要你人工修正？

练习 3：更新你的 LinkedIn

在 LinkedIn 的 Skills 和 Experience 里加入 AI 相关内容。至少：

  • 加 3 个 AI 相关 skill
  • 在最近一份工作经历里加一条 AI 相关描述
  • 发一篇关于你用 AI 的经验分享 post

常见问题

Q: 我不是开发者，AI 编程工具跟我没关系吧？

Prompt Engineering 和 AI 辅助决策跟岗位无关。BA 用 AI 写 user story，QA 用 AI 生成 test case，PM 用 AI 做竞品分析，DevOps 用 AI 写 IaC。每个 IT 岗位都有 AI 能帮忙的地方。Cursor 确实主要给开发者用，但 ChatGPT/Claude 是所有人都应该掌握的。

Q: AI 工具这么多，学不过来怎么办？

不要追工具，追能力。Prompt Engineering 是底层能力，ChatGPT 换成 Claude 换成 Gemini 都用得上。AI 编程工具今天是 Copilot 明天可能是别的，但"指导 AI 写代码"这个能力不变。选一个工具用深，比浅尝辄止 10 个工具有用得多。

Q: 面试的时候怎么展示 AI 能力？

不用等面试官问。在 behavioral interview 的回答里自然带入："In this project, I used Cursor to prototype the feature in 2 hours instead of the usual 2 days, which gave us time to do more thorough testing." 展示的是结果和思维方式，不是工具本身。

小结

2026 年 AI 能力已经不是"nice to have"，是 IT 从业者的基础技能。三个方向：Prompt Engineering（所有人必学）、AI 辅助编程（开发者必学）、AI 辅助决策（所有人必学）。不需要同时学所有工具，选一个深入使用，积累作品集，在简历和 LinkedIn 上展示。记住，面试官不关心你用什么工具，他们关心的是你能不能用 AI 把活干得更快更好。
