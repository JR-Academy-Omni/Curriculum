## 3. 现场翻车点（4 个）

### 3.1 开场延误 ~1 小时
场地对接人员迟到，全场枯等近 1 小时才开始。直接打乱节奏、消耗参与者耐心。

### 3.2 听众水平分层严重，内容「一刀切」失败
来的人 AI 基础和职级差异大：
- **Junior** 觉得学得不错；
- **一半以上**觉得太浅、意义不大，尤其是 **FC（Financial Controller）/ FA（Financial Analyst）** 这类资深人群。

→ 单一难度 + 单一场景，无法同时满足小白和资深财务。

### 3.3 讲师不专业、缺教学方法论
- 讲师**非财务背景**，内容不够落地；
- 没有教学法，**一上来就 case study**；
- 没有先讲清 **Claude Cowork 和普通 Claude 的区别**；
- 没有介绍 **Skills**。

### 3.4 发票素材完全不合规
实操用的 10 张发票是 AI 生成的，**完全不符合财务要求和规则**，内容和质量都很差。

> 核查确认（看了 `invoices/invoice_01.png`）：发票只有「Bill To 客户名 + 金额 + GST + Total」，**缺供应商名称、缺 ABN、无「Tax Invoice」字样、无费用明细行**，GST 仅按金额简单 ×10%。对会计而言这根本不算合规的澳洲 tax invoice——资深财务一眼看穿，专业感崩塌。

## 4. 标杆参照：协会之前那场 1 小时 Webinar（反馈极好）

协会反馈：之前办过一次 1 小时线上 webinar，**反馈特别好**，关键是**由浅入深、分层**——先介绍 Claude 界面/功能，有 layer 设计，**不同程度和 level 都能学到东西**。

那场的议程（值得借鉴的结构）：

**一、Microsoft Copilot 提效实战（15 分钟）**
- Word / Outlook / Excel / Teams 日常应用 + Copilot Studio 自定义 Agent 搭建
- 演示：解析 ATO Ruling 生成初稿 · 现场搭建税务政策问答 Agent

**二、Claude Dispatch — 随时随地的 AI 协作（8 分钟）**
- 让财务人摆脱办公室限制（出差/客户现场/居家均可实时协作）
- 重点：实时文件标注——AI 直接在文档上批注、修订、高亮关键条款，边看边改
- 演示：外出场景下用 Dispatch 对税务协议实时标注审阅

**三、Vibe Coding — 财务人自建 AI 工具（15 分钟）**
- 「为什么懂业务比懂代码更重要」；Division 7A 计算器、转让定价筛选工具等案例
- 演示：用 Claude Code 零代码现场搭建澳洲 GST 计算器（约 8 分钟）

**Q&A（4 分钟）**

> 关键启示：**分层 + 由浅入深 + 具体到澳洲税务场景（ATO Ruling、Division 7A、GST、转让定价）**，工具讲解（界面/功能/区别）放在前面。这正是本次线下场缺的。

## 5. 协会电话后的真实需求（6/22 通话）

### 5.1 真正的痛点 / 需求场景
1. **Close the month（月结）**
   - 小公司：需求不大，量小；
   - 大公司：已有系统能处理；**high volume 场景 AI 才有用**，比如折旧（depreciation）等。
2. **Report 自动化**
3. **Technical & Admin 自动化**：如自动分类 email——**任何级别都需要**。

### 5.2 三类目标群体（都会涉及）
- 事务所会计
- 公司会计
- 事务所 owner

### 5.3 对本次 workshop 内容的具体反馈
- **invoice 这一趴可以直接 PASS**（不合规 + 价值低）；
- **report 有用**，但有些人已经在用，**内容偏基础**。

## 8. 经验教训（一句话）

> 给资深财务（FC/FA）办的课，**专业度是底线**——讲师背景、素材合规、内容分层缺一不可；AI 工具课要先讲清「这工具是什么、和你已知的有什么不同」，再上 case study。单一难度 + 不合规素材 + 非专业讲师 = 资深人群当场流失。
