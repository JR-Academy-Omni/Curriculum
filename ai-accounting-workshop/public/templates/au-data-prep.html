1FY 澳洲财年 — July 1 → June 30 FINANCIAL YEAR

🚨 最高频踩坑：你粘了 12 个月 P&L 给 Cowork，没说清是 FY 还是 calendar year，Cowork 默认按自然年切 — 你 Q3 涨了 30% 它说是『年初到现在』，但你单位口径是『FY Q3 = 1-3 月，是上半财年的第 3 季度』。所有跨期分析必须开篇说明 FY vs calendar year 口径。

实操：在 prompt 里第一行就写清楚：

PROMPT 增项 口径说明：本数据使用澳洲财年（FY），FY2025 = 2024-07 至 2025-06。所有"Q1/Q2/Q3/Q4"指 FY 季度（7-9 / 10-12 / 1-3 / 4-6 月）。所有"年度同比"按 FY 同比，不要按自然年同比。
2GST 字段 — 10% / GST-free / Input-taxed 三类 GST TREATMENT

澳洲 GST 不是单一 10% — 一笔交易可能是 三种 GST treatment 之一，Cowork 看不见 treatment 标签会算错应缴 GST 和 BAS 上的 G1/G3/G10 等格子。

Treatment含 GST 吗典型场景列名建议
Taxable+10%大部分销售 / 采购GST_treatment=taxable
GST-free0%新鲜食品 / 教育 / 部分医疗 / 出口GST_treatment=gst_free
Input-taxed不抵扣住宅租金 / 金融服务GST_treatment=input_taxed

CSV 列怎么准备：每行至少要有 amount_ex_gst（不含税）+ gst_amount（GST 金额，可以是 0）+ gst_treatment（三选一标签）三列。如果只有总额 amount_inc_gst，Cowork 会要你回头去算，浪费 token。

实操简化：大部分 SME 单位日常分析不需要那么细，只要把 amount（总额含 GST）+ gst_amount（GST 部分）两列出齐，Cowork 就能算准毛利、归因费用、判断净额走势。treatment 三类是给 BAS 报送用的，月度损益分析不一定需要。
PROMPT 增项 GST 口径：所有 amount 列已是含 GST 总额（GST-inclusive）。GST 部分在 gst_amount 列单独标出。做毛利分析时用不含税金额（amount - gst_amount）；做现金流分析时用含税总额。不要把不同 GST treatment 的行加总（taxable + gst-free 加起来没有意义）。
3BAS / IAS 报送周期 REPORTING PERIOD

澳洲单位按规模走不同的 BAS / IAS 报送周期。Cowork 做"本期 vs 上期"分析必须知道你按哪种周期切：

  • 月度 BAS（年营业额 ≥ $20M）：每月 21 号前报
  • 季度 BAS（绝大多数 SME）：FY Q1 (10-28) / Q2 (2-28) / Q3 (4-28) / Q4 (7-28)
  • 年度 IAS（小规模注册 GST 但 turnover < $75K）：年度报送
怎么影响数据准备：① 时间序列分析要按你的报送周期切（月度报送 → 12 期对比；季度报送 → 4 期对比 + FY 同期对比）。② 异常归因要参考报送日期 — 比如『为什么 4 月费用突增』可能是 Q3 BAS 在 4 月 28 日前要交，做了一些 timing 调整。在 prompt 里告诉 Cowork 你的报送周期它会少瞎猜。
PROMPT 增项 报送口径：本单位按季度 BAS报送（FY Q1 = 7-9 / Q2 = 10-12 / Q3 = 1-3 / Q4 = 4-6），所有跨期对比按 BAS 季度。异常归因时请考虑报送时点（4 月 / 7 月 / 10 月 / 2 月底前后）可能存在的 timing 调整。
4Cowork 读得懂的标准字段表 CSV SCHEMA

下面这张表是 Cowork 跑 AU 会计 prompt 时最稳的列结构。从 Xero/MYOB/Quickbooks 导出后做一次列名映射，对齐到这个 schema 再放进 Filesystem 目录。

字段名必/选类型含义 / 例子
dateREQYYYY-MM-DD交易日期。如 2025-04-15。Cowork 会按这个切 FY/季度/月度
fy_yearRECinteger财年标签，如 2025 表示 FY2025（即 2024-07 至 2025-06）。算了你也可以让 Cowork 自己从 date 推
account_codeREQstring会计科目代码。如 4000 = Sales Revenue / 6300 = Marketing。每个单位 chart of accounts 不同，Xero/MYOB 导出自带
account_nameREQstring科目名称。如 Subscription Revenue / Marketing - Online
categoryREQenum大类：Revenue / COGS / OpEx / Other。Cowork 用这个做毛利率拆分
amountREQdecimal含 GST 总额（AUD）。正数 = 收入或费用本身，负数 = 退款/冲销
gst_amountREQdecimalGST 部分（AUD）。可以是 0（GST-free）或 amount/11（taxable，10% on ex-GST = 1/11 of inc-GST）
gst_treatmentRECenumtaxable / gst_free / input_taxed。BAS 分析需要，月度分析可省
currencyRECstring默认 AUD。多币种单位才需要这列 + amount_aud 折算列
contactRECstring客户/供应商脱敏后编号。如 客户 12 / 供应商 V3。禁真名/真公司名
descriptionOPTstring交易备注（脱敏后）。Cowork 用于异常归因时找线索
tracking_categoryOPTstringXero 的 Tracking / MYOB 的 Job Code。如部门 / 项目 / 渠道。多维度分析用
notesOPTstring你自己的口径备注（如『6 月含一次性赞助』）— 帮 Cowork 不瞎猜

禁止出现在 CSV 里的列（脱敏阶段就要删）：

❌ employee_name · tfn（Tax File Number）· bank_bsb · bank_account · abn（除非已脱敏成假码）· email · phone · address · 未公开并购/估值/上市相关

建议的最简列集（先跑通分析够用了）：date, account_code, account_name, category, amount, gst_amount, contact — 7 列起步，缺什么再加。

5从 Xero / MYOB / Quickbooks Australia 导出 EXPORT GUIDE

三大主流 AU 会计系统的导出路径不同。下面是 "导出 → 列重命名 → 放进 Cowork Filesystem" 三步走：

XERO

Xero

  1. Reports → Account Transactions / Profit & Loss
  2. 选时间范围（按 FY 季度）
  3. 右上 Export → CSV
  4. 列名对照：
    Date → date
    Account Code → account_code
    Description → description
    Reference → contact（脱敏后）
    Net → amount_ex_gst
    Tax → gst_amount
    Gross → amount
  5. 另存到 ~/Documents/AI-Workshop/
MYOB

MYOB AccountRight

  1. Reports → Index to Reports → Accounts → General Ledger Detail
  2. Customise → 时间范围 + Account categories
  3. Send To → Excel → 另存为 CSV
  4. 列名对照：
    Date → date
    Account No. → account_code
    Account Name → account_name
    Memo → description
    Amount → amount
    Tax Amount → gst_amount
    Tax Code → gst_treatment
  5. 另存到 ~/Documents/AI-Workshop/
QUICKBOOKS

Quickbooks Online (AU)

  1. Reports → Profit and Loss Detail / Transaction Detail by Account
  2. 选 FY 季度时间范围 + Customise → Show GST columns
  3. Export → Export to Excel → 另存 CSV
  4. 列名对照：
    Date → date
    Account → account_name
    Memo/Description → description
    Customer/Vendor → contact（脱敏后）
    Amount → amount
    Tax Amount → gst_amount
    Tax Type → gst_treatment
  5. 另存到 ~/Documents/AI-Workshop/
简化做法：不想手工映射列名？让 Cowork 自己映射。把原始导出 CSV 放进 Filesystem，然后开 prompt：『读这份 CSV，按 第 4 节 schema 的列名映射到一份新文件 ~/Documents/AI-Workshop/standardized.csv』 — Cowork 一次跑完，下次就用 standardized.csv。
6脱敏要素 — 哪些必删、哪些可保留 DATA SANITIZATION

Cowork 通过 Filesystem Connector 读你电脑上的 CSV — 数据本身在你电脑没"上云"，但 prompt 出去时 Cowork 会读这些字段。所以放进 Filesystem 之前先脱敏是最稳做法。

❌ 必删（PII / 财税敏感）
  • 员工姓名 → 编号（员工 A / 员工 B）
  • 员工 TFN（Tax File Number）→ 整列删除
  • 客户/供应商真实公司名 → 编号（客户 12）
  • 真实 ABN → 假码或删除（除非分析必须）
  • 银行 BSB / 账号 → 整列删除
  • 个人邮箱 / 电话 / 地址 → 删除
✅ 可保留（且应保留）
  • 日期（保留原值，时间序列分析要）
  • 金额（保留原值，扰动会破坏归因）
  • account_code / account_name（聚合层级，不算敏感）
  • category（Revenue / COGS / OpEx 大类）
  • GST 字段（金额 + treatment）
  • tracking_category（部门 / 项目 / 渠道，已聚合）
映射表本地保存原则：『客户 12 = XXX Pty Ltd』这张映射表 不进 Filesystem 授权目录 — 用 Excel 加密码或直接打印纸版，放抽屉。Cowork 永远不该看到这张表。
7给 Cowork 的『澳洲口径』Prompt 增项 PROMPT BOOSTER

课堂上 5 个 prompt 是通用版。回去用真实 AU 数据时，每个 prompt 顶部加一段澳洲口径声明，Cowork 会少瞎猜：

AU 口径声明 · 通用前缀（粘到任何 prompt 顶部） 口径声明：本数据为澳洲单位。 - 财年（FY）：7 月 1 日 → 次年 6 月 30 日（如 FY2025 = 2024-07 至 2025-06）。 - 报送：季度 BAS（FY Q1=7-9 / Q2=10-12 / Q3=1-3 / Q4=4-6）。 - GST：amount 列为含 GST 总额（GST-inclusive），gst_amount 单列。 - 跨期对比：所有"同比/环比"按 FY 季度，不要按自然年。 - 货币：默认 AUD。 - 异常归因：考虑 BAS 报送时点（4 月 / 7 月 / 10 月 / 2 月底前后）的 timing 调整。

融合后的 Prompt ① 损益三段式（AU 版）样例：

# 顶部：AU 口径声明（同上）
# 然后：原 prompt-pack 第 ① 个 prompt（损益三段式 What/Why/So What）

你是一位资深财务分析师。我给你一份 [公司名] FY[2025 Q3] 的损益数据
（CSV 已通过 Filesystem 读取：~/Documents/AI-Workshop/standardized.csv）。

请用『What / Why / So What』三段式给我一份 1 页简报，所有跨期对比按 FY 季度。

【What】用 5-7 句话客观陈述本期数据特征 ...
【Why】3 个最可能的归因 ... 归因时考虑 BAS 报送时点（4 月底前后的 timing 调整）
【So What】3 条管理层决策建议 ...
把这段口径前缀也存成 Cowork Skill：命名 au-context，调用 /au-context 即可在任何 prompt 前自动注入。这样你 5 个会计场景 Skill + 1 个 au-context = 『AU-aware 会计 Skill 包』。
8一页 Checklist（打印贴工位） PRINTABLE

每次拿真实数据放进 Cowork 之前，按这张表过一遍：

① 数据导出
  • ☐ 从 Xero/MYOB/QB 导出 CSV
  • ☐ 时间范围按 FY 季度切
  • ☐ 含 GST 列（amount / gst_amount）
② 列名映射
  • ☐ 7 列必填齐：date / account_code / account_name / category / amount / gst_amount / contact
  • ☐ 列名英文小写连字符
③ 脱敏（不可省）
  • ☐ 员工姓名 → 编号
  • ☐ TFN / 银行账号整列删
  • ☐ 客户公司名 → 客户 N
  • ☐ 映射表加密码本地保存
④ 放进 Filesystem
  • ☐ 存到 ~/Documents/AI-Workshop/
  • ☐ Filesystem Connector 已授权该目录
  • ☐ 文件名：YYYY-MM-{slug}.csv
⑤ Prompt 加 AU 口径
  • ☐ 顶部加 au-context 前缀（或 Skill 调起）
  • ☐ 明确写 FY 季度 vs 自然年
  • ☐ 明确 GST 含税口径
⑥ 跑完复核
  • ☐ 数字：用 Excel 抽 2-3 行复算
  • ☐ 归因：问 Cowork『还有什么解释』(prompt ②)
  • ☐ 结论：自己读出来一遍听有没有套话
⏱ 习惯了之后这 6 步从原始数据到 Cowork 跑出第一份分析大概 15-20 分钟。第一次会久一些（30-45 分钟），主要花在列名映射和脱敏。把这两步存成你自己的『准备模板』（Excel 公式 + Skill），之后每月只是换数据，不用重做。
